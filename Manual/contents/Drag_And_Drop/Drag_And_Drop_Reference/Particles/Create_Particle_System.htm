创建粒子系统

这个动作将初始化粒子系统，为你创建和使用粒子类型和发射器做好准备。你将层的名称作为string ，例如 "Particle Layer"（必须存在于room 编辑器中），然后flag 。 系统是否是持久性的。如果系统被标记为持久性，那么它将在创建后的所有rooms 中保持活跃，如果初始层在随后的rooms 中不存在，它将被分配一个被管理的层。 如果没有标记持久性，那么该系统将在其创建的room 结束时自动销毁，与该系统相关的任何发射器也将被销毁。如果你想从内存中手动删除该系统，你需要使用 销毁粒子系统"的动作。

你还需要提供一个目标变量，它将保存唯一的粒子系统ID值。这个ID值将在其他函数中被用来设置系统的属性，或者告诉GameMaker在创建粒子时使用什么系统。 你可以选择将这个变量作为一个临时的局部变量flag ，但这将意味着它在动作script 或事件结束时被丢弃，这意味着这个系统仍然存在，但你没有任何方法来访问它，所以 所以在使用时必须小心（一般来说，系统永远不需要存储在一个临时的局部变量中，但是对于某些事情，比如将粒子绘制到一个表面，可能需要这样做）。

注意：一个持久的粒子系统，一旦创建，在游戏的过程中就会一直驻留在系统内存中，因此，如果你需要在任何时候重新创建它（例如，如果你调用重启游戏动作），那么你必须先销毁当前的系统，否则就会出现内存泄漏。当一个资源被添加到内存中，然后被 "取消引用 "时，就会发生内存泄漏，这意味着GameMaker没有办法与该资源进行交互。 资源进行交互。这导致内存被不可访问的资源堵塞，这反过来会导致你的游戏出现问题。要销毁不再需要的持久性粒子系统，请使用Destroy Particle System动作。
