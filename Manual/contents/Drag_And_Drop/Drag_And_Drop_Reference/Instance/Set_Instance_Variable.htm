设置实例变量

通过这个动作，你可以设置其中一个内置的变量。所有objects 的所有实例都已经在其中创建了一些变量，以保存基本信息，如当前的X/Y位置、速度、方向、可见度 和许多其他东西。你可以使用的实例变量是。

  • X坐标--在room ，沿X轴的位置。
  • Y坐标- 在room ，沿Y轴的位置。
  • 方向- 实例的运动方向（默认为0，其中右为0°，上为90°，左为180°，下为270°）。
  • 速度- 实例移动的速度（以每游戏帧的像素为单位）。
  • Horizontal Speed- 实例移动的水平速度（以每游戏帧的像素为单位）。负的速度意味着它在 左边，正的速度是右边，默认值是0。
  • Vertical Speed- 实例移动的垂直速度（以每一游戏帧的像素为单位）。负的速度意味着它在向上移动。 正的速度是向下的，默认值是0。
  • Visible- 可见性切换变量（默认为 true ，除非在对象编辑器中另外设置。 设置为 false 以使其不可见）。注意，一个不可见的实例仍然运行它的所有事件，除了绘图事件。
  • Persistent- 持久性切换变量（默认为 false ，除非在对象编辑器中另行设置。 设置为 true 来使其持久化）。这将使一个实例从room "持久化 "到room ，即：该实例不会在room 结束时被清理掉，而是被带到下一个room 。请注意，当一个实例被持久化时 它不会调用其Create或Destroy事件，但会调用Clean Up、Room End和Room Start事件。
  • Layer- 该实例被分配到的层。
  • Depth- 分配给该层的深度值。
  • Sprite- 分配给实例的sprite 资源。
  • 图像索引- 分配给sprite 的图像索引。这个值设置分配给实例的sprite 的帧数，如果 值大于动画中的帧数，它将 "循环"，例如一个有5个帧的sprite ，如果设置为第7帧，实际上将显示第2帧。
  • 图像角度- 实例应设置的角度（默认为0，其中右为0°，上为90°，左为180°，下为270°。 是270°）。
  • 图像阿尔法--实例的阿尔法值（默认为1），从0到1，其中0是完全透明，1是完全不透明。
  • Image Blend- 与实例混合的颜色sprite （默认为白色）。
  • Image Speed- 实例的动画速度。
  • 水平比例- 绘制实例sprite 的水平比例（这也会影响实例的边界 箱和碰撞）。1表示不缩放（1:1），较小的值将会缩小（例如0.5，将是sprite 宽度的一半），较大的值将会放大，负的值将会镜像sprite ，并缩小它，除非 使用的值正好是-1（在这种情况下，sprite 只是沿Y轴镜像，没有缩放）。
  • Vertical Scale- 绘制实例sprite 的垂直比例（这也会影响实例的边界框和碰撞）。 箱和碰撞）。1表示不缩放（1:1），较小的值将会缩小（例如0.5，将是sprite 高度的一半），较大的值将会放大，负值将会翻转sprite ，并缩小它，除非 使用的值正好是-1（在这种情况下，sprite 只是沿Y轴翻转，没有缩放）。
  • Mask-实例的掩码索引。默认值是-1（这意味着该实例将使用当前分配给 索引的碰撞掩码）。 当前分配的sprite 索引），但如果设置为任何其他sprite 资源，该实例的碰撞掩码将被使用，而不是分配给sprite 索引的sprite 。
  • 摩擦力- 在游戏的每一帧中使用的摩擦力，以减慢实例的速度。这个值将简单地从当前的 扣除（如果速度值是负的，则增加）。
  • 重力- 重力将在实例的重力方向（见下文）施加一个恒定的力，影响其速度（默认为 0).请注意，重力是一个累积的力，如果你不选择限制最终速度的话，会加速object 。
  • 重力方向- 这可以用来设置当实例的重力大于0时的运动方向（默认为270）。 是270）。注意，GameMaker中的方向计算为0°是右，90°是上，180°是左，270°是下。
  • Score- 设置 score 变量。
  • Lives- 设置 lives 变量。
  • 健康- 设置 health 变量。

一旦你选择了要使用的内置变量，你就可以为该变量设置值。你使用的值将取决于所选择的变量（见上面的默认值和范围列表），你可以flag ，使其与变量的当前值相对应。 变量的值。你也可以在同一个动作中添加额外的变量来进行设置，方法是点击旁边的加号图标 旁边的加号图标 旁边的加号图标，然后选择另一个实例变量并给它一个值。
