创建回滚游戏

这个动作可以主持一个新的多人游戏。它获取游戏开始时必须连接的玩家数量，并可选择允许你禁用同步测试。

当调用同步测试功能时，系统会等待指定数量的玩家加入，然后开始游戏。请确保在游戏开始前不要创建或修改任何被管理的实例。

提示你可以选择在所有玩家加入之前强制开始游戏，方法是运行开始游戏。

目前，你在一个游戏中最多只能有4名玩家。

警告不建议在这个动作中设置默认区域，除非你允许播放器通过菜单来选择它。

争论。

争论 描述
Count of players 游戏开始时必须连接的玩家数量。如果使用定义玩家，这就是自动创建的实例数量。
Sync test 指定是否应启用同步测试，默认为启用。当禁用时，在线功能被启用。
Region 指定将创建虚拟room 的区域。关于获取区域的信息，见 rollback_create_game().
