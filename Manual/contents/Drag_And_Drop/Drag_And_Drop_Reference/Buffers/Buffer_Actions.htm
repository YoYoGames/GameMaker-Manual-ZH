缓冲器动作库

缓冲区是物理内存的一个区域，当数据从一个地方移到另一个地方或以某种方式进行操作时，用来暂时保存数据，例如，你可以使用buffer 来保存游戏状态数据。GameMaker有许多与创建、修改和加载/保存buffers 有关的具体操作，但在使用它们之前，请注意以下几点。

  • 当创建一个buffer ，它就会自动被清除并填充到0。
  • buffer 函数的 "size "参数总是指buffer 的字节大小。
  • GML Visualbuffers 总是1字节对齐。对齐 "指的是数据在buffer 中的存储方式，在使用GMLbuffer 函数时使用，但对于GML Visual，它并不相关。
  • buffers 中的数据是以顺序方式读写的（即：你调用写缓冲区动作三次，那么它将写入第一个数据，向前移动，对第二个数据，向前移动，然后写入最后一个数据），有一个叫做 "寻找 "值的东西告诉你下一次读/写将在buffer 。
  • 所有的数据都被写入buffer ，并从buffer ，作为一个字符串返回，所以如果你存储了任何实数，你将不得不使用一个数据类型操作来把它从string ，转换成数值。

用于创建buffers 的内存是系统内存，所以即使游戏没有焦点，存储在buffer 的任何数据都应该是安全的，但是如果应用程序被关闭或重新启动，那么它就会丢失。还值得注意的是，buffers ，也可以保存为文件，然后使用文件操作加载到游戏中。
