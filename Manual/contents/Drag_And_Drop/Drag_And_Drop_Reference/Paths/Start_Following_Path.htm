开始跟随路径

这个动作将启动一个实例，沿着在path 浏览器中创建的Asset 。然后你将使用这个动作来告诉你的实例要跟随哪个path ，以什么速度跟随path （以每一游戏帧的像素为单位），当它到达 的终点时如何 当它到达path 的末尾时如何行动，以及是跟随绝对位置还是相对位置path 。在设置速度时，你可以使用负值来表示实例应该反向跟随path ，值得注意的是 你在这里设置的速度可以被path 编辑器中为每个path 点设置的速度修改器所修改，所以如果你将path 的速度设置为2，而在path 中有一个点的速度修改器为200%，那么这个点的实际速度将是 每个游戏帧4个像素。

一旦你设置了初始的path 速度，你就可以从以下选项中设置当实例到达path 的终点时应该如何表现（注意，当实例到达path 的终点时，这也将触发一个其他--Path 结束事件）。

  • 停止：结束path ，在最后一个path 点上停止实例。
  • 重新开始。从起点继续进行path ，如果path ，在Path 编辑器中没有标记为关闭，则再次跳到起点位置。
  • 循环。从当前位置继续，再次运行path 。
  • 反转。再沿着path （通过逆转path 跟随速度实现），当它到达path 的起点时，它将再次逆转，等等。

flag 最后，你可以把path ，因为它是绝对的 或者不是。默认情况下，实例将从它的当前位置（相对位置）开始跟随path ，但如果你flag "绝对"复选框，那么它将开始跟随 path 完全按照你在Path 编辑器中设计和放置的位置（绝对位置）。下面的图片说明了这一点。
