评论代码

当与其他人合作时，当处理一个大型项目时，甚至只是为了你自己的参考和debugging ，在你的事件代码和scripts ，留下注释是非常重要的。在GameMaker中，你有各种机制来帮助你留下关于代码部分甚至整个代码块的注释和评论，可以在你目前正在编辑的object ，在事件列表中看到。通过这种方式，你可以给自己和你的同事留下注释，或者解释一段特别棘手的代码，或者甚至只是留下一个提醒，告诉他们某件事情是什么，或者你希望某件事情在未来做什么。

要留下一个简单的单行评论，你可以做的第一件事是在文本前使用// 。比如说。

//initialize variables
sr = 10;
hp = 100;

你也可以留下多行注释，以表示感谢，省略一整段代码debugging ，甚至可以留下关于你将来要如何改变代码的注释。为此，你可以使用 /* ... */像这样。

/*
Setup GIF Variables to initial values (these can be edited so you don't need to use the setter functions all the time if you are always using the same values)

IMPORTANT! Please see the individual "set" scripts for more extensive information about each of the different variables.
*/

frame_num = 30;
frame_steps = 5;
frame_width = 256;
frame_height = 256;
frame_quality = gifcam_quality_high;

除了注释之外，你还可以使用特殊的 #region 标签来标记代码折叠的部分，每个区域可以包含一个注释。其工作原理是，你在一段代码的顶部添加一个 #region 标签--连同任何注释文本--然后在该部分的底部添加一个相应的 #endregion 标签，然后你就可以 "折叠 "代码，即：将其隐藏起来，这样你就可以在任何时候减少编辑器中可见的代码行的数量了。

实际的代码看起来是这样的。

#region This is an example region

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Further comment here if required

最后，值得一提的是，在对象编辑器中处理事件时，你可以在事件的开头使用一个特殊的"description"注释，说明事件的一般内容，这将显示在Object 编辑器的事件窗口中。这应该放在代码编辑器的第一行，其格式如下。

/// @description <This Is An Event Description>
