多次迭代一个（或多个）语句的最常用方法之一是使用 for loop ，其形式是这样的。

for (<assignment>; <expression>; <operation>;)
{
    <statement>;
    <statement>;
    ...
}

其工作原理如下--首先执行赋值，为变量赋值，然后对表达式进行评估，如果是true ，则执行大括号{}中的语句，执行。接下来对被赋值的变量进行操作，然后再次对表达式进行评估。这样loop ，直到发现表达式是false 。

现在，这样写可能听起来很复杂，但你应该把它理解为。

  1. 第一个赋值初始化了for-loop
  2. 该表达式测试是否应该结束loop 。
  3. 该声明的执行
  4. 操作的进行
  5. 回到第2步，再次执行表达式，继续或退出循环。

这对于做重复性的工作是非常有用的，因为以任何其他方式都会涉及到多行代码，而且通常被用作评估数组、绘制东西、设置增量值等的计数器。下面的代码例子说明了这种类型的语句的典型用途。

for (var i = 0; i < 10; i += 1)
{
    draw_text(32, 32 + (i * 32), string(i) + ". "+ string(scr[i]));
}

上面的代码初始化了一个 for loop ，从0开始，一直数到（包括）9，然后使用loop 的值 i ，在屏幕上绘制存储在数组中的数值。请注意 for loop 变量 i ，它不仅用来在数组中loop ，而且还用来画一个数字，并告诉GameMaker在room 。这种灵活性是 for loops 在编程中如此重要的主要原因之一。

什么时候应该使用 for loop?任何时候你需要对一个或多个语句进行固定数量的迭代，同时跟踪当前正在运行的迭代并使用该迭代值。

注意：在与其他人一起工作时，你会看到多个例子，变量 "i" 和 "j" 被用于loop 计数器变量。这些并不是强制性的变量名称，你可以使用任何像 "a" 或 "foo" 或其他的东西。使用 "i" 和 "j" 只是编程中的一个标准惯例。

值得注意的是，你也可以在一个 break 和 continue 语句中使用特殊的 for loop 。使用 break ，将立即退出loop ，并在loop 应该结束后，转入事件或函数中的任何代码，例如。

var _inst = noone;
for (var i = 0; i < 10; i += 1)
{
    _inst = instance_find(obj_Enemy_Parent, i);
    if instance_exists(_inst)
    {
        if _inst.object_index == obj_Enemy_Melee
        {
            break;
        }
    }
}
target = _inst;

上述代码loops ，通过给定的 "父 "object 的10个最近的实例，如果找到一个实例，它将检查实例object 的ID，如果它是 obj_Enemy_Melee 的实例，那么就用 break 结束loop ，并将ID值分配给一个变量（如果没有找到实例，那么关键字 noone 将被添加到该变量中）。

在 for loop ，一个使用 continue 的例子是。

var _val = 0;
for (var i = 0; i < 10; i += 1)
{
    if (val_array[i] <= 0)
    {
        continue;
    }
    _val += val_array[i];
}
draw_text(32, 32, "Positive Values Total = " + string(_val));

continue 这段代码将检查存储在一个10长度的1D数组的每个数组位置的值，如果有小于或等于0的，它将loop ，这意味着当前的迭代将结束， i 将被递增，并开始下一个loop 的迭代。如果数值大于0，那么它将被添加到本地变量 _val ，在loop ，结束后，总数值将被画到屏幕上。
