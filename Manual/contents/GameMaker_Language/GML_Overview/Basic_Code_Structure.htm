基本代码结构

代码是以块的形式编写的，一个典型的代码块由一组指令组成，称为语句，然后由GameMaker解释并用于使游戏中发生一些事情。这个 "东西 "可以简单到2和2相加得到4，也可以复杂到当敌人的健康状况低于某个值时让他们逃跑。程序的实际结构可以有很大的不同，这取决于它所使用的功能，但从基本的角度来说，它看起来就是这样。

<statement>;
<statement>;
...

语句应该用'；'符号隔开，以防止变量声明出错，并保持代码的干净整洁，它们可以由变量声明、表达式和对特定函数的调用组成。你也可以用大括号 {} ，将语句作为一个块 "分组"，使它们一起运行，如下面的条件例子。

if (<conditional>)
{
    <statement>;
    <statement>;
    ...
}

注意 GameMaker Language 也会接受 begin 和 end 而不是大括号 {} ，尽管这通常不是最常见的方式。

if (<conditional>)
begin
    <statement>;
    <statement>;
    ...
end

下面是一个更直观的代码块的样子，这次是在GameMaker 脚本编辑器中作为脚本创建的。

有许多不同类型的语句、表达式、条件式和函数，所有这些都将在手册的后续章节中详细讨论。

注意如果你是编程新手，那么你可能想在继续之前查看一下《快速入门指南》。
