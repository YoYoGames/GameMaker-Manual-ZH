运行时功能

函数的一般定义是这样的。

一个函数有一个输入和一个输出，而输出与输入有关。

在GameMaker中，这转化为两种不同的东西，但它们都以同样的方式工作。

  • 运行时函数-运行时函数是作为GameMaker语言的一部分提供的(GML)。这些都是语言中内置的，可以用来在你的游戏中发生一些事情，有大量的GML runtime ，所有这些都在手册的语言参考部分有解释。
  • 脚本函数和方法 脚本 职能和 M方法变量是你使用GameMaker语言的基本构件自己创建的函数(GML)，这些函数可以包括runtime ，作为其代码的一部分。

在上述两种情况下，一个函数的形式是一个函数名，后面是在大括号 () 之间的输入参数，并用逗号隔开（如果函数没有输入参数，则只使用大括号）。本页集中讨论运行时函数（即：内置到GameMaker ，形成GameMaker Language ），但下面所示的一般规则适用于所有函数类型。

下面是一个函数的结构概要。

<function>(<arg0>, <arg1> ,... <argN>);

一些函数可以返回值，并可以在表达式中使用，而另一些函数只是执行命令，如下面两个runtime 函数的例子所示。

// Destroy the calling instance
// This requires no arguments and returns nothing
instance_destroy();

// Get the distance from the current instance position to the mouse position
// This takes four arguments and returns a real value
dist = point_distance(x, y, mouse_x, mouse_y);

注意凡是手册中以 "N/A "作为函数返回值的地方，都意味着该函数不打算作为其操作的一部分而返回任何值。如果你试图从这些函数中检查一个返回值，那么你将得到一个值 undefined 。

有时一个函数的返回值可能是你想在赋值中使用的值，但你应该注意，不可能直接使用任何函数作为赋值的左侧。例如，下面这段在实例上设置速度的代码会给你一个错误。

instance_nearest(x, y, obj).speed = 0;

该代码示例中表达式的返回值是一个整数（最近的实例的唯一ID值），因此必须用括号括起来，才能以这种方式使用，并正确寻址所需的实例（参见在其他实例中寻址变量以获得更多信息）。上面的代码可以正确地写成。

(instance_nearest(x, y, obj)).speed = 0;

//or more correctly still

var inst = instance_nearest(x, y, obj);
inst.speed = 0;

function test ()
{
    test();

    if (test)
    {
        test();
    }
}

手册的语言参考部分列出了所有可用的runtime 函数，并给出了如何使用这些函数的例子，以及关于它们可能返回的内容或可能触发的任何事件等其他重要信息。关于用户定义的script 函数和方法的更多信息，请参见 "脚本函数"和 "方法变量"部分。
