绘制路径

通过这个函数，你可以让GameMaker在屏幕上绘制一个path 。path 将被绘制成一条简单的线，可以是相对于调用实例的，也可以是在path 编辑器中或通过代码创建的绝对位置。这个函数在debugging 动态paths （例如，为实例创建的 mp_grid_path()函数）。

注意：如果你想用shader ，你应该知道，大多数shaders ，希望有以下输入。vertex,texture, 颜色。然而，当使用这个函数时，只有vertex 和颜色数据被传入，因此shader 可能不会画出任何东西（或画出东西但不正确）。如果你需要以这种方式绘制形状，那么在定制shader 的时候就应该考虑到这一点。

语法。

draw_path(path, x, y, absolute);

争论类型 描述
path The path to draw
x The x coordinate of where the path is drawn
y The y coordinate of where the path is drawn
absolute Whether the path is drawn at the absolute position (true) 或相对位置 (false)

返回。

例子。

if mp_grid_path(grid, path, x, y, obj_Player.x, obj_Player.y, 1)
{
    draw_path(path, x, y, false);
}

上面的代码将使用 mp_grid_path 函数来生成一个path ，并将其存储在变量 "path "中。如果成功地创建了path ，它就会在屏幕上以相对于运行代码的实例的位置绘制出来。
