精灵和瓷砖

即使你可以在room 编辑器中设置背景，你也可以给实例分配sprites ，但对于你正在制作的游戏来说，这往往不够灵活。例如，如果你有一个必须绘制菜单的实例，你可能想在同一时间显示各种按钮sprites ，或者如果你有一个带有滚动视差背景的游戏，你将需要在某种类型的控制器实例中通过代码设置它们。在这些情况下，你应该使用以下函数，这些函数可以在sprite 和背景assets ，在room ，其中一些函数还允许你操作sprite 的形状、大小、颜色和alpha，以达到特殊效果。

你可以给一个实例指定一个sprite ，默认情况下GameMaker会用各种精灵变量给出的属性来绘制它，但是如果你在绘制事件中添加任何东西，你就 "控制 "了，必须告诉GameMaker到底要绘制什么。这意味着，如果你必须使用以下函数来定义实例要绘制的内容，以及它在room 。

  • draw_self
  • draw_sprite
  • draw_sprite_ext
  • draw_sprite_general
  • draw_sprite_part
  • draw_sprite_part_ext
  • draw_sprite_stretched
  • draw_sprite_stretched_ext
  • draw_sprite_pos
  • draw_sprite_tiled
  • draw_sprite_tiled_ext

如果你使用的是SWF格式的sprites ，那么还有额外的选项来控制绘制时使用的抗锯齿（关于这些类型的sprites ，请参阅导入非位图精灵的更多信息）。请注意，这些函数是在全局层面上工作的，在它们被调用后，将对所有绘制的矢量SWFsprites 设置AA。

  • draw_enable_swf_aa
  • draw_set_swf_aa_level
  • draw_get_swf_aa_level

除了上面列出的那些函数，还有两个专门为骨骼动画设计的函数（关于这些类型的sprites ，请参阅导入非位图精灵的更多信息，以及与这些类型的sprites 有关的具体函数的骨骼动画部分）。

  • draw_skeleton
  • draw_skeleton_instance
  • draw_skeleton_collision
  • draw_skeleton_time

瓷砖是根据其包含的数据、使用的tile set 和tile 地图属性自动绘制的。然而，你可以使用以下函数绘制单个tiles 或tile 地图。

  • draw_tile
  • draw_tilemap
