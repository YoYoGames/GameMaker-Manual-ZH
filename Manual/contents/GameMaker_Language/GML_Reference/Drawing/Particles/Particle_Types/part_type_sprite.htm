部分_类型_斯普利特

这个函数可以用来设置一个粒子类型，以使用游戏中的自定义sprite assets 。你可以选择sprite ，然后告诉GameMaker使用sprite ，并对其进行动画处理或不处理，如果你选择动画处理，那么你还可以选择让子图像以一定的间隔变化，这样动画就会在创建的每个粒子的寿命开始和结束时开始和结束（因此，如果你将寿命设置为最小15步，最大30步，无论寿命有什么值，都会使动画适应，寿命越短运行越快，越长则越慢）。你也可以选择选择一个随机的子图像，这样如果sprite ，就会随机选择一个不同的图像来创建sprite ，或者如果粒子有动画，就会在一个随机点开始动画。

语法。

part_type_sprite(ind, sprite, animate, stretch, random);

争论 类型 描述
ind 粒子类型ID The index of the particle type to change.
sprite 雪碧资产 The index of the sprite to set the particle type to.
animate 布尔型 Whether to follow the sprite's animation normally (true) or not (false).
stretch 布尔型 Whether to stretch the sprite's animation to match the particle lifespan (true) or not (false).
random 布尔型 Whether to choose a random sub-image (true) or not (false).

返回。

N/A

例子。

global.p1 = part_type_create();
part_type_sprite(global.p1 , spr_Coins, 1, 0, 0);
part_type_size(global.p1, 1, 3, 0, 0);
part_type_scale(global.p1, 1, 1);
part_type_colour1(global.p1, c_white);
part_type_alpha2(global.p1, 1, 0);
part_type_speed(global.p1, 2, 4, 0, 0);
part_type_direction(global.p1, 0, 180, 0, 0);
part_type_gravity(global.p1, 0.20, 270);
part_type_orientation(global.p1, 0, 0, 0, 0, 1);
part_type_blend(global.p1, 1);
part_type_life(global.p1, 15, 60);

上述代码将全局变量 "p1 "中索引的粒子类型设置为 "spr_Coins "中索引的sprite 的形状，然后设置其他各种粒子属性。
