粒子类型

粒子是可以通过在游戏中使用粒子系统创建的图形效果。你用这些函数定义的粒子可以在你创建的任何系统中使用，通常的程序是在游戏开始时一次性定义所有的粒子，并将它们的索引值存储在全局变量中。然后，这些全局变量可以在整个游戏过程中的任何时候用于爆裂、发射或创建粒子。如果内存是个问题，或者你希望有许多不同类型的粒子，但只在游戏中的特定点使用，你也可以根据需要创建、使用和销毁粒子。无论你选择什么，粒子都是一种非常通用的资源。

粒子需要被创建，并将其索引存储在一个变量中，以便所有进一步的粒子例程能够正确访问它们。以下函数用于创建、检查、清除和销毁粒子类型。

  • part_type_exists
  • part_type_create
  • part_type_destroy
  • part_type_clear

一旦你创建了你的粒子类型，你应该给它一个形状。这可以是14个内置的形状之一，这些形状是64x64像素的sprites ，并带有阿尔法混合，或者你可以使用你自己的sprites ，它可以是动画的、静态的或使用随机的子图像。下面的函数定义了每个粒子的视觉基本形状。

  • part_type_shape
  • part_type_sprite

下图显示了你可以选择的14种内置粒子类型。

一旦你为粒子类型选择了sprite （无论是默认的形状还是你自己的形状），你就可以指出它的大小和比例。尺寸可以被指定为一个范围，所有的粒子都会在给定的范围内选择一个尺寸来创建。在添加任何尺寸变化之前，比例是粒子的X和Y轴比例的 "基础 "值，以下函数用于控制这些值。

  • part_type_size
  • part_type_scale

粒子也可以有一个方向和运动。与大多数粒子函数一样，速度、方向和方位被解析为一个随机值，从一个给定的范围内选择，可以用以下函数来设置。

  • part_type_speed
  • part_type_direction
  • part_type_gravity
  • part_type_orientation

一旦你确定了你的粒子的风格和运动，你就可以设置它们的颜色和阿尔法值，这两个值可以在粒子的整个生命周期内设置为一个绝对值，或者设置为一个逐渐从一个到另一个的刻度值。请注意，还有一个特殊的功能，允许你设置粒子和它们周围环境的混合，这可以是正常的，也可以是加法的，并提供一些奇妙的效果--管理颜色、阿尔法和混合的功能是。

  • part_type_colour_mix
  • part_type_colour_rgb
  • part_type_colour_hsv
  • part_type_colour1
  • part_type_colour2
  • part_type_colour3
  • part_type_alpha1
  • part_type_alpha2
  • part_type_alpha3
  • part_type_blend

粒子在屏幕上存在的时间是有限的，这被称为其 "寿命"。粒子的寿命是以步数来衡量的，（和以前的函数一样）这被表示为一个数值范围，粒子将从中得到一个特定的寿命。还有几个有趣的函数可以用来使粒子产生其他的粒子，无论是每一步还是当它们 "死亡 "时（即：从屏幕上移开），但要非常小心使用这些函数，因为它们可以迅速产生成千上万的粒子并使游戏崩溃设置寿命和步骤或死亡发射的函数如下。

  • part_type_life
  • part_type_step
  • part_type_death

值得注意的是，由于有大量的参数可以改变和调整，通过代码创建粒子往往是一个费力和费时的过程。出于这个原因，我们建议你四处搜索并获得某种类型的粒子设计器（或自己制作一个！），以消除粒子创建的苦难。
