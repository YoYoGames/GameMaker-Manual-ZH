弦乐

在制作游戏的某个时候，你将需要使用文本。游戏中的文本是通过使用字符串函数来处理的（一个string 只是表示一行文本的另一种方式），GameMaker有一套完整的函数，允许你以多种方式处理strings ，包括在另一个string ，复制strings ，并能够解析strings ，以确定其包含的数字或字母。一般来说，string 只能通过在双引号""内添加文本来创建，不接受单引号strings ，也不能将string 分成多行，并期望GameMaker将其作为换行符来呈现（除非使用字符串字面的 @ 标识符，如下文所述）。

值得注意的是，在创建strings ，有一些惯例可以使用，主要涉及到使用转义字符。这些字符前面有一个"\"符号。因此，举例来说，如果你想把引号放在string ，你会有这样的结果。

str = "Hello\"World\"";

GameMaker也有完整的四字节宽的 Unicode 字符支持，允许你在标准的上限范围内解码和编码 Unicode 字符（包括 - 但不限于 - 表情符号）。为了处理Unicode字符，你可以使用"\"在任何Unicode字面前--在 "u"前的十六进制数字，例如 "á "的 "\u00e2"--其中的数字是Unicode字符的编号。当以这种方式处理Unicode时，你需要注意GameMaker将解释"u"之后的所有数字，所以如果你想写 "áa"，你应该使用。

"\u00e2\a"

或

"\u00e2\u61"

或

"\u00e2" + "a"

因为仅仅使用 "\u00e2a" 实际上会导致Unicode字符 "ส" （实质上成为 "\ue2a"）。

GameMaker还可以处理任何十六进制的文字--通常写成 "0x"后面的十六进制数字，例如 "0xff"，其中的数字是要使用的字符的编号。在GameMaker中，这些都是用 "\x"写的，然后是十六进制值。这些和其他预定义的转义字符在下表中列出。

恒定 描述
\n 换行
\r 回车
\b 退格 (0x08)
\f 换页 (0x0c)
\t 水平标签 (0x09)
\v 垂直标签 (0x0b)
\\ 反斜杠本身 (0x5c)
\a 警报 (0x07)
\u[Hex Digits] 插入Unicode字符
\x[Hex Digits] 插入十六进制字头字符
\[Octal Digits] 插入八进制的Unicode字符

注意：Strings 支持form feed, vertical tab等...，但这并不意味着说渲染也支持，在绘制strings ，这些字符可能被忽略。

你也可以通过在整个string ，在前面加上 @ 字符来创建逐字字符串。

var test = @"
Line breaks
over multiple
lines
";

上述代码将在多行上呈现string ，就像包含有一个换行符一样。一个逐字记录的string 字符与以前的GameMaker 版本string 字符相似，但它们也使用双引号或单引号，并且必须以 @ 符号为前缀，它们可以在代码文件中分多行显示，并且它们不支持转义字符，即 @"Hello\World" 不会尝试转义World上的W，并且将逐字存储。但是请注意，当像这样使用string 字母时，如果你想把引号作为string 的一部分，你将需要打破string ，即。

var test = @"Hello " + "\"" + @"World" + "\""

另一件要注意的事情是，Unicode字符9647（▯）是用来替代你在绘制事件中渲染你指定的font 中可能有的任何缺失的文字符号。因此，如果你的font 中没有，例如，°符号，那么写90°实际上会产生90▯。
