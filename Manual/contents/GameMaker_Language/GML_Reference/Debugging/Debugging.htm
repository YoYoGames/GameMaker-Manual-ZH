调试

在制作游戏的过程中，你往往需要知道游戏机制在特定时间点上的某些细节，或者能够为某些函数输入某些键值，以测试出游戏的不同方法和想法，这一点非常重要。为此，GameMaker为你提供了一些调试函数，这些函数在调试模式下（或在Windows 目标上）会显示信息并要求输入，但在为其他目标模块编译时，这些函数会被忽略，这是因为这些函数会使游戏处于封闭的loop ，直到输入被解决。

注意虽然他们可能在某些平台上工作，但他们不能用于任何其他用途，debugging 。

  • debug_mode
  • debug_event
  • debug_get_callstack
  • exception_unhandled_handler
  • get_integer
  • get_string
  • show_error
  • show_message
  • show_question
  • show_debug_message
  • show_debug_overlay
  • code_is_compiled
  • fps
  • fps_real

除了这些专业功能，为了帮助你创建游戏，GameMaker还有一长串错误信息，可以在编写游戏代码、编译游戏或游戏本身运行时显示。这些信息指出了有关的具体错误，可以用来确定特定的object 或script 功能中的问题，对debugging 你的游戏的过程有很大帮助。然而，错误信息的确切含义可能并不总是很明显，因此下面的章节列出了所有可能的信息，并解释了它们的含义，以使你的生活更容易。

  • 语法错误
  • 编译器错误
  • 跑步者的错误

你还可以在 "调试器"部分找到关于如何调试你的游戏的进一步信息。
