buffer_async_group_option

通过这个功能，你可以为正在保存/加载的buffer 组设置一些平台特定的选项。可用的选项如下。

  • "subtitle" 或 "slottitle" - 这个选项的值将是一个string ，当用户在操作系统中管理他们的保存数据时，这个值将显示给用户。这只在保存数据时很重要，在再次加载时不重要。
  • "showdialog" - 这个选项的值是一个布尔值 true 或 false 。如果设置为 true ，当你保存或加载时，GameMaker会显示系统用户界面，否则它将做一个快速保存/快速加载，不显示用户界面。通常只有当你支持多个保存槽时才需要这个选项，以允许用户选择一个槽，但如果你只支持每个用户一个槽，将其设置为 false 。
  • "savepadindex" - 对于这个选项，你将指定正在保存或加载的玩家的pad索引，系统将向这个玩家的保存文件夹写入数据并从其读取数据。
  • "saveslotsize" - 这个选项要求你指定你想保存的实际大小（以字节为单位）（所以你可以做一个 buffer_seek()和 buffer_tell()来得到这个值）。请注意，提供这个值不是必须的，因为所有的保存都预先分配了一个最小的空间，这个空间通常随目标平台而变化。
  • "ps_create_backup" -(仅PlayStation)- 将此设置为 true ，确保在保存目录被卸载时（即调用 buffer_async_group_end() 函数时），保存的数据会被备份。更多信息请参见控制台文档。

重要的是!该功能目前只对PSVita、PS4和XBox One 目标模块有效。在所有其他目标上，它不会做任何事情。

语法。

buffer_async_group_option(option, value);

争论 类型 描述
option 字符串 The option to set.
value 真实或 弦乐 The value to set (can be a real or string, depending on the option).

返回。

N/A

例子。

buffer_async_group_begin("save_folder_name");
buffer_async_group_option("showdialog", false);
buffer_async_group_option("slottitle", "Catch The Haggis Save");
buffer_async_group_option("subtitle", "All your haggis are saved here!");
save = buffer_save_async(buff, "Player_Save.sav", 0, 16384);
buffer_async_group_end();

上面的代码启动了一个buffer 组，然后在它设置4个文件异步保存之前设置了组的选项。然后结束组的定义（这时将开始保存）。
