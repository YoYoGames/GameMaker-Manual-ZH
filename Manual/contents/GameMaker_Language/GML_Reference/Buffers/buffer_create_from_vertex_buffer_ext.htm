buffer_create_from_vertex_buffer_ext

你使用这个函数在你的游戏中分配一部分内存作为buffer ，其中充满了先前创建的顶点缓冲区的数据。该函数返回一个唯一的缓冲区ID，该ID应该存储在一个变量中，并用于所有对buffer 的进一步函数调用。该函数为vertex buffer 使用的ID（如函数返回的ID）。 vertex_create_buffer()例如），并使用以下常数来定义buffer 类型。

恒定 描述
buffer_fixed 一个固定尺寸的buffer 。
buffer_grow 一个buffer ，它将随着数据的增加而动态地 "增长"。
buffer_wrap 一个buffer ，其中的数据将被 "包裹 "起来。当正在添加的数据达到buffer 大小的极限时，覆盖的数据将被放回buffer 的起点，并从该点继续写下去。
buffer_fast 特殊的 "剥离 "buffer ，读/写速度极快。只能与 buffer_u8 数据类型一起使用，并且必须是1字节对齐。
buffer_vbuffer 这种类型的buffer ，只能作为一个vertex buffer 。

除了buffer ，你还必须设置buffer 的字节对齐。这个值将取决于你希望存储在buffer 的数据，在大多数情况下，值为1是完全可以的。然而，请注意，对于某些操作来说，特定的对齐方式是必不可少的，不正确的对齐方式可能会导致错误（关于对齐方式的进一步细节，请参见缓冲区对齐方式）。下面是一个一般的指南，说明哪种值最适合于每种数据类型（有疑问时，使用1的对齐方式）。

  • 字符串应该被对齐到1个字节。
  • 有符号或无符号的8位整数可以对准任何数值，但请注意，对于快速buffer （见 buffer_write())，它必须对齐到1。
  • 有符号或无符号的16位整数应该被对齐到2个字节。
  • 有符号或无符号的32位整数应该被对齐到4个字节。
  • 16位以内的浮点应该被对齐到2个字节。(目前不支持！)
  • 多达32位的浮点应该被对齐到4个字节。
  • 多达64位的浮点应该被对齐到8个字节。

注意：Vertex buffers 是1字节对齐的，但你可以根据你想处理数据的方式，用任何对齐方式创建buffer ，因为vertex 数据只是一个原始内存拷贝到buffer 。

最后两个参数用于指定你希望复制到新创建的vertex 的buffer 数据的范围。开始vertex ，可以是0和顶点数-1之间的任何地方，然后你给出从该点开始要复制的顶点数。你可以在vertex buffer 上使用函数 vertex_get_number() 来获得存储的顶点总数。

注意：当你不再需要这样的动态创建的资源时，必须从内存中删除，以防止内存泄漏，所以当你完成了你所创建的buffer ，你应该再次使用 buffer_delete().还要注意，用于创建新的vertex 的buffer buffer 也不会从内存中删除，你应该使用函数 vertex_delete_buffer()当它不再被需要时，你应该使用该函数。

语法。

buffer_create_from_vertex_buffer_ext(vertex_buffer, type, alignment, start_vertex, num_vertices)

争论 类型 描述
vertex_buffer 顶点Buffer ID The vertex buffer index to use.
type 缓冲区类型常数 The type of buffer to create (see the constants list below).
alignment 真实的 The byte alignment for the buffer
start_vertex 真实的 The starting vertex.
num_vertices 真实的 The total number of vertices to copy.

返回。

Buffer ID

例子。

var v_num = vertex_get_number(model_buff); player_buffer = buffer_create_from_vertex_buffer(model_buffer, buffer_grow, 1, 0, v_num - 1);

上述代码为一个buffer ，然后将所有顶点的数据从给定的vertex buffer 中复制到其中，并返回新buffer 的索引，该索引被存储在变量 "player_buffer "中，以供将来使用。
