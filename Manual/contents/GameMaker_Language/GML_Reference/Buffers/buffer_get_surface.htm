buffer_get_surface

通过这个函数，你可以把一个表面的信息写到一个指定的buffer 。buffer 必须是之前创建的，并且应该是一个1字节对齐的buffer ，其大小足以存储你要写的表面的数据（如果不确定，可以使用一个增长缓冲区或看本页面底部的例子）。

表面的信息将总是被写到buffer 的开头（不管寻址位置如何），但你可以选择一个偏移值（以字节为单位），将写的位置从buffer 的开头移动。

请注意，该函数在buffer 目标上使用RGBA格式将表面的每个像素写到Windows ，但在其他目标上可能有所不同，这取决于操作系统甚至是设备。

注意：从GameMaker的2.3.1版本开始，这个函数已经发生了变化。以前的版本使用常量 buffer_surface_copy ，现在已经废弃了。

语法。

buffer_get_surface(buffer, surface, offset);

争论 类型 描述
buffer 缓冲区ID The index of the buffer to use.
surface 表面标识 The index of the surface to use.
offset 真实的 The data offset value (in bytes).

返回。

N/A

例子。

buffer = buffer_create(surf_width * surf_height * 4, buffer_fixed, 1);
buffer_get_surface(buffer, surface, 0);

这段代码将创建一个固定大小的buffer ，它的容量为每个像素存储4个字节（用于R、G、B和A成分），基于曲面的确切大小，然后将曲面数据复制到其中。
