buffer_async_group_begin

当你想开始把多个buffers 保存到多个文件（或加载多个buffers ）时，就会调用这个函数。组名 string 是一个path ，将作为文件保存的目录名，在以后加载文件到IDE （使用任何一个 buffer_load()函数）。这个函数只适用于 buffer_save_async()和 buffer_load_async() ，你还必须通过调用 buffer_async_group_end()函数，否则文件将不会被保存/加载。

请注意，对于控制台平台（例如PS4），组名将被用作保存槽的描述，使用这个功能可以帮助你避免每一个被保存出来的文件的UI显示。还要注意的是，当使用UWP时，你可以在原生的Xbox和微软商店窗口之间漫游你的保存游戏，例如，通过创建一个与ERA Xbox组相同的组。

if os_type == os_uwp
{
    buffer_async_group_begin("root/yourgroupname");
}
else
{
    buffer_async_group_begin("yourgroupname");
}

语法。

buffer_async_group_begin(groupname);

争论 类型 描述
groupname 字符串 The name of the group (as a string).

返回。

N/A

例子。

buffer_async_group_begin("SaveGame");
save1 = buffer_save_async(buff1, "Player_Save1.sav", 0, 16384);
save2 = buffer_save_async(buff2, "Player_Save2.sav", 0, 16384);
save3 = buffer_save_async(buff3, "Player_Save3.sav", 0, 16384);
save4 = buffer_save_async(buff4, "Player_Save4.sav", 0, 16384);
buffer_async_group_end();

上面的代码启动了一个buffer 组，然后将其设置为异步保存出4个文件。然后结束该组的定义（此时开始保存）。
