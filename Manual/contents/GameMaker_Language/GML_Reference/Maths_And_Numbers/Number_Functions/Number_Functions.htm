数字功能

GameMaker中的实数被认为是双精度浮点数--也就是说，它们有一个小数点，在小数点之前或之后都没有固定的数字，或者是整数--也就是说，它们是没有小数点值的整数。例如，2是一个整数，但2.01是一个浮点实数。

注意：在HTML5目标上，所有实数都是双数。

在处理跨平台开发时，整数和浮点数之间的这种区别非常重要，因为在Windows PC上进行的计算的精度与在移动设备上进行的这些计算的精度是不一样的。这意味着你在进行比较时要特别注意，比如说。

if (image_index == 1)
{
    // Do something...
}

在上面的例子中，如果我们一直将 image_speed 设置为0.25--例如--那么在四步之后，你可能会认为 image_index 的值是1，但是由于浮点数学的工作方式，它可能是一个像1.0000002的值，因此评估将不会是 true 。这些类型的错误可能相当难以调试，因此最好能意识到这些错误，并提前计划，使用其他方法来检查数值，或者使用适当的落地或四舍五入函数（列在下面），将要检查的数字转换成整数（关于浮点数学的更多信息以及为什么这是一个问题，请看这里）。例如，上面的代码可以写成。

if (floor(image_index) == 1)
{
    // Do something...
}

还值得注意的是，当使用YoYo Compiler目标时，所有的表达式和函数都是从左到右评估的，而在所有其他目标平台上，它们是从右到左评估的，这意味着--例如--会根据平台的不同得到不同的结果。

val = max(num, ++num, num++);

注意：更多信息，请参见 "评价顺序"部分。

你还可以使用GameMaker中的一个特殊函数来设置浮点数学的epsilon值。当一个实数被四舍五入到最接近的浮点数时，epsilon（也称为 "机器epsilon"）形成了相对误差的上限，你可以使用以下函数获得和设置epsilon值。

  • math_set_epsilon
  • math_get_epsilon

这些函数都涉及到使用随机数和数值。

注意 当使用随机函数时，GameMaker在你每次启动游戏时都会保持相同的随机种子。这使得debugging ，因为你可以保证随机函数最初总是返回相同的值，但是如果你不希望发生这种情况，你必须在游戏开始时首先设置一个新的随机种子，可以使用 randomise()或 random_set_seed().

  • choose
  • random
  • random_range
  • irandom
  • irandom_range
  • random_set_seed
  • random_get_seed
  • randomise

这些都是以某种方式对数值进行四舍五入的函数，或者从各种给定的数值中选择一个单一数值。

  • round
  • floor

  • sign
