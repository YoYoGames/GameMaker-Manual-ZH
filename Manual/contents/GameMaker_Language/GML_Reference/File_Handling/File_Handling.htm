文件处理

在创建游戏的过程中，有必要从外部存储信息并能够检索这些信息。例如，你可能希望存储登录信息，或高分，或者是玩家通过关卡的进展情况。不管是什么原因，你都需要知道如何在使用GameMaker时使用不同的文件功能。

非常重要的一点是，GameMaker默认是沙盒式的 ，这意味着在使用可用的文件系统功能时有一些限制和小麻烦，而且值得注意的是，文件系统的工作方式会因你所针对的平台而略有不同。欲了解更多信息，请参见以下部分。

  • 文件系统

你可以从所选平台的游戏选项中关闭桌面目标(Windows,macOS 和Ubuntu (Linux))的沙箱，这意味着你可以在操作系统的权限范围内，从任何地方保存和加载。

如果你已经被Steam ，允许在你的游戏中使用他们的专用API，你会发现也有一套新的文件功能可以使用。这些在本手册的Steam API部分有解释。

最后，你还应该注意到，某些目标平台之间有一些明显的差异，在使用文件功能时将会被考虑到。这些是

  • iOS。在iOS ，如果你从一个文件夹加载一个包含的文件，即：在path ，像 "Backgrounds/background1.png"，你不需要把文件夹作为path 的一部分，只要提供文件名就足够了。在所有其他平台上，你必须将文件夹作为文件的一部分提供给path 。
  • macOS和Ubuntu (Linux)。在macOS 和Ubuntu 上，文件名和paths 在这些平台上是区分大小写的，因此为了防止任何潜在的问题，GameMaker将自动使所有名称/路径默认使用小写。
  • HTML5。在HTML5上，保存和加载到本地存储，将按照不同功能页面中的解释进行，但是如果你要从server ，你必须只使用功能buffer_load_async()，因为同步加载在大多数浏览器上已经过时，最终会被淘汰。还要注意的是，你不能在浏览器本地存储中创建或销毁目录。
