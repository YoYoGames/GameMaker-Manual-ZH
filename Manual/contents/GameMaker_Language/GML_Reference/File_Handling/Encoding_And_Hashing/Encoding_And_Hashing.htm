编码和哈希

在处理外部文件时，总是有这样的可能性，即最终用户可以打开并改变它们所包含的信息，从而改变你的游戏。这可能会导致游戏元素的损坏或在线分数的欺诈（例如），因此GameMaker为你提供了一些基本的编码功能，以及对strings 和文件进行散列检查的功能，以确保它们在使用前保持其完整性。还提供了一些用于编码和解码JSON格式的函数strings 。

注意 编码不是加密!base64编码使文件无法用肉眼看清，需要用户努力解码，但它并不安全，不能防止黑客入侵。script 建议你将这些功能与你自己的加密混合起来（互联网上有许多形式的加密和GameMaker 的功能）。
