文本文件

文本文件是一种从游戏外部存储大量数据的有用方法。与ini文件相反，你可以同时打开多达32个文本文件，你也可以向它们读写大块的数据。然而，由于没有像*.ini 文件那样固定的 "部分 "或 "关键 "结构，所以数据的结构由你决定。

请注意，对于带有本地化的游戏来说，可以从外部文件读取重音字母是非常重要的。这意味着你应该首先创建UTF8格式的txt文件，然后将其添加到GameMaker中，作为一个包含的文件，这样它就会在运行游戏时被导出，并代替默认的ANSI格式的txt文件，该文件是由GameMaker的文件功能在之前没有提供文件时创建的。通过这种方式，你可以正确地阅读和写入它，所有的重音和非罗马字母都被保留。

存在以下处理文件的功能。

  • file_text_open_read
  • file_text_open_write
  • file_text_open_append
  • file_text_open_from_string
  • file_text_read_real
  • file_text_read_string
  • file_text_readln
  • file_text_write_real
  • file_text_write_string
  • file_text_writeln
  • file_text_eoln
  • file_text_eof
  • file_text_close
