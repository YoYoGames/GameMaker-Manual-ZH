工作目录

Working_directory 实际上可以返回两个不同的值，这取决于你使用它的目的。

如果你要把文件写入磁盘， working_directory ，指向目标设备上为你的游戏保留的本地存储区域（在Windows ，这是 "%LOCALAPPDATA%/gamedir/" ，其中 "gamedir "是你的游戏名称的目录）。

然而，当从磁盘读取时， working_directory 可以是本地存储(game_save_id)，也可以是存储Included Files的区域。

注意 working_directory 变量将返回包括最后反斜杠在内的path 。

例如，如果你有一个默认的 *.txt 文件，并使用 working_directory 将其读入内存，那么该文件将从存储包括文件的区域中取出。如果你选择使用 working_directory 将该信息写入文本文件，这将把文件写入目标平台的本地存储，下次将从那里读取，除非你指示GameMaker删除该文件，在这种情况下， working_directory 将再次指向存储 Included Files 的区域。

值得注意的是，一般来说，你不需要使用这个，只需指定文件名本身（加上任何额外的path 信息），这就足够了。例如，这个。

file_text_open_read(working_directory + "my_file.txt");

最好是这样做。

file_text_open_read("my_file.txt");

警告：由于GameMaker是沙盒式的，这个功能可能不会像你预期的那样工作。请参阅文件系统一节以了解更多信息。

语法。

working_directory

返回。

String

例子。

ini_open(working_directory + "temp_ini.ini");

这将从游戏的工作目录中打开一个ini文件（如果它不存在，则创建它）。这可能是本地存储区，也可能是包含文件的区域，取决于要寻找的文件是否存在于这两个地方。
