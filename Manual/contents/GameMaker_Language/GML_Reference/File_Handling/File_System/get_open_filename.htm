get_open_filename

这个函数打开一个对话，要求玩家提供一个文件名，用给定的过滤器打开。过滤器的形式是 "name1|mask1|name2|mask2|..."，其中掩码可以包含不同的选项，中间有分号，你也可以用 "*" 表示任何string 。例如："bitmaps|*.bmp;*.wmf"，将要求用户从系统中选择一个位图文件的名称，但只有那些有两个指定的extensions (bmp 和 png)。这个函数不会打开文件本身!它只返回一个带有文件全名和string 的path 。如果用户按"取消"，将返回一个空的string ""。

使用这个功能时要注意的一件事是，它授予你该文件的某些权限，在你的游戏过程中。因此，一旦你从用户那里得到了文件path ，你就可以一次又一次地访问它，而不需要询问。然而，由于这个功能是用来读取文件的，所以授予的权限只用于读取，保存将不被操作系统所允许。如果你需要对文件进行读写，你可以使用函数获得权限 get_save_filename().

注意：这个功能只对Windows 和macOS 目标有效。

语法。

get_open_filename(filter, fname);

争论类型 描述
filter The file type filter and/or name.
fname The suggested filename to use.

返回。

例子。

var file;
file = get_open_filename("text file|*.txt", "");
if (file != "")
{
    file_text_open_read(file);
}

上述代码将要求用户选择一个文本文件进行打开，然后检查返回的文件名和path ，如果它存在，将打开该文件进行阅读。

返回。文件系统
