匹配性的查找

这个函数将使用会话模板、匹配跳板和安全设备协会模板搜索多人游戏会话，你应该事先在XDP仪表板上创建这些模板。你可以通过调用函数来检索特定游戏板的用户ID。 xboxlive_user_for_pad()函数。该函数将返回一个独特的请求ID值，然后可以用来确定该函数的正确的社交异步事件。这个事件将在会话被创建时被触发，并在变量中包含一个DS Map async_load中包含一个DS Map，有以下键/值对。

  • "requestid" - 包含由调用函数返回的请求ID
  • "status" - 包含string "session_created"，以告知你已经触发了什么类型的事件。
  • "num_results" - 如果发现了一个会话，则包含1或更多，否则包含0。
  • "sessionid<index>" - 包含会话ID，如果失败则为-1（<index/gt;是一个从0到 "num_results"-1的值）。
  • "sessionOwner<index>"- 包含会话主机的ID，如果已经找到了一个会话（<index>是一个从0到 "num_results"-1的值）。
  • "error" - 成功时包含0，失败时包含-1

语法。

xboxlive_matchmaking_find(user_id, template, hopper, sdatemplate, [matchattributes]);

争论 类型 描述
user_id Xbox用户ID The user ID pointer to use
template 字符串 The name of the session template
hopper 字符串 The name of the matchmaking hopper
sdatemplate 字符串 The name of the secure device association template
matchattributes 真实的 OPTIONAL 匹配属性

返回。

Async Request ID

例子。

var userID = xboxone_user_for_pad(global.PadIndex);
result = xboxone_matchmaking_find(userID, "MatchTicketSession", "MatchTicketHopper", "PeerServerTraffic");

以上将检索给定游戏板上用户的用户ID，然后尝试为他们找到一个匹配会话。
