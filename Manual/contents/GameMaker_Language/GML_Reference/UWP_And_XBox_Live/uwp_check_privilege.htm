uwp_check_privilege

通过这个函数，你可以检查一个给定的用户是否有一个选定的权限。该函数将在系统异步事件中返回true或false，这取决于该权限是否被启用，如果你将attempt_resolution参数设置为true，而该权限没有被启用，它还将打开一个系统对话框（暂停游戏），提示用户升级账户或任何获得该权限所需的东西。如果用户随后启用了所需的选项，该函数将返回true。

该函数要求你使用以下其中一个 UWP特权常数的特权请求。

UWP特权常数
恒定 描述
uwp_privilege_internet_browsing 检查是否允许浏览互联网
uwp_privilege_social_network_sharing 检查是否允许分享到社交网络
uwp_privilege_share_kinect_content 检查是否允许共享Kinect控制器
uwp_privilege_video_communications 检查是否允许视频通信
uwp_privilege_communications 检查是否允许互联网通信
uwp_privilege_user_created_content 检查是否允许访问用户创建的内容
uwp_privilege_multiplayer_sessions 检查是否允许在线多人游戏会话
uwp_privilege_sessions 检查是否允许在线会议
uwp_privilege_fitness_upload 检查是否允许上传健身数据

一旦函数被调用，在XBox One ，当收到回调结果时，它将触发一个异步系统事件，它将具有特殊的DS映射 async_load.然后，这个地图应该被解析为以下关键。

  • "event_type" - 如果事件是由这个函数触发的，应该保持值"check_privilege_result"。

如果事件类型与该函数有关，那么在XBox上调用该函数时还会有以下附加键。

  • "结果" - 将是以下的一个或多个结果 UWP 特权结果常数s（组合值）。
    UWP特权结果常数
    常数 描述 价值
    uwp_privilege_result_no_issue 用户没有特权问题 0
    uwp_privilege_result_purchase_required 用户必须购买额外的东西，通常是订阅，才能访问。 1
    uwp_privilege_result_aborted 检查被中止了 2
    uwp_privilege_result_banned 该用户被禁止使用 4
    uwp_privilege_result_restricted 用户被限制访问，通常是通过家长控制。 8

  • "特权" - 将是上面列出的常数之一，在函数中使用，所以你可以在多重检查的情况下检查你所请求的特权。

如果项目运行在不是XBox One 的UWP平台上，那么Async事件将不会被触发，相反，该函数将立即返回-1、0或1，其中。

  • -1表示-1，用户没有这个权限。
  • 0表示该函数尚未能查询到用户
  • 1表示该用户拥有该权限。

语法。

uwp_check_privilege(user_id, privilege, attempt_resolution);

争论 类型 描述
user_id Xbox用户ID The User ID pointer to check.
privilege UWP特权常数 The privilege constant to check (see description, above).
attempt_resolution 布尔型 Enable attempting a resolution if set to true，如果设置为 false ，则忽略不计。

返回。

Real

例子。

var user_one = xboxlive_get_user(0);
if (xboxlive_user_is_signed_in(user_one))
{
    uwp_check_privilege(user_one, xboxlive_privilege_multiplayer_sessions, true);
}

上面检查用户是否登录，如果他们登录了，则检查多人游戏会话的权限。
