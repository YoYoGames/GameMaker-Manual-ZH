xboxlive_get_stats_for_user

这个函数可以用来从Xbox Liveservers 检索有关特定统计的数据。你提供函数所返回的用户ID xboxlive_get_user()然后是你的游戏服务配置ID（如XDP控制台上显示的），最后是所需的统计信息。你最多可以请求14个统计信息，但这并不保证你会得到14个统计信息的结果，因为请求的总长度是有限制的，因此最大的统计信息数量取决于统计信息本身的名称长度。

如果统计请求被发送，该函数将返回0，如果有错误则返回-1，回调将触发一个系统异步事件。这个事件将有一个特殊的DS地图 async_load然后应该对其进行解析，以获得以下关键。

  • "event_type" - 如果事件是由这个函数触发的，应该持有string "stat_result" 。

如果事件类型与此功能有关，那么也会有以下的附加键。

  • "user" - 持有所请求的统计资料的用户ID。

也可能有一组额外的键值对，其中键是请求的状态名称，而值是该状态在server 。注意，如果一个状态从未为这个用户创建（因为他们没有触发设置它的事件），可能不会返回该状态的值（ds_map_exists()可以用来检查 async_load 地图中的统计信息）。

如果由于请求长度过大而导致请求失败，在控制台输出中应该有一条信息，说明错误代码。

xboxlive_get_stats_for_user - exception occurred getting results - 0x80190190

当这种情况发生时，异步事件DS地图应该有一个值为 "0 "的 "成功 "键，在这种情况下，你应该尝试在一次调用中检索更少的统计信息。

语法。

xboxlive_get_stats_for_user(user_id, serviceconfig_id, statname1, …);

争论 类型 描述
user_id Xbox用户ID The user ID pointer.
serviceconfig_id 字符串 The service config file ID
statname1 (2, 3, etc...) 弦乐 The stat names to retrieve the information for.

返回。

Real

例子。

var _uid = xboxlive_get_user(0);
var _configid = "00000000-0000-0000-0000-000000000000";
xboxlive_get_stats_for_user(xbuser, _configid, "GameProgress", "CurrentMode");

上面的代码得到了用户的ID，然后用它来请求该用户的特定统计信息。

返回。统计资料和排行榜
