xboxlive_read_player_leaderboard

该函数将返回给定用户名的排行榜信息。排行榜必须是先前在你的游戏的XDP仪表板上创建的，调用该函数将触发一个回调的Social Asynchronous Event，其中包括 async_load地图上的相关键/值对。这个DS地图的 "id "键用于识别正确的回调（对于任何给定的异步事件可以有多个触发函数），IT将与常量 achievement_leaderboard_info ，以及其他一些键/值对配对。该地图的具体内容如下所示。

  • "id" - 对于这个函数，它应该是 achievement_leaderboard_info
  • "领先优势" - 在提供者仪表板上定义的排行榜的唯一ID。
  • "纪念品" - 你在排行榜上获得的条目数。
  • "闲人免进" - 玩家的名字，其中 "N "是一个整数值，对应于他们在排行榜条目列表中的位置。
  • "玩家IDN" - 球员的唯一用户ID，"N"。
  • "等级N" - 玩家 "N "在排行榜上的排名。
  • "分数N" - 球员 "N "的得分。


该函数要求你给出以下常数之一来设置过滤器的属性。

恒定 描述
xboxlive_achievement_filter_all_players 获得所有的分数。
xboxlive_achievement_filter_friends_only 只获得朋友的分数，按升序排列。
xboxlive_achievement_filter_favorites_only 只获取最喜欢的分数，按升序排列
xboxlive_achievement_filter_friends_alt 只获取朋友的分数，按降序排列。
xboxlive_achievement_filter_favorites_alt 只获取按降序排列的最喜欢的分数。

语法。

xboxlive_read_player_leaderboard(leaderboard_name, user_name, num_items, friend_filter);

争论 类型 描述
leaderboard_name 字符串 The name of the leaderboard to read, as set up on XDP.
user_name Xbox用户ID The ID of the user to read from.
num_items 真实的 The number of items to retrieve.
friend_filter 习近平总书记指出："习近平总书记的重要讲话，为我们指明了前进的方向。 One of the filter constants (see the description, above).

返回。

Real

例子。

var _user = xboxlive_user_for_pad(0);
var _uid = xboxlive_user_id_for_user(_user);
xboxlive_read_player_leaderboard("MyLeaderboard", _uid, 10, achievement_filter_all_players);

上面的代码得到用户ID，然后用它来请求给定排行榜前10名的所有玩家数据。
