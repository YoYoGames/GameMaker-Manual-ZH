xboxlive_get_file_error

这个函数将返回一个常数（如下图所示）以帮助调试文件函数失败的原因，这样你就可以在游戏代码中采取适当的行动。当某个文件函数失败时，你可以调用这个函数来找出错误的确切原因。

注意：当使用异步保存/加载函数时， async_load地图将有一个额外的键，"error"，它也将包含一个常数（如下所示）。

语法。

xboxlive_get_file_error();

返回。

Xbox File Error Constant

Xbox文件错误常数
恒定 描述
xboxlive_fileerror_outoflocalstorage 如果在保存时发生这种情况，这意味着所有可用的游戏保存空间已被用完，而用户拒绝了（操作系统提出的）释放空间的提议。游戏应该向用户确认他们是否打算不保存而继续游戏。如果他们仍然希望能够保存，游戏应该重新进行保存操作。
xboxlive_fileerror_quotaexceeded 如果发生这种情况，游戏应该通知用户，他们不能再保存任何数据，直到他们释放出一些空间，然后向他们提供一种方法。
xboxlive_fileerror_noerror 文件保存或加载功能的进行没有问题。

例子。

buffer_save(buff, "SaveGame.bin");
if (xboxlive_get_file_error() != xboxlive_fileerror_noerror)
{
    show_debug_message("Save failed. Error code = " + string(xboxlive_get_file_error()));
}

上面的代码检查是否有保存错误，如果有，错误代码将被打印到控制台。
