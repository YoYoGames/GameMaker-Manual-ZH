摄像机_创建_视图

此函数将创建一个新的摄像机，然后将其视图设置到room ，然后返回唯一的ID值，以便在以后的函数调用中识别该摄像机。摄像机的创建将使用你在此函数中为位置、大小和其他细节设置的值，这些值与你在Room 编辑器中设置的值相关（见这里）。请注意，这个函数可以接受不同数量的参数，只有X/Y位置和宽度/高度参数是必须的。任何没有提供的参数将使用其默认参数。

请注意，摄像机和所有的动态资源一样，在创建时将占用内存。这意味着你需要注意不要不必要地创建相机，并通过调用函数销毁那些你不需要的相机 camera_destroy().这也意味着，如果分配给 global 变量，摄像机将从room 持续到room （分配给实例或局部变量时也会持续存在，但在改变room 后就不能访问它们了，因此会有内存泄漏），因此你可以--例如--创建一个新的摄像机，将其分配给一个全局变量，然后在每个room 的开始，在将其分配给一个视图端口前使用 camera_set_* 函数来设置属性。

语法。

camera_create_view(room_x, room_y, width, height, [angle, object, x_speed, y_speed, x_border, y_border])

争论 类型 描述
room_x 真实的 The x position (within the room) of the left side of the area the camera view captures.
room_y 真实的 The y position (within the room) of the top of the area the camera view captures.
width 真实的 The width of the area the view captures.
height 真实的 The height of the area the view captures.
angle 真实的 OPTIONAL 设置摄像机视角的角度（从0°-360°，默认为0）。
object 对象资产或实例ID OPTIONAL 要跟踪的摄像机视图的对象ID（或实例ID）（默认为-1，没有 ）。object
x_speed 真实的 OPTIONAL 视图可以移动的最大水平速度（设置为-1表示 "即时 "跟随，默认为-1）。
y_speed 真实的 OPTIONAL 视图可以移动的最大垂直速度（设置为-1表示 "即时 "跟随，默认为-1）。
x_border 真实的 OPTIONAL 视图区域的边缘和它被设置为跟随的实例之间的最小水平空间（在视图开始移动之前，如果它被设置为跟随一个实例，默认为0）。
y_border 真实的 OPTIONAL 视图区域的边缘和它被设置为跟随的实例之间的最小垂直空间（在视图开始移动之前，如果它被设置为跟随一个实例，默认为0）。

返回。

Camera ID

例子。

camera_destroy(view_camera[0]);
view_camera[0] = camera_create_view(0, 0, 640, 480, 0, obj_Player, 5, 5, -1, -1);

上面的代码删除了当前分配给视口[0]的摄像机，然后创建一个新的摄像机，定义其视图属性，然后将其分配给视口[0]。
