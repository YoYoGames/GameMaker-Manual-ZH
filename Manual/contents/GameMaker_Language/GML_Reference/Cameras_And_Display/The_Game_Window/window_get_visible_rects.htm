window_get_visible_rects

通过这个函数，你可以在每个附加的显示器上找到由(x1,y1)到(x2,y2)定义的矩形的重叠区域。该函数将返回一个数组，每个显示器有8个值（即：如果你有两个显示器，该数组将有一个 长度为16个索引），其中值[0 ... 3]对应于overlapx1, overlapy1, overlapx2, overlapy2 - 定义这个显示器上的重叠区域，如果没有重叠，将被设置为0,0,0,0 - 而值[4 ... 7] 对应于 monitorx1, monitory1, monitorx2, monitory2 - 显示屏在虚拟显示空间中的坐标。这可以用来测试保存的窗口位置是否可见（用户可能已经断开了外部显示器的连接或 例如，用户可能断开了外部显示器的连接，或者将窗口移出了屏幕，导致保存的窗口位置无效）。

语法。

window_get_visible_rects(x1, y1, x2, y2);

争论类型 描述
x1 The left edge of the rectangle to check
y1 The top edge of the rectangle to check.
x2 The right edge of the rectangle to check
y2 The bottom edge of the rectangle to check.

返回。

例子。

var wx = window_get_x();
var wy = window_get_y();
var ww = window_get_width();
var wh = window_get_height();
display_data = window_get_visible_rects(wx, wy, wx + ww, wy + wh);
display_num = array_length(display_data) / 8;

上述代码将生成一个1D数组，保存在变量 display_data ，其中包含关于显示器的信息，同时创建变量 display_num ，以保存发现的活动显示器的数量。

返回。游戏窗口
