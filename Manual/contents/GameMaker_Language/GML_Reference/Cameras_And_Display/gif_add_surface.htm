gif_add_surface

用这个函数，你可以从一个表面保存一个帧到GIF文件。你提供GIF文件的ID（由函数返回 gif_open())和表面ID(由函数返回) surface_create())，以及帧之间的延迟时间。延迟时间是以百分之一秒计算的，但是请注意，一旦GIF文件被创建，如果你把它用于宣传目的，不同的浏览器对帧延迟的解释可能略有不同--更多细节请看这里。您可以（选择性地）提供用于添加到GIF的表面的X和Y偏移值，在这种情况下，绘制到GIF的部分将从偏移值开始，而不是默认的（0,0）位置，允许您只选择表面的一部分来添加。你也可以提供一个量化值，这是一个介于0和3之间的整数，其中0是完全量化，3是零量化（较低的值会有一个后续的质量下降，但会创建一个较小的GIF）。请注意，默认的质量为2，与最高的3之间没有太大区别，使用3会大大降低GIF的创建速度，所以在使用最大值时要注意。

注意，在你的GIF中，对帧的数量或这些帧的大小没有内置的限制，但如果你使用了太多的内存，那么该函数将失败并返回-1，否则它将返回0。

语法。

gif_add_surface(gif_index, surface, delay_time, [xoffset], [yoffset]);

争论 类型 描述
gif_index GIF ID The ID of gif to add the surface data to
surface 表面标识 The ID of the surface to use as the added frame
delay_time 真实的 The delay time for the frame (in 100ths of a second)
[xoffset] 真实的 OPTIONAL 沿着曲面的X轴添加的偏移量，从
[yoffset] 真实的 OPTIONAL 沿着曲面的Y轴添加的偏移量，从
[quantization] 真实的 OPTIONAL 从0到3的量化数量（0的质量最低，3的质量最高，默认为2）。

返回。

Real

例子。

if save_gif == true
{
    if (count == 0)
    {
        gif_image = gif_open(room_width, room_height);
    }
    else if (count < 30)
    {
        gif_add_surface(gif_image, application_surface, 6/100);
    }
    else
    {
        gif_save(gif_image, "GameCapture.gif");
        count = 0;
        save_gif = false;
    }
    count++;
}

上面的代码将创建一个GIF图像文件，其中有30个从应用程序表面拍摄的帧，然后将其保存。
