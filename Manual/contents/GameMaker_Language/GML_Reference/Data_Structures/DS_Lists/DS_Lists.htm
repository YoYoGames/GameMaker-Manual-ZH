DS列表

DS列表是一种数据结构，它在添加信息时按顺序存储（很像一个数组）。DS列表是非常灵活的数据结构，它允许你在列表的末尾添加数值或在中间某处插入数值，同时还允许你洗牌（使数值随机化）或按升序或降序排序的能力。

注意：建议使用数组而不是DS列表，因为它们有类似的功能，更容易使用，而且可以自动收集垃圾。

当访问DS列表数据结构时，你应该总是尽量使用整数值来表示列表位置，如果你不这样做，所有非整数指数都会被GameMaker浮动。如果这不是你所要求的，那么你就需要在传递你希望检查的索引之前，自己事先进行舍入。

注意：与所有的动态资源一样，数据结构会占用内存，所以当不再需要时，应该总是销毁，以防止内存泄漏，这将减缓并最终使你的游戏崩溃。

除了下面列出的特定函数外，你都可以使用一个表达式（称为访问器）来添加或修改你的DS列表的内容。这个访问器看起来类似于一个1D数组，语法如下。

list_index[| index]

你可以从Accessors的GML Overview页面中找到更多的信息和例子。

存在以下处理DS列表的函数。

  • ds_list_create
  • ds_list_destroy
  • ds_list_clear
  • ds_list_empty
  • ds_list_size
  • ds_list_add
  • ds_list_set
  • ds_list_delete
  • ds_list_find_index
  • ds_list_find_value
  • ds_list_insert
  • ds_list_replace
  • ds_list_shuffle
  • ds_list_sort
  • ds_list_copy
  • ds_list_read
  • ds_list_write

Ds 列表也可以用来保存DS 地图和其他 DS 列表，因此我们有一些特殊的函数用来 "标记 "或 "标注 "列表中保存这些数据结构的位置。一般来说，这不是必须的，但它的好处是当父列表被销毁时，可以自动收集（销毁）被标记的数据结构，而且在与 json_encode()函数一起使用时，它也是必需的，以便对列表和它的数据进行正确的编码。这方面的函数有：。

  • ds_list_mark_as_list
  • ds_list_mark_as_map
  • ds_list_is_list
  • ds_list_is_map
