DS地图

DS地图数据结构是一个非常有用的结构，因为它允许你存储键和值对。例如，在你的游戏中，一个角色可以拥有一定数量的不同物品（键），对于每个单独的物品，他可以拥有一定数量的物品（值），比如在RPG游戏中，你可以拥有10个健康药水、5个法力药水和100个金币。地图将这些配对放在一起，你可以在地图上添加配对，也可以使用一些简单的功能搜索某些键所对应的值。然而，在你使用地图之前，有几件事情你应该了解一下!地图没有以任何（可识别的）方式进行排序，这意味着为了找到某个键，你可能不得不遍历整个地图（这非常慢）。也没有办法保持两个相同的键，你也不能给一个键分配两个值。

注意：建议使用结构体而不是DS图，因为它们有类似的功能，更容易使用，并且可以自动收集垃圾。

注意与所有的动态资源一样，数据结构会占用内存，所以当不再需要时，应该总是销毁，以防止内存泄漏，这将减缓并最终使你的游戏崩溃。

除了下面列出的特定函数外，你都可以使用一个表达式（称为访问器）来添加或修改你的DS地图的内容。这个访问器看起来类似于一个1D数组，语法如下。

map_index[? key]

你可以从Accessors的GML Overview页面中找到更多的信息和例子。

存在以下处理DS地图的函数。

  • ds_map_exists
  • ds_map_create
  • ds_map_add
  • ds_map_clear
  • ds_map_copy
  • ds_map_replace
  • ds_map_delete
  • ds_map_empty
  • ds_map_size
  • ds_map_find_first
  • ds_map_find_last
  • ds_map_find_next
  • ds_map_find_previous
  • ds_map_find_value
  • ds_map_keys_to_array
  • ds_map_values_to_array
  • ds_map_set
  • ds_map_read
  • ds_map_write
  • ds_map_destroy

还有四个功能可用于保存和加载DS地图。这些功能将混淆地图，并将其存储在目标平台上的一个安全位置，使用的文件格式意味着最终文件无法在设备之间传输。

  • ds_map_secure_save
  • ds_map_secure_save_buffer
  • ds_map_secure_load
  • ds_map_secure_load_buffer

DS地图的用途之一是在处理JSON时，因此有一些专门的功能。

  • ds_map_add_list
  • ds_map_add_map
  • ds_map_replace_list
  • ds_map_replace_map
  • ds_map_is_list
  • ds_map_is_map

注意虽然这些函数允许你在地图中添加列表和地图，但它们对JSON以外的任何东西都是无用的，如果写入磁盘或以任何其他方式访问，嵌套的地图和列表将不会被正确读取。
