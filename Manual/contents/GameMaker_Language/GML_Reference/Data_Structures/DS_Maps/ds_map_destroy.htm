ds_map_destroy

DS地图会占用内存空间，这是在它们被创建时分配给它们的。这意味着，当不需要DS地图时，你也需要释放这部分内存，以防止运行游戏时出现错误、内存泄漏和性能损失。这个函数 就是这样做的。请注意，销毁地图将取消对地图中存储的任何数据结构的引用，从而导致内存泄漏，所以你需要在销毁地图之前手动销毁所有数据结构项，以防止这种情况。唯一不需要这样做的时候是 唯一不需要这样做的时候是当你把地图中的任何项目标记为DS列表或另一个DS地图，在这种情况下，这些项目将被销毁，它们的内存也将被自动清理掉 也会自动清理。

重要的是!当你创建一个数据结构时，识别它的索引值是一个从0开始的整数值。 这意味着不同类型的数据结构可以有相同的索引值，所以如果有疑问，你应该使用 ds_exists()函数来访问它们。还要注意的是，索引是可以重复使用的，所以一个被破坏的数据结构索引值可能会被之后新创建的数据结构使用，所以我们建议 所以我们建议在销毁数据结构后，总是将持有DS索引的变量设置为-1。

语法。

ds_map_destroy(id);

争论类型 描述
id The id of the map to destroy.

返回。

例子。

ds_map_destroy(inventory);
inventory = -1;

上面的代码将破坏DS地图，其id被索引在变量 "库存 "中。

返回。DS地图
