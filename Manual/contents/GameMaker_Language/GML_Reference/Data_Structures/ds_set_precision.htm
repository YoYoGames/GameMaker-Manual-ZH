ds_set_precision

当比较数值时，例如在地图中搜索或对列表进行排序时，GameMaker必须决定两个数值是否相等。对于strings 和整数值来说，这很清楚，但是对于实数来说，由于浮点舍入误差，看似相等的数字很容易变成不相等。例如，(5/3)*3有可能不等于5!为了帮助避免这种情况，所有的实数函数都使用一个精度值，当两个数字之间的差值小于这个精度时，它们就被认为是相等的。所有的数据结构函数默认使用0.0000001的精度，除非被这个函数改变。

注意：这个精度在所有的数据结构中使用，但在GML 的其他比较中不使用 !

语法。

ds_set_precision(prec);

争论 类型 描述
prec 真实的 The precision value (default 0.0000001)

返回。

N/A

例子。

ds_set_precision(0.0001);

上述代码将改变所有数据结构函数的默认精度设置。
