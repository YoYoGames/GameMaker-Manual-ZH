数据结构

在游戏中，你经常需要以精确和有序的方式存储信息。例如，你可能需要存储一个人携带的物品清单，或者你可能想存储一个还需要访问的地方的网格。现在，你可以使用数组来做这些事情，但如果你想做更复杂的操作，比如对数据进行排序或搜索某个特定的项目，你需要编写大段的GML ，这些代码执行起来很慢，难以调试，写起来也很麻烦

为了弥补这一点，GameMaker有一些内置的数据结构，可以通过专业函数来访问。有六种不同类型的数据结构可用，每一种都有自己的好处，这取决于你要存储的信息类型和你希望以后如何操作它：堆栈、queues 、列表、地图、优先级queues ，以及网格。

注意：建议使用数组和结构而不是DS列表和地图，因为它们有类似的功能，更容易使用，而且可以自动收集垃圾。

从本质上讲，所有数据结构的工作方式都是一样的--你创建一个数据结构并将其索引值存储在一个变量中。然后，你使用这个索引在所有进一步的函数调用中引用该数据结构，这些函数可以对其进行操作。最后，一旦你完成了，你就再次销毁该数据结构，将其从内存中删除。你可以根据需要同时使用多个结构，而且所有结构都可以存储任何可用的数据类型。

注意 与所有的动态资源一样，数据结构会占用内存，所以当不再需要时，应该总是销毁，以防止内存泄漏，这将减缓并最终使你的游戏崩溃。

当你创建一个数据结构时，用来识别它的索引值是一个从0开始的整数。 这意味着不同的数据结构可以有相同的索引值，所以如果有疑问，你应该在访问它们之前使用 ds_exists函数来访问它们。还要注意的是，索引是可以重复使用的，所以一个被破坏的数据结构索引值可能会被之后新创建的数据结构使用，我们建议你在破坏了数据结构之后，将任何持有DS索引的变量设置为-1。

有关不同数据结构的信息可以在以下章节中找到。

  • 网格
  • 列表
  • 地图
  • 优先队列
  • 队列
  • 堆栈

在你的游戏中使用数据结构之前，你还应该注意，在某些时候，由于四舍五入的错误，你可能会得到一个与你预期不同的结果。这可以通过使用以下函数改变DS的精度来解决。

  • ds_set_precision

还有一个特殊的函数用于检查任何给定类型的数据结构是否存在。

  • ds_exists
