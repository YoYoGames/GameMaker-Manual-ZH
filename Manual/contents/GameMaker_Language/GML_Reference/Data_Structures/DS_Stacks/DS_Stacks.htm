DS栈是一个所谓的后进先出（LIFO）数据结构，类似于DS队列。你可以把值推到堆栈上，然后通过从堆栈中弹出的方式再次删除它们，其中最近推到堆栈上的值是第一个被再次弹出的（想想一摞硬币，在你能得到下面的其他硬币之前，你增加的每个硬币都必须先被删除）。当有中断需要处理时，或有递归函数时，甚至在为你的游戏构建一个基本的人工智能时，经常使用堆栈。

注意：与所有的动态资源一样，数据结构会占用内存，所以当不再需要时，应该总是销毁，以防止内存泄漏，这将减缓并最终使你的游戏崩溃。

注意：如果你需要检查一个数据结构是否存在，你可以使用 ds_exists()函数。

存在以下处理DS堆栈的函数。

  • ds_stack_create
  • ds_stack_destroy
  • ds_stack_clear
  • ds_stack_empty
  • ds_stack_size
  • ds_stack_copy
  • ds_stack_top
  • ds_stack_pop
  • ds_stack_push
  • ds_stack_read
  • ds_stack_write
