游戏保存

这是一个传统的函数，可以用来保存游戏的当前状态，不建议再使用。使用 "文件 "功能来创建一个自定义的保存系统，只保存和加载特定的游戏数据。

assets 这个函数将保存游戏的状态，但是它不会保存当时正在使用的任何动态资源，比如数据结构、表面、runtime ，等等。如果加载这样的保存文件，任何在保存后进行的游戏更新都将不可见，因为它将恢复创建保存时使用的游戏版本。

请注意，使用该功能创建的保存文件可能不被更新的Runtime版本所支持，因此用新的保存系统取代该功能是必不可少的，以确保与未来的GameMaker ，以及由于上述原因，对你自己的游戏所做的更新的兼容性。

语法。

game_save(filename);

争论类型 描述
filename The name of the file to save the game to.

返回。

例子。

if keyboard_check_pressed(ord("S"))
{
    global.Saved = true;
    game_save("Save.dat");
}

上面的代码将设置一个全局变量为 true ，然后在按下 "S "键时将游戏保存到 "Save.dat "文件。

返回。一般游戏控制
