一般游戏控制

这些函数和变量都与游戏的具体功能有关，如加载/保存、重新启动等。

以下全局范围的内置变量对于debugging 你的游戏是非常有用的--例如，在出现问题的情况下，它们可以被用作输出到日志文件中--以及向用户显示当前的构建ID等。

  • game_id
  • game_save_id
  • game_display_name
  • game_project_name

以下功能用于一般游戏控制。

  • game_end
  • game_restart
  • game_load
  • game_load_buffer
  • game_save
  • game_save_buffer
  • game_get_speed
  • game_set_speed

为了帮助你为你的游戏创建和维护高分，GameMaker创建了一个由10个地方组成的全局高分数组，你可以访问、添加和更改，以创建你自己的自定义高分列表，而不需要太多麻烦。这个功能是游戏的本地功能，在所有平台上都可以使用，所以你可以使用这些函数和文件处理函数轻松地存储和维护一个基本的高分表。

存在以下处理内部高分榜的功能。

  • highscore_add
  • highscore_name
  • highscore_value
  • highscore_clear

GameMaker还包括一个在屏幕上以列表形式画出高分的功能，但这是非常基本的，建议你更多地将其用于debugging ，因为制作你自己的高分表很容易，而且会让你对它的显示方式有更大的控制。

  • draw_highscore

最后，我们有一个函数，允许你为鼠标光标分配一个sprite 。

  • cursor_sprite
