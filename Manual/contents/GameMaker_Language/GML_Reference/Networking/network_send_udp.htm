network_send_udp

通过这个函数，你可以使用UDP在网络上发送数据到一个server 。这个函数接收要连接的套接字ID，要连接的URL和要使用的端口。然后，你必须提供包含要发送的数据的缓冲区ID（关于buffers 的更多信息，见参考-缓冲区），最后是数据的大小（字节）。UDP是 "无连接 "的，因为你实际上没有进行连接，你只是直接向一个IP发送一个数据包，而server ，从一个IP地址获得传入的数据，并必须 "按原样 "处理。该函数将返回所发送数据的字节数，如果发送失败，则返回一个小于0的数字。值得注意的是，该函数返回的正在发送的数据的最终大小也将包括GameMaker 头信息，这是一个额外的12字节。

这个功能在运行在HTML5目标上的项目中使用时将不会起作用，而且HTML5项目也不能接收UDP。

语法。

network_send_udp(socket, url, port, bufferid, size);

争论 类型 描述
socket 网络插座ID The id of the socket to use.
url 字符串 The url or IP to connect to (a string).
port 真实的 The port to connect to.
bufferid 缓冲区ID The id of the buffer to get the data from.
size 真实的 The size (in bytes) of the data.

返回。

Real

例子。

network_send_udp(sock, "www.macsweeneygames.com", 6510, buff, buffer_tell(buff));

上面的代码将发送一个UDP数据包到6510端口上由URL定义的server 。数据取自变量 "buff "中索引的buffer 。
