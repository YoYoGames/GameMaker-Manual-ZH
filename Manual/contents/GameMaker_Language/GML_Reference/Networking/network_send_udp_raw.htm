network_send_udp_raw

通过这个函数，你可以使用UDP在网络上发送数据到一个server 。这个函数接收要连接的套接字ID，要连接的URL和要使用的端口。然后，你必须提供包含要发送的数据的缓冲区ID（关于buffers 的更多信息，见参考-缓冲区），最后是数据的大小（字节）。UDP是 "无连接 "的，因为你实际上没有进行连接，你只是直接向一个IP发送一个数据包，而server ，从一个IP地址获得传入的数据，并必须 "按原样 "处理它。发送的数据不会被GameMaker以任何方式格式化，接收设备将以流的形式获得数据，这意味着你必须自己处理它。该函数将返回发送的数据字节数，如果发送失败，则返回一个小于0的数字。

注意这个功能在运行在HTML5目标上的项目中使用时将不起作用，而且HTML5项目也不能接收UDP。

语法。

network_send_udp_raw(socket, url, port, bufferid, size);

争论类型 描述
socket The id of the socket to use.
url The url or IP to connect to (a string).
port The port to connect to.
bufferid The id of the buffer to get the data from.
size The size (in bytes) of the data.

返回。

Real

例子。

network_send_udp_raw(sock, "www.macsweeneygames.com", 6510, buff, buffer_tell(buff));

上面的代码将发送一个原始的UDP数据包到6510端口上由URL定义的server 。数据取自变量 "buff "中索引的buffer 。
