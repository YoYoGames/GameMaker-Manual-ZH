网络_send_broadcast

有了这个功能，你可以将数据从buffer 本地广播到一定范围的IP地址（关于buffers 的更多信息，见参考-缓冲区）。该范围仅限于运行server 的设备，例如，如果该设备的IP是92.168.11.130，那么数据将在92.168.11.*范围内广播。该函数将返回发送的数据字节数，如果发送失败，则返回一个小于0的数字。

这个功能只有在与UDP一起使用时才能工作--你的 需要是TCP的，你的客户需要有一个UDP客户套接字，用server network_create_socket_ext()来接收从server 发送的任何广播。

在运行于HTML5目标的项目中使用该功能时，将无法工作。

语法。

network_send_broadcast(socket, port, bufferid, size);

争论 类型 描述
socket 网络插座ID The id of the socket to use.
port 真实的 The port that the server will use.
bufferid 缓冲区ID The id of the buffer to get the data from.
size 真实的 The size (in bytes) of the data.

返回。

Real

例子。

buffer_seek(broadcast_buffer, buffer_seek_start, 0);
buffer_write(broadcast_buffer, buffer_string, global.ServerName);
network_send_broadcast(server, 6511, broadcast_buffer, buffer_tell(broadcast_buffer));

上面的代码写下了当前string （存储在 "global.ServerName "中）的名称server ，然后将其写入一个ID为 "broadcast_buffer "的二进制buffer 。然后这个数据被本地广播到一个IP范围（设备IP目前被暗示为广播基础范围）到6511端口。

返回。联网
