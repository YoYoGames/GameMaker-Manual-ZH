成绩_帖子_分数

你可以使用这个函数将你的分数发送到所选择的排行榜和成就服务。在Android ，你将分数表的ID作为string （这是你在设置排行榜时分配的唯一的排行榜ID），而在所有其他平台上，你将提供定义好的排行榜名称作为 ，然后给出实际的分数值。 而在所有其他平台上，你提供定义的排行榜名称作为string ，然后你给出实际的分数值。该函数将触发一个社交异步事件，返回的 异步加载DS地图将有以下键/值对。

  • "类型" - 这是已经触发的事件的类型，这将是这个函数的string "achievement_post_score_result"。
  • "id" - 这是事件的ID，对于这个函数，它将返回一个GML 常数GooglePlayServices_EVENT_ID_POST_SCORE。
  • "leaderboard_id" - 这是被更新的排行榜的唯一ID。
  • "status" - 如果分数被成功更新，这将是1，如果请求失败，这将是0。
  • "daily_new_best" - 如果提交的分数是用户最近一天的最好成绩，这将是1，否则就是0。
  • "daily_best" - 这将保存用户在最后一天游戏中的最高得分。
  • "weekly_new_best" - 如果提交的分数是用户在上周的最好成绩，这将是1，否则就是0。
  • "weekly_best" - 这保存了用户上周的最高得分。
  • "all_time_new_best" - 如果提交的分数是用户有史以来最好的分数，这将是1，否则就是0。
  • "all_time_best" - 这是用户在所有时间内的最高得分。

语法。

achievement_post_score(leaderboard, score)

扩展实例。

if achievement_available()
    {
    achievement_post_score(global.LeaderboardID, 1000);
    }

上面的代码检查所选择的排行榜和成就服务是否可用，如果可用，它就会向名为 "排行榜 "的分数表发布一个分数。这将触发一个Social Async事件，可以这样解析。

if async_load[? "id"] == GooglePlayServices_EVENT_ID_POST_SCORE
    {
    if async_load[? "type"] == "achievement_post_score_result"
        {
        if async_load[? "status"] == 1
            {
            global.Best_Day = async_load[? "daily_best"];
            global.Best_Week = async_load[? "weekly_best"];
            global.Best = async_load[? "all_time_best"];
            }
        }
    }

上面的代码检查了async_loadDS map中与这个函数相对应的 "id "和 "type "键，然后它检查了事件的状态。如果状态为1，那么DS地图中的一些值将被存储在全局变量中。 变量中。
