绩效_获取图片

这个函数将向server ，以获取玩家或朋友的图像，并将触发一个回调社会异步事件，其中包含了 async_load 地图填充了相关的键/值对。这个DS地图的id键用于识别正确的回调（对于任何给定的异步事件，可以有一个以上的触发函数），并将与常数 achievement_pic_loaded以及其他一些键/值对搭配。该地图的具体内容如下所示。

注意：该功能仅适用于iOS。

  • "id" - 这个函数应该是achievement_pic_loaded。
  • "playerid" - 我们要接收图像的玩家或朋友的ID。
  • "spriteid" - 图像的sprite id，然后可以用任何drawsprite 函数在屏幕上绘制它。

警告!这个函数创建了sprites ，因此，它们的id应该被存储在变量中，然后使用 sprite_delete() 函数从内存中删除，否则重复使用会造成内存泄漏，最终会使游戏变慢或崩溃。

语法。

achievement_get_pic(char)

争论 描述
char 要获取图片的人的ID（可以从好友列表或排行榜上获取）。

返回。

N/A

扩展实例。

在你得到一个朋友列表或排行榜列表后，将调用以下代码，使用 achievement_load_friends()或 achievement_load_leaderboard(). 这些函数将为每个人返回唯一的 "id "值，然后在函数中使用这些值来检索他们的个人资料图片。

achievement_get_pic(global.player_id[0]);

这将发送一个与存储在全局数组中的玩家ID相关的图像请求，并生成一个异步回调，其中包含以下数据的特殊async_loadDS map。

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_pic_loaded
    {
    var picid = ds_map_find_value(async_load, "playerid");
    var spriteid = ds_map_find_value(async_load, "spriteid");
    if global.playerid[0] == picid)
        {
        global.playerpic[0] = spriteid;
        }
    }

上面的代码在社交异步事件中检查返回的DS Map，如果它的 "id "与被检查的常量相匹配，它就会检查与图像相关的id是否与被请求的id相同，如果是，它就会设置另一个全局数组来保存图像的唯一id（这可以作为）。 它就会设置另一个全局数组来保存图像的唯一ID（可用于将图像绘制为sprite ）。
