push_get_first_local_notification

这个函数将用一系列的键/值对来填充一个预制的DS地图，用于显示queue 中的第一个本地推送通知。如果没有排队的通知，它将返回-1，否则将返回一个代表通知的ID的实际值。然后这个ID可以用来取消通知，使用函数 push_cancel_local_notification().

注意：该功能仅限于iOS和Android目标模块。

DS地图将持有以下键/值对。

  • "title" - 通知的标题（这只在安卓平台上可用，因为iOS ，不存储标题信息）。
  • "信息" - 信息主体文本
  • "data" - 数据包字符串

语法。

push_get_first_local_notification(map);

争论类型 描述
map The DS Map 要使用的ID。

返回。

Push Notification ID or -1

例子 :

var map = ds_map_create();
var ntf = push_get_first_local_notification(map);
while(ntf >= 0)
{
    var data = ds_map_find_value(map, "data");
    if data == "Daily_Reward"
    {
        push_cancel_local_notification(ntf);
    }
    ntf = push_get_next_local_notification(map);
}
ds_map_destroy(map);

上述代码创建了一个 DS Map ，然后请求queue 中第一个可用的通知的数据。如果存在一个通知，地图就会被填充，然后代码会检查 "数据 "键，寻找一个特定的string 。如果找到了string ，则取消该通知，并继续检查下一个通知，直到queue 中的所有通知都被检查过。
