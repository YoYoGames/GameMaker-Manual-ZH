云端字符串保存

这个函数将提交一个string 到所选择的云服务进行存储。该函数将返回一个唯一的ID值，然后应该在适当的异步事件中使用，以识别作为云服务 "回调 "返回的DS地图。string 应该包含你需要为你的游戏保存的所有信息，因为你只能将一个单一的 "数据块 "存储到云端，再次运行这个函数将覆盖任何先前存储的值（正如使用 cloud_file_save()函数）。描述应该是一个简短的string ，描述保存的信息，例如："Level2, Stage2"。

关于返回的异步数据的进一步信息，请参见函数 cloud_synchronise().

语法。

cloud_string_save(string, description);

争论类型 描述
string The data string to be uploaded.
description A brief description of the data being stored.

返回。

Async Request ID

例子。

var t_str = "";
for (var i = 0; i < 10; i++;)
{
    t_str += string(global.Highscore[i]) + "|"
}
save_check = cloud_string_save(t_str, "Current Highscores");
var file = file_text_open_write("Highscores.txt");
file_text_write_string(file, t_str);
file_text_close(file);

上述代码从存储在全局数组 "Highscores "中的数值中创建了一个string ，然后将这个string ，供云服务存储，同时也将其写入一个文件，供本地存储。
