显示_消息_async

这个函数打开一个窗口，向用户显示你在函数中定义的信息。这是一个异步函数，因此GameMaker在等待回答时不会阻止它所运行的设备，而是继续正常地运行事件。一旦用户按下了 "好 "的按钮，一个异步的 对话框事件被触发，仅在该事件的持续时间内，将有一个ds_map存储在变量 async_load.

这个映射将包含两个键，"id "和 "status"，"id "是函数调用时返回的值，"status "是 true ，表示 "Okay "按钮被按下，或者 false ，表示消息被取消（如果有，因为不是所有目标平台都允许取消对话）。

注意这个功能 只用于调试和测试，不应该在已发布的游戏中使用。为此，你应该创建一个自定义的用户界面来接收玩家的输入，这样你就可以完全控制对话框的外观和行为。

语法。

show_message_async(string);

争论 类型 描述
string 字符串 The message to show to the user.

返回。

Async Request ID

扩展实例。

正在显示信息的object 的鼠标左键事件（例如）将有以下代码。

msg = show_message_async("You will now be taken to the store");

以上将显示一个给定的消息string 。消息ID被存储在变量 "msg "中，并将在异步对话框事件中使用，如下所示。

var i_d, stat;
i_d = ds_map_find_value(async_load, "id");
if (i_d == msg)
{
    if (ds_map_find_value(async_load, "status"))
    {
        url_open("https://play.google.com/store");
    }
}

上述代码将返回的 DS Map 的 "id "键与存储在变量 "msg "中的值进行检查。如果它们相同，它就会检查 "好 "按钮是否被按下（而不是窗口被关闭/取消），如果返回 true ，就会打开一个网址。

返回。对话
