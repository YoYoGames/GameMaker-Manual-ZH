对话

大多数现代计算机和移动设备都不喜欢在显示对话窗口时停止游戏。他们倾向于将此解释为游戏运行器已经停止，并可能认为这是一个错误，并过早地关闭应用程序。为了防止这种情况，GameMaker有异步对话功能，允许你请求某些形式的用户反馈或输入，但在任何时候都不会阻止游戏运行器，这意味着--当被调用时--游戏将在后台正常运行，同时用户对显示的对话窗口做出回应。一旦用户做出反应，一个对话异步事件将被调用，这样你就可以处理结果了。

注意：除了 shop_leave_rating()之外，这些函数 只用于调试和测试，不应该在已发布的游戏中使用。为此，你应该创建一个自定义的用户界面来接收玩家的输入，这样你就可以完全控制对话框的外观和行为。

下面是所有可用的异步对话功能的列表。

  • get_login_async
  • get_string_async
  • get_integer_async
  • show_message_async
  • show_question_async
  • shop_leave_rating
