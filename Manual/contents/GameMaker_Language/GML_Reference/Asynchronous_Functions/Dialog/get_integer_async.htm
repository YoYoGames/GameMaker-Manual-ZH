get_integer_async

这个函数打开一个窗口，显示信息以及一个空间，供用户输入一个值（它将包含提供的默认值开始）。这是一个异步函数，因此GameMaker在等待回答时不会阻止它所运行的设备，而是继续正常地运行事件。一旦用户输入了他们的string ，并按下了 "好 "的按钮，一个异步的 对话框事件被触发，仅在该事件的持续时间内，将有一个DS地图存储在变量 async_load.

这个映射将包含三个键， "id", "status", 和 "value"。 "id"是函数调用时返回的值，"状态 "将是 true ，表示 "好 "按钮被按下，或 false ，表示消息被取消（在适用的情况下，不是所有的目标平台都允许消息被取消）。最后 "value" 将返回用户输入的整数，作为string （如果没有输入，将返回一个空的string "" ）。

注意这个功能 只用于调试和测试，不应该在已发布的游戏中使用。为此，你应该创建一个自定义的用户界面来接收玩家的输入，这样你就可以完全控制对话框的外观和行为。

语法。

get_integer_async(string, default);

争论 类型 描述
string 字符串 The message to show in the dialog.
default 真实的 The default value.

返回。

Async Request ID

扩展实例。

正在显示信息的object 的鼠标左键事件（例如）将有以下代码。

msg = get_integer_async("How old are you?", 0);

以上将显示一条信息，要求用户输入一个string ，并按下 "好"。该函数ID存储在变量 "msg "中，并将在异步对话事件中使用，如下所示。

var i_d = ds_map_find_value(async_load, "id");
if i_d == msg
{
    if ds_map_find_value(async_load, "status")
    {
        global.Age = ds_map_find_value(async_load, "value");
    }
}

DS Map 上述代码将返回的"id"键与存储在变量 "msg "中的值进行检查。如果它们相同，它就会检查 "Okay "是否被按下（而不是窗口被关闭/取消），如果它返回 true ，它就会从返回的整数中设置一个全局变量。
