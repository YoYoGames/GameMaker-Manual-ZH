get_string_async

这个函数打开一个窗口，显示消息以及用户输入string （其中将包含提供的默认string 开始）的空间。这是一个异步函数，因此GameMaker在等待回答时不会阻止它所运行的设备，而是继续正常地运行事件。一旦用户输入了他们的string ，并按下了 "Okay "按钮，就会触发一个异步的Dialog事件 ，在该事件的持续时间内，将有一个DS地图存储在变量中 async_load.

这个映射将包含三个键，"id"、"status"和"result"。"id "是函数调用时返回的值，"status "将是 "Okay "按钮被按下时的 true ，或者是消息被取消时的 false （在适用情况下，不是所有目标平台都允许消息被取消）。最后，"结果 "将返回用户输入的string （如果没有提供，则返回一个空的string ""）。

注意这个功能 只用于调试和测试，不应该在已发布的游戏中使用。为此，你应该创建一个自定义的用户界面来接收玩家的输入，这样你就可以完全控制对话框的外观和行为。

语法。

get_string_async(string, default);

争论 类型 描述
string 字符串 The message to show in the dialog.
default 字符串 The default string.

返回。

Async Request ID

扩展实例。

正在显示信息的object 的鼠标左键事件（例如）将有以下代码。

msg = get_string_async("What's your name?","Anon");

以上将显示一条信息，要求用户输入一个string ，并按下 "好"。该函数ID存储在变量 "msg "中，并将在异步对话事件中使用，如下所示。

var i_d = ds_map_find_value(async_load, "id");
if i_d == msg
{
    if ds_map_find_value(async_load, "status")
    {
        if ds_map_find_value(async_load, "result") != ""
        {
            global.Name = ds_map_find_value(async_load, "result");
        }
    }
}

上述代码将返回的 DS Map 的 "id "键与存储在变量 "msg "中的值进行检查。如果它们相同，它就会检查 "Okay "是否被按下（而不是窗口被关闭/取消），如果返回 true ，它就会检查string 的 "结果"，以确保在设置全局变量之前没有返回空strings 。
