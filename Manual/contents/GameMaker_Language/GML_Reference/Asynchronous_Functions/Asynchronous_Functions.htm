异步函数

在GameMaker的上下文中，异步函数是指向网络server ，或要求用户输入，甚至是流式数据，同时让GameMaker继续运行而不阻塞正在运行的游戏。当收到信息回调时，一个特殊的Async事件被触发，反过来，它允许任何分配有Async事件的实例执行进一步的功能（尽管这不一定是事实）。所以，基本上，它是一种与一些外部资源（server ，用户或设备）沟通的方式，在等待回复时游戏或项目不会停止。

为什么这很重要？嗯，因为它允许GameMaker在发送或接收数据时继续运作，这意味着你可以做很多事情，都不会阻挡游戏loop ，比如。

  • 将数据从设备上流到你的游戏中，而不需要玩家等待东西的加载。
  • 有回调事件，只有在收到正确的信息时才做事情。
  • 与网络服务器通信并交换数据

这使得一切都以一种更流畅和不显眼的方式运作。应该注意的是，异步事件将被触发给所有 拥有它们的实例，所以你可以在一个实例中使用异步函数，并在另一个实例中拥有处理回复的异步事件，甚至是其他各种实例。你可以到这里了解更多关于异步事件如何工作的信息 -异步事件。
