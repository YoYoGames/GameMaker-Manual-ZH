gc_get_stats

通过这个函数，你可以检索到关于垃圾收集器当前状态的信息。该函数将返回一个结构体，该结构体将有以下成员变量（注意，这里的 "对象 "是指任何可以被垃圾收集的东西，而不是 object 浏览器中定义的一般Asset 实例）。

GC统计结构
变化的 类型 描述
objects_touched 真实的 这是垃圾收集器在前一帧中发现的活动objects 的数量。这将取决于收集的是哪一代。
objects_collected 真实的 垃圾收集器确定在前一帧中没有活动，因此可以删除的objects 的数量。
traversal_time 真实的 这是以微秒为单位的时间（在主线程上），垃圾收集器花了多少时间来确定哪些objects 是活动的。
collection_time 真实的 这是以微秒为单位的时间（在一个单独的线程上），垃圾收集器花了多少时间来清理被认为不活跃的objects 。
gc_frame 真实的 这是一个计数器，每次发生垃圾收集时都会递增。如果垃圾收集被禁用，它将不会增加。
generation_collected 真实的 这是最后收集的那一代的索引。0是最年轻的一代，3是目前最老的一代。
num_generations 真实的 这是垃圾收集的总次数。
num_objects_in_generation 阵列的 真实的s 这是一个数组（大小为 num_generations ），包含每一代中的objects 。

注意：在HTML5目标平台上，垃圾回收由JavaScript引擎处理，因此，如果在该平台上使用该函数，上述struct 中的成员变量将全部返回0。

当使用这个功能时，请注意objects ，只有当一个完整的生成处理完成后才会更新，这可能需要几帧，取决于帧时间的设置（关于帧时间的更多信息，见这里）。

语法。

gc_get_stats();

返回。

GC Stats Struct

例子。

if (global.debug == true)
{
    var _s = gc_get_stats();
    var _t = _s.traversal_time;
    var _c = _s.collection_time;
    show_debug_message("Traversal time = " + string(_t))
    show_debug_message("Collection time = " + string(_c))
}

上面的代码检查一个全局变量，如果它是 true ，它从垃圾收集器中获得信息并作为调试信息输出到控制台。
