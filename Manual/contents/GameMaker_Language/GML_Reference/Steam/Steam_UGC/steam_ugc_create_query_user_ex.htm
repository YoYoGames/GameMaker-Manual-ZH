steam_ugc_create_query_user_ex

这个函数可以用来查询UGC数据库。该函数需要用户的ID值和将要消费该物品的游戏的ID和/或创造该物品的游戏的ID。你还需要使用以下常量 用于查询的数据类型，查询的项目类型和返回项目的排序顺序，以及一个页码--注意，一个查询最多会返回50个项目。

ugc_list_Published 见Steam文档。
ugc_list_VotedOn -
ugc_list_VotedUp -
ugc_list_VotedDown -
ugc_list_WillVoteLater -
ugc_list_Favorited -
ugc_list_subscribed -
ugc_list_UsedOrPlayed -
ugc_list_Followed -

匹配类型常数 描述

ugc_match_Items_Mtx -

该函数返回一个唯一的查询句柄值，该句柄值应存储在一个变量中，以便在其他查询函数中使用。请注意，这个函数只是准备查询，而不是实际发送 - 为此，你必须调用函数 steam_ugc_send_query()- 而且你可以使用进一步的steam_ugc_query_*()函数在实际发送之前完善搜索请求。

语法。

steam_ugc_create_query_user_ex(list_type, match_type, sort_order, page, account_id, creator_app_id, consumer_app_id);

争论 描述
list_type 要创建的数据列表的类型（见下面列出的常数）。
match_type 要查询的UGC项目的类型（见下面列出的常数）。
sort_order 数据应该被排序的方式（见下面列出的常数）。
page 要查询的页码。
account_id Steam 的账户ID。
creator_app_id 项目创建者的应用程序ID。
consumer_app_id 消费者应用程序的ID。

返回。

Query Handle

例子。

query_handle = steam_ugc_create_query_user_ex(ugc_list_Published, ugc_match_Items, ugc_sortorder_TitleAsc, page, global.AccountID, 0, global.GameID);

上面的代码创建了一个查询请求，并把它的句柄存储在一个变量中，以便将来使用。

返回。UGC
