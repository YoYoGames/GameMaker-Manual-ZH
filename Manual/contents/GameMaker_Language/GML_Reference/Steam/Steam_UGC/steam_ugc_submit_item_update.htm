steam_ugc_submit_item_update

这个函数将提交由给定句柄索引的UGC项目到Steam Workshopservers ，并添加用于给定项目的更改注释。修改说明应该是一个string ，更新句柄是你调用该函数时返回的值 函数时返回的值。 steam_ugc_start_item_update（）。.当使用这个函数时，它将返回一个异步ID值，然后可以在触发Steam异步事件来报告项目的创建时进行解析。该事件将包含以下关键/映射值，在 非同步加载DS地图。

  • "id" - 调用函数返回的异步ID
  • "result" - 操作的结果（一个实值）。这将是GML 常数ugc_result_success或其他实数。所以你应该检查这个常数以确保调用成功，如果 否则就会出现工作不正常的情况。其余可能返回的值显示为Steam "EResult "的结果，你应该在SDK头文件中看到steamclientpublic.h的所有89个可能值。
  • "event_type" - 这个键将持有"ugc_update_item"的值。
  • "legal_agreement_required" - 将为true或false（更多细节见Steam文档）。

语法。

steam_ugc_submit_item_update(ugc_update_handle, change_note);

争论 描述
ugc_update_handle 要更新的UGC的唯一手柄。
change_note 将用于该项目的变更说明（astring ）。

返回。

Async ID

例子。

var app_id = steam_get_app_id();
var updateHandle = steam_ugc_start_item_update(app_id, global.Publish_ID);
steam_ugc_set_item_title(updateHandle, "My workshop item(3)!");
steam_ugc_set_item_description( updateHandle, "testing workshop...");
steam_ugc_set_item_visibility(updateHandle, ugc_visibility_public);
var tagArray;
tagArray[0] = "Test";
tagArray[1] = "New";
steam_ugc_set_item_tags(updateHandle, tagArray);
steam_ugc_set_item_preview(updateHandle, "promo.jpg");
steam_ugc_set_item_content(updateHandle, "WorkshopContent1");
requestId = steam_ugc_submit_item_update(updateHandle, "Version 1.2");

上面的代码得到了游戏ID，然后用这个ID和之前存储的发布的文件ID来为项目生成一个更新句柄。在更新被推送到工作室之前，这个句柄被用来更新各种信息。 servers 。

返回。UGC
