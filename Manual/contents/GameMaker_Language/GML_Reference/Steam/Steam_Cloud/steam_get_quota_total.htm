蒸汽_获取配额_总数

当使用Steam 云来存储和同步文件时，你必须设置你的游戏需要的空间配额。这个配额在每个支持云计算的游戏上执行，以每个用户每个游戏为基础，因此，例如，如果游戏X的配额是1兆字节，那么每个拥有游戏X的Steam 帐户最多可以在云中存储与该游戏相关的1兆字节的数据。用户拥有的任何其他支持云计算的游戏（例如，游戏Y）将不会受到游戏X所存储的数据的影响。新的Steamworks游戏的默认配额是1GB，但你可以在Steamworks控制面板中为你的游戏改变这个配额。

注意：一旦配额用完，文件写入将失败。如果你认为你的游戏用户的配额有可能被用完，你应该创建代码来处理它，因为如果你什么都不做，你就会让用户处于一种无法解决的情况，这将导致你的游戏体验不佳。

语法。

steam_get_quota_total();

返回。

Real

例子。

if steam_is_cloud_enabled_for_app()
{
    quota = steam_get_quota_total();
}

上面的代码检查是否为游戏启用了steam 云，如果是，它就会获得存储配额的大小，并将其存储在一个变量中。
