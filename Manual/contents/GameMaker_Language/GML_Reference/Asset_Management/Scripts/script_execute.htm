脚本_执行

这个函数最初是为以动态方式调用script 资产而设计的，但随着用户定义函数的引入，它现在可以以两种方式使用。

  • 第一种方法是支持传统的 scripts ，其中每个script 是一个单一的函数，script 的标题也是函数的名称。以这种方式使用，它将像以前一样工作，它将运行一个用户创建的script asset ，并将给定的参数传递给它。请注意，在这种情况下，它实际上并不是在调用脚本，而是在script 中调用同名的脚本函数。
  • 第二种方式是在一个包含多个函数的script asset ，调用一个函数。在这种情况下，你将提供函数的名称和它所需要的参数，而不是script asset 的名称。请注意，这只对脚本函数有效，而对方法无效，因为只有script 函数才有分配给它们的script ID。

虽然这个函数主要是为传统支持而设计的，但它并没有被废弃，因为它在网络和远程过程调用的情况下仍然有其存在的价值。还值得注意的是，这个函数很慢，经常使用它可能会对性能产生不利影响。

注意该功能不能与内置功能一起使用。

语法。

script_execute(scr, arg0, arg1, arg2..., etc...);

争论 类型 描述
scr 脚本资产或 脚本功能 The function/script that you want to call
arg0, arg1, arg2..., etc... 变化的 The different arguments that you want to pass through to the function/script

返回。

Variable (Will depend on the return value from the script/function being called)

例子。

script_execute(choose(move_up, move_down, move_left, move_right), irandom(5));

上面的示例代码将使用 script_execute ，从四个用户定义的函数中选择一个（本例中与运动有关），并将一个随机的整数作为第一个参数传给它。
