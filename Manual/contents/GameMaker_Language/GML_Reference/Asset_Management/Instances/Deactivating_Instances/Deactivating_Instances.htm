停用实例

GameMaker为你提供了 "关闭 "实例的可能性，使它们不再以任何方式被处理。从技术上讲，它们其实已经不存在了，只是作为停用过程本身的一个指针，这意味着一个停用的实例在再次被激活之前，根本不能被操作或改变。因此，使用这些函数时应该非常小心，因为如果使用不当就会引起问题，特别是对于持久性objects ，因为一个已经停用的持久性object ，除非先重新激活，否则不会被移到下一个room （有效地从游戏中删除）。还要注意的是，激活和停用并不是瞬间完成的，一个以这种方式改变了状态的实例在调用该函数的事件结束之前不会被认为是活跃/非活跃的。

通常没有必要在游戏的每一步都停用实例，这实际上会导致你的游戏出现性能问题。相反，建议你只在警报中每隔几步就运行这些函数（例如），或者当摄像机视图改变了位置时，尤其重要的是，你不要在绘制事件中使用这些函数，因为这可能导致你的游戏出现严重错误。

一个停用的实例实际上在游戏中已经不存在了，但单个实例仍然可以访问它们的变量。然而，你不能使用 with(object) 或 with(instance) ，而是必须使用它的唯一ID以下列形式直接访问实例。

val = inst.variable;

在上面的例子中，"inst "将是一个保存停用实例的ID的变量，这是停用的实例可以被引用的唯一方式，而它之前没有被激活。

注意如果你在room 启动时停用一个实例（即：从room 创建代码，或从room 内的实例创建事件），所有从room 编辑器放置在room 内的实例在停用前仍将运行其创建事件。

警告： 停用启用了物理的实例不会停止它们的灯具在物理模拟中的互动。为此，你应该在激活/停用这些实例时，将它们的phy_active变量设置为 true 或 false 。

有以下功能用于停用实例。

  • instance_activate_all
  • instance_activate_object
  • instance_activate_region
  • instance_activate_layer
  • instance_deactivate_all
  • instance_deactivate_object
  • instance_deactivate_region
  • instance_deactivate_layer
