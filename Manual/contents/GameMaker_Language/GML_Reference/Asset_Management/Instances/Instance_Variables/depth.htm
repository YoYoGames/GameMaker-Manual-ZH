深度

当你创建一个object ，你可以给它分配一个初始深度，这个深度定义了当游戏进行时，object 的实例将如何在room ，这个变量可以用来在游戏运行时获得和改变这个深度值。你通常不需要使用这个变量，因为你应该设置实例被添加到独立的图层中，而这些图层又被设置为一个特定的深度，但是你可能想用这个值来改变一个实例的深度，在这种情况下，一个 "临时图层 "将在给定的深度下专门为该实例创建。请注意，当没有实例处于相同的深度时，这个临时层将从内存中删除（不像普通的层，即使它们上面什么都没有，也会保留）。

重要的是!当你修改 depth 变量并且GameMaker管理图层时，内置的 layer变量将返回-1，而不是图层ID，因为管理图层不能通过代码操作。

在GameMaker中，一个实例的深度值越小，该实例就会被画得 "离摄像机越近"，而深度值越大，就意味着该实例会被画得 "离摄像机越远"，即：-1000被画在-100的上面，而后者被画在0的上面，以此类推。

请注意，具有相同深度的object 实例可以被画在彼此的上方或下方，即使它们在room 编辑器中看起来是一个放在另一个上面。如果你想保证某个东西被画在其他东西的上方或下方，你应该总是明确地设置深度（或层）。还要注意，有一个最小（-16000）和最大（+16000）的深度值，超出这个值的实例将不会被绘制，尽管它们仍然存在并处理事件。

注意 你不能在一个实例的绘图事件中设置其深度（所有其他事件都可以）。

请记住，修改一个实例的深度可能会改变应用在该实例上的滤镜和特效，因为将深度改为低于特效层的深度将不再在该实例上应用其效果。

语法。

depth;

返回。

Real (single precision floating point value)

例子。

if y != yprevious
{
    depth = -y;
}

上面的代码将检查y位置是否有变化，如果有，那么深度也将被设置为与之对应。
