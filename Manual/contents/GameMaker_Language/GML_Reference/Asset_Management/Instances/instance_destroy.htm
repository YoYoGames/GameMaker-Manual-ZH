实例_销毁

每当你想 "销毁 "一个实例时，你就会调用这个函数，通常会触发一个销毁事件和一个清理事件。这将把它从room ，直到room 重新启动（除非room 是持久的）。object 调用不带参数的函数将简单地销毁当前在范围内并运行代码的实例，但你可以提供一个可选的 "id "参数，并通过使用实例 id 的值来锁定一个特定的实例，或者你可以通过使用一个 object_index.比如说

instance_destroy(other);      // destroy the "other" instance in a Collision Event
instance_destroy(obj_Bullet); // destroy ALL instances of the object "obj_Bullet"

第二个可选的 flag ，允许你 "关闭 "正在销毁的实例的Destroy事件。默认情况下，Destroy事件将一直被执行，但是如果你把这个flag 设置为 false ，那么你可以让实例自己销毁，并跳过执行该事件。

重要的是!这将跳过销毁事件，但清理事件仍将被调用。

值得注意的是，当你销毁一个实例时，其销毁事件会在调用销毁函数的代码或动作之后立即被调用。另外，虽然执行了destroy事件，但该实例并没有立即从游戏中移除，它将继续执行当前事件中包含的代码。只有当当前事件结束时，它才会被从游戏中删除。

所以，如果你有，比如说，这个代码。

if (hp <= 0) instance_destroy();
score += 10;

变量 "score "将被递增，即使 instance_destroy 函数已经被调用，而实例最终将在事件结束时从你的游戏中删除。要注意这一点，因为如果你（例如）为这个实例创建了一个动态资源，像一个数据结构，然后在destroy事件中销毁了这个资源，如果在执行了destroy函数或动作后还有对它的引用，那么你会得到一个 "未知资源 "的错误，因为destroy事件把它从游戏中删除了。

语法。

instance_destroy([id, execute_event_flag]);

争论 类型 描述
id 实例ID或对象资产 OPTIONAL 要销毁的实例ID，或 销毁的实例（可选，默认为调用实例）。object asset
execute_event_flag 布尔型 OPTIONAL 设置为 或 ，以执行或不执行销毁事件（可选，默认为 ）。 true false true

返回。

N/A

例子。

if (bbox_right < 0 || bbox_left > room_width || bbox_bottom < 0 || bbox_top > room_height)
{
    instance_destroy(id, false);
}

如果实例在room 界限之外，上述代码将销毁运行代码的实例而不调用Destroy事件。
