asset_get_tags

通过这个函数，你可以从asset 浏览器中检索分配给一个Asset 的所有标签。你可以提供asset 的名称（作为一个string ）或它的asset 的索引，该函数将返回该asset 的标签阵列。如果没有找到标签或者有错误（即：给出的名称string 不存在），那么返回的数组将是空的。如果你提供了一个asset 索引值，那么你需要提供可选的asset 类型参数（一个常数），因为不同类型的assets 可以有相同的索引，尽管它们不能有相同的名字。可用的asset 类型在下表中列出。

恒定 描述
asset_object 给定的名称指的是一个对象。
asset_sprite 给定的名称指的是一个精灵。
asset_sound 这个名字指的是一种声音。
asset_room 这个名字指的是一个 room.
asset_tiles 给定的名称指的是一个瓦片组。
asset_path 给定的名称指的是一个路径。
asset_script 这个名字指的是一个 script.
asset_font 该名称指的是一种字体。
asset_timeline 这个名字指的是一条时间线。
asset_shader 给定的名称指的是一个着色器。
asset_animationcurve 该名称指的是一个动画曲线。
asset_sequence 给定的名称指的是一个Sequence。

语法。

asset_get_tags(name_or_index, [asset_type]);

争论 类型 描述
name_or_index 字符串或资产 The name of the asset (a string) or its index.
[asset_type] 资产类型不变 OPTIONAL! The type of asset to get the tags for, only used when supplying an index for the first argument.

返回。

Array

例子。

my_tags = asset_get_tags(object_get_name(object_index));

上述代码将检索分配给运行该代码的实例所创建的object 的所有标签。

返回。资产和标签
