精灵碰撞掩码

通过这个功能，你可以设置一个sprite 应该具有的碰撞遮罩的属性。如果你选择自动（0）或全图（1）作为边界框模式，那么单个边界框值可以设置为0。然而，对于用户定义的掩码（2），你将不得不设置这些值。不同的边框值总是相对于sprite 的左上角（无论x和y原点如何），这被认为是位置（0，0）。

设置掩码的种类可以设置掩码本身的一般形状，但要注意，除矩形掩码外的任何东西在解决碰撞时都需要更多的处理能力，性能也会随之下降。一般来说，你应该只在绝对必要时使用矩形以外的掩码类型。

注意：这个函数不允许旋转的矩形碰撞掩码种类。

可以设置的掩码种类将是以下常数之一。

界限盒种类（形状）常数
恒定 描述
bboxkind_rectangular 一个长方形（非旋转）的矩形碰撞面具形状
bboxkind_ellipse 一个椭圆的碰撞掩码形状
bboxkind_diamond 一个钻石碰撞的面具形状
bboxkind_precise 一个精确的碰撞掩码，该掩码将符合sprite ，基于给定的公差值（见下文）的非透明像素）。
bboxkind_spine Apart from rectangle, this is the only valid option for Spine sprites. It enables more precise collision checking against the mesh in the sprite.


最后，公差用于定义遮罩的精确程度（当与 "全图 "遮罩一起使用时，这将没有影响），公差为0意味着遮罩将跟随透明度超过0的每一个像素，而其他值将根据像素的透明度移动碰撞遮罩的周长。

注意：这个功能只对位图 sprites ，对SWF或JSON (Spine)sprites ，不会有作用。

注意：这个功能只对添加的sprites 或复制的sprites ，而不能直接对预先制作的资源起作用。你可以使用函数复制sprites sprite_duplicate().

语法。

sprite_collision_mask(ind, sepmasks, bboxmode, bbleft, bbtop, bbright, bbbottom, kind, tolerance);

争论 类型 描述
ind 雪碧资产 The index of the sprite to set the bounding box of.
sepmasks 布尔型 Whether to create collision masks for each sub-image of the sprite (true)，或一个面具用于所有(false)。
bboxmode 真实的 What kind of bounding box to use. 0 = automatic, 1 = full image, 2 = user defined.
bbleft 真实的 The maximum left position of the bounding box.
bbtop 真实的 The maximum top position of the bounding box.
bbright 真实的 The maximum right position of the bounding box.
bbbottom 真实的 The maximum bottom position of the bounding box.
kind 界限盒种类（形状）常数 The kind of mask, a constant (see the table in the description).
tolerance 真实的 Indicates the tolerance in the transparency value (0=no tolerance, 255=full tolerance).

例子。

spr = sprite_add("player_5.png", 16, true, true, 0, 0);
sprite_collision_mask(spr, true, 1, 0, 0, 0, 0, 0, 0);

上面的代码从外部来源加载一个sprite ，并将新的索引存储在变量 "spr "中。然后，该代码将新的sprite ，使其每个子图像都有一个精确的碰撞屏蔽。
