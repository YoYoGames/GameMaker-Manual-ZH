精灵合并

这个函数将合并参数1（"ind2"）中索引的sprite 与参数0（"ind1"）中索引的 。图像本身并没有被合并在一起，而是图像的索引被合并。 合并，来自sprite "ind2 "的子图像被附加到sprite "ind1 "的图像上，即：它们被加在最后。请注意，如果sprites 是不同的尺寸，那么附加的sprites 会被拉伸以适应 "ind1 "的图像尺寸。 的尺寸。

这种改变是永久性的，从你使用这个功能的那一刻起，直到游戏关闭或删除sprite ，被并入的sprite 将被改变，然而被并入的sprite 将保持不变。

注意：无论是被合并的图像asset ，还是用于合并的图像asset ，都不能是原始游戏资源之一。你必须使用 sprite_duplicate()首先 来创建两个新的资源（如果你要合并包括游戏assets ），然后再合并这些资源。

注意：这个功能只对位图 sprites ，对SWF或JSON (Spine)sprites ，不会有作用。

语法。

sprite_merge(ind1, ind2);

争论类型 描述
ind1 The index of the sprite to merge.
ind2 The index of the sprite that ind1 is to be merged with.

返回

例子。

var tspr;
tspr[0] = spr_add(working_directory + "Sprite Assets\explode1.png", 16, true, true, 0, 0);
tspr[1] = sprite_duplicate(spr_Explosion);
sprite_merge(tspr[0], tspr[1]);
sprite_index = t_spr[0];
sprite_delete(tspr[1]);

上面的代码将一个sprite 加载到一个局部变量中，然后将其与另一个已经重复的sprite 合并。最后，合并后的sprite 被分配给实例 sprite_index ，而加载的sprite 则被删除。 以释放其使用的内存。请注意，在这个操作之后的某个时刻，比如在Destroy或Room End事件中，你还需要使用 sprite_delete()来删除合并的精灵 从内存中删除。
