轨道结构

轨道struct ，既可用于顶层资产轨道，也可用于子轨道 参数轨道，轨道的行为由两点定义，即其名称和类型。轨迹结构是使用函数 sequence_track_new()创建，并且可以使用序列实例结构中的activeTracks 属性或序列sequence 结构中的tracks 属性从assets Object 中检索。

在轨道struct ，可用的属性是。

序列轨道结构
变化的 类型 描述
type 序列轨道类型常数 这包含一个 序列轨道类型常数来描述该轨道的类型。它可以是下面 "类型 "部分中的任何一个常数。
name 字符串 当创建一个 "顶级 "asset 轨道时，你在这里给出的名称可以是你需要的任何string ，以识别该轨道。然而，对于参数轨道，你需要指定具体的strings ，以告诉GameMaker你正在创建什么样的参数轨道。这一点将在下面的 "名称 "部分进行阐述。
tracks 阵列的 序列轨道结构s 该属性允许访问属于该轨道的子轨道的列表。获取此属性时，会返回一个数组序列轨道结构的数组被返回，而在设置此属性时，应指定一个 序列 轨迹结构的数组。
visible 布尔型 这表示该轨道是否可见（值为true ）或不可见（值为false ）。你可以获取或设置这个值，如果一个轨道不可见，那么它的子轨道也不会被绘制。
keyframes 阵列的 序列Keyframe 结构s 此属性允许访问该轨道的关键帧结构列表。获取此属性时，会返回一个keyframe structs 的数组；设置此属性时，应指定一个keyframe structs 的数组。

命名

name 属性可以是以下任何一个字符串。

  • "position"--当设置在类型为seqtracktype_real 的轨道上时，表示该轨道的内容应被用作位置数据。
  • "scale"--当设置在类型为seqtracktype_real 的轨道上时，这表示该轨道的内容应被用作缩放数据。
  • "rotation（旋转）" - 当在类型为seqtracktype_real 的轨道上设置时，这表明该轨道的内容应被用作旋转数据。
  • "blend_multiply（混合）"或"image_blend（图像混合）"--当设置在类型为seqtracktype_colour 的轨道上时，这表明该轨道的内容应被用作色彩乘法数据。这种轨道的keyframe 数据应该是一个由四个数值组成的数组，其格式为[A, R, G, B]。请注意，每个分量的值都表示在0和1之间，其中0对应的是HEX值#00，1对应的是HEX值#FF（0-255，在Sequences Editor中的图像混合轨道的颜色选择器中显示）。
  • "image_speed"--当设置在类型为seqtracktype_real 的轨道上时，表示该轨道的内容应被用作图像速度值。
  • "image_index" - 当设置在类型为seqtracktype_real 的轨道上时，这表示该轨道的内容应作为图像索引值使用。
  • "image_angle"（图像角度）--当设置在类型为seqtracktype_real 的轨道上时，表示此轨道的内容应作为图像角度值使用。


    文本轨道参数名称 （仅在文本轨道下使用）。
  • "frameSize（帧尺寸）" - 当设置在seqtracktype_real 轨道上时，这表示此轨道的内容应被用作文本的帧尺寸。该轨道的每个keyframe ，应该有两个通道，即0和1，其中0存储框架的X尺寸，1存储框架的Y尺寸。
  • "characterSpacing"（字符间距）--也用于seqtracktype_real 轨道，用作文本的字符间距。该轨道的每个keyframe ，都应该有一个包含间距值的单一通道。
  • "lineSpacing （行距）" - 也用于seqtracktype_real 轨道，用作文本的行距。该轨道的每个keyframe ，都应该有一个包含间距值的单一通道。
  • "paragraphSpacing（段落间距）" - 也用于seqtracktype_real 轨道，用作文本的段落间距。该轨道的每个keyframe ，都应该有一个包含间距值的单一通道。
