关键帧数据结构

当您为一个轨道定义关键帧结构时，您还需要定义与之相关的数据。这是由不同的通道组成的，其中每个通道只是一个Keyframe 数据Struct 。一个通道可以被赋予特定的数据类型，这取决于您为哪种类型的轨道设置keyframe 数据。

一个Keyframe 数据Struct ，包含以下变量。

序列Keyframe 数据结构
变化的 类型 描述
channel 真实的 这是keyframe 数据应该被应用的通道。它是一个从0开始的正整数，值得注意的是，当为 "位置 "或 "比例 "keyframes ，创建参数轨道时，你需要使用非常具体的通道值。这就是：通道0是X位置或X比例，通道1是Y位置或Y比例。
spriteIndex 雪碧资产 轨迹要使用的sprite asset 的索引。该属性仅适用于类型为 seqtracktype_graphic 的轨道，你可以获取或设置它。
soundIndex 声音资产 要用于该轨道的音频asset 的索引。这个属性只适用于 seqtracktype_audio 类型的轨道，你可以获得或设置它。
playbackMode 序列音频键常数 声音的播放模式。这个属性只适用于 seqtracktype_audio 类型的音轨，你可以获得或设置它。该属性应该是该属性下面的表格中给出的常数之一。
curve 动画曲线结构 这个属性需要一个animation curve struct （更多信息请看这里），并且只适用于类型为seqtracktype_real 的轨道。如果没有使用曲线struct ，那么这个属性的值将是-1。
value 真实的 此属性只是一个与keyframe 数据通道相关的值，只有在没有提供 curve struct 的情况下，才对 seqtracktype_real 类型的轨道可用。例如，如果放在"位置 "参数轨道内，这可以是该轨道的X或Y位置。
colour 阵列的 真实的s 此属性会返回（或要求，如果被设置）一个数组，用于格式为[A, R, G, B]的keyframe 的颜色值。这仅适用于类型为 seqtracktype_colour 的轨道。请注意，每个分量的值应表示为0和1之间，其中0对应的是HEX值#00，1对应的是HEX值#FF（0-255，在Sequences 编辑器中的彩色轨道的颜色选择器中显示）。
sequence 序列Object 结构 该属性将返回（或需要，如果被设置）一个序列object 结构，并且只适用于类型为 seqtracktype_sequence 的轨道。
objectIndex 对象资产 该属性将返回（或要求，如果被设置）一个object 的索引，并且只适用于 seqtracktype_instance 类型的轨道。
events 阵列的 弦乐s 此属性允许访问与keyframe 数据相关的事件和广播信息 struct 。你可以获取或设置这个属性，获取时返回一个strings 的数组，设置时应指定一个strings 的数组。关于事件的更多信息，请参见Sequence Events And Moments部分。该属性仅适用于类型为 seqtracktype_message 的轨迹。
event 方法 该属性将返回（或可设置为）与 method与keyframe 数据相关的struct 。如果没有指定方法或者希望删除该方法，那么该属性应该为-1。该属性仅适用于类型为 seqtracktype_moment 的轨道。

playbackMode 变量可以是以下常数之一。

序列音频键常数
恒定 描述 价值
seqaudiokey_loop 播放时，声音会loop 。 0
seqaudiokey_oneshot 声音只会播放一次然后停止。 1

文本跟踪数据

struct ，如果分配给一个文本轨道(seqtracktype_text)，将包含以下附加变量。

序列Keyframe 数据结构
变化的 类型 描述
text 字符串 这是在轨道上绘制的文本string 。
wrap 布尔型 这是一个布尔值，表示文本是否应该被包裹 (true) 或不被包裹 (false)。
alignmentH 序列文本对齐常数 这是文本的水平对齐，将是下面给出的一个 序列文本对齐常数中的一个。
alignmentV 序列文本对齐常数 这是文本的垂直对齐，将是下面给出的一个 序列文本对齐常数中的一个。
fontIndex 字体资产 这就是 字体资产文本轨道所使用的。

alignmentH 可以是以下任何一个常数。

序列文本对齐常数(alignmentH)
恒定 描述
seqtextkey_left 文本将在框架的左边对齐。
seqtextkey_right 文本将在框架的右边对齐。
seqtextkey_center 文本将在框架内水平居中。
seqtextkey_justify 文本将在框架内进行调整

alignmentV 可以是以下任何一个常数。

序列文本对齐常数(alignmentV)
恒定 描述
seqtextkey_top 文本将与框架的顶部对齐。
seqtextkey_bottom 文本将被对齐到框架的底部
seqtextkey_middle 文本将在框架内垂直居中。

返回。序列
