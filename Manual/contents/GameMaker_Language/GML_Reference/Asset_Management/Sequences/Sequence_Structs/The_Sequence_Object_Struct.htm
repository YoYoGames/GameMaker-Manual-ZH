序列Object 结构

A 序列Object 结构是给从struct Sequence 检索到的asset 的名称。一个Sequence 的所有数据都存储在这个struct ，在runtime ，并被Sequence 的任何元素所引用，这些元素是在room 中创建的。这意味着如果这个结构中的任何变量（或其任何嵌套的structs 内）被修改，原始Sequence 将改变（直到游戏被关闭和重新打开），其任何现有Sequence 的元素将被影响。

这个结构将有以下可以改变的属性。

序列Object 结构
变化的 类型 描述
name 字符串 sequence 这是作为string 的名称，你可以根据需要获得或设置这个值。注意，使用函数创建的sequences sequence_create()创建的string 将没有名字，这将只是一个空的 ""。
loopmode 序列播放模式不变 这是sequence object 的播放模式，可以得到或设置。这可以是这个下面的表格中显示的任何一个常数。
playbackSpeed 真实的 这指定了sequence 的播放速度，根据 playbackSpeedType （见下文），它被解释为每秒钟一帧或每游戏帧一帧。你可以获得或设置这个值。
playbackSpeedType 精灵速度常数 这指定了应该如何解释 playbackSpeed ，你可以获得或设置这个值。
length 真实的 sequence 的长度，以帧为单位。你可以获得或设置这个值，但要注意，如果引用这个sequence 的sequence 实例的播放头设置为超过新的长度，使sequence 更短可能会引起问题。
volume 真实的 这是一个从0到1的标量值，用于缩放sequence 中所有音轨的音量。你可以获得或设置这个值，它将修改所有音轨的全局音频输出--例如，如果你有一个音量为0.8的音轨，然后将sequence 音量属性设置为0.5，该音轨的最终音量将为0.4。
xorigin 真实的 这是沿X轴sequence 的原点。
yorigin 真实的 这是沿Y轴sequence 的原点。
messageEventKeyframes 阵列的 序列Keyframe 结构s 这允许访问keyframes 的消息事件sequence 。你可以获取或设置这些消息事件，当获取这个属性时，会返回一个keyframe structs 的数组，而对于设置这个属性，你应该提供一个keyframe structs 的数组。更多信息，请参见关于序列事件和时刻的页面。
momentKeyframes 阵列的 序列Keyframe 结构s keyframes 这允许访问sequence 的时刻事件。你可以获取或设置这些时刻事件，当获取这个属性时，会返回一个keyframe structs 的数组，而对于设置这个属性，你应该提供一个keyframe structs 的数组。更多信息，请参见关于序列事件和时刻的页面。
tracks 阵列的 序列轨道结构s 这允许访问sequence 顶层的资产轨道列表。你可以获取或设置该属性，当获取该属性时，会返回一个轨道structs 的数组，而对于设置该属性，你应该提供一个轨道structs 的数组。欲了解更多信息，请参见关于 轨迹结构.

loopMode 可以是以下任何一个常数。

备注

注意，如果你想访问已经在sequence 浏览器中创建的asset 的属性，你必须首先调用 索引上的函数，以检索 。 sequence_get()asset 来检索sequence object struct 。还要注意的是，对这个sequence struct 所做的任何改变将意味着这个sequence asset 的所有进一步的实例也会有这些改变，只要游戏在运行，它们就会被维持，即使你调用 game_restart() 函数。
