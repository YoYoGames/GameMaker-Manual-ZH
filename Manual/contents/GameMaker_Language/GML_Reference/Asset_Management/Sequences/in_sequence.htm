in_sequence

这是一个内置变量，是为你游戏中每个object 实例创建的实例变量的一部分。如果这个实例是由一个sequence 。 这个变量将返回 true ，否则它将返回 false 。这是一个只读的变量，不能被改变。

请注意，在控制的Sequence 结束后，这个变量将变成false ，即使它的Sequence 元素仍然存在，如果那个Sequence 元素再次被播放，它将变成 true 。

你可以在你的玩家（或CPU）控制的objects 中使用这个变量，以确保它们只在不在Sequence 中时才被代码移动。这可以证明对切入点很有用，当你需要在一个Sequence （使用 sequence_instance_override_object() 函数），并且你需要确保当该场景处于活动状态时，它们不能被它们的常规代码所移动。

语法。

in_sequence

返回。

例子。

if (!in_sequence)
{
    x += move_x;
    y += move_y;
}

上面的代码检查了 in_sequence 变量，如果它是 false （意味着这个实例没有被sequence ），那么它将 move_x 和 move_y 添加到实例的 位置，确保它只在不在Sequence 的情况下移动。
