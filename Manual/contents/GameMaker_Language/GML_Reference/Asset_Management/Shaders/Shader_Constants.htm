着色器常量

除了OpenGL ES着色语言（GLSL ES）参考页中定义的shader 函数和常量外，还有一些GameMaker特有的shader 常量可供你使用。

当使用shader 数组常数时，以下显示矩阵常数可作为数组索引gm_Matrices 。

恒定 描述
MATRIX_VIEW 这个数组索引常数持有当前视图矩阵的索引。当在shader 代码中调用 gm_Matrices 常数时，返回的索引将作为一个数组值使用。
MATRIX_PROJECTION 这个数组索引常数持有当前投影矩阵的索引。当在shader 代码中调用 gm_Matrices 常数时，返回的索引将作为一个数组值使用。
MATRIX_WORLD 这个数组索引常数保存了当前世界矩阵的索引。如果你有世界空间的光线信息，这可以用于照明等方面。当在shader 代码中调用 gm_Matrices 常数时，返回的索引将作为一个数组值使用。
MATRIX_WORLD_VIEW 这个数组索引常数持有世界和视图矩阵相乘的结果的索引。这通常用于像雾一样的东西。当在shader 代码中调用 gm_Matrices 常数时，返回的索引将作为一个数组值使用。
MATRIX_WORLD_VIEW_PROJECTION 这个数组索引常数持有世界、视图和投影矩阵相乘的结果的索引。这是用于vertex 位置的正常变换矩阵。当在shader 代码中调用 gm_Matrices 常数时，返回的索引将作为一个数组值使用。
MATRIX_MAX 这不是一个数组索引，而是返回vertex shader 中的矩阵数组的大小。
MAX_VS_LIGHTS 这不是一个数组索引，而是返回vertex shader 中的灯的数量。

以下常数也可用于照明。

恒定 描述
MAX_VS_LIGHTS 着色器中可用的最大的点光源和定向光源的数量。

以下预定义的矩阵制服和常数可以在你的shader ，以访问GameMaker的特定值。

恒定 描述
gm_Matrices[matrix] 这个数组常数返回一个变换矩阵，是GameMaker为你创建的可用的预定义制服之一，可以在shader 代码编辑器中使用。数组索引是从上面列出的常数中选择的。
gm_BaseTexture texture 这是一个2D采样常量，返回当前object ，由GameMaker设置。因此，它将是当前的sprite 、表面或texture ，在没有调用shader 的情况下，通常会在绘图时使用。
gm_LightingEnabled 当使用3D时，这可以用来获取或设置GameMaker的照明。
gm_FogStart 这可以用来获得多边形开始与雾色混合的距离。
gm_RcpFogRange 这可以用来得到雾气最大的距离，在这个距离上什么都看不到了。
gm_PS_FogEnabled 如果GPU启用或未启用像素雾，这将返回 true 或 false 。
gm_FogColour 这可以用来获得GameMaker使用的雾的颜色。
gm_VS_FogEnabled 如果GPU启用或未启用vertex fog，这将返回 true 或 false 。
gm_AlphaTestEnabled 这可以用来在shader 。 见 gpu_set_alphatestenable()以了解更多关于alpha测试的信息。
gm_AlphaRefValue 这可以用来获得当前的阿尔法测试参考值。参见 gpu_set_alphatestref()以了解更多关于阿尔法测试参考的信息。
