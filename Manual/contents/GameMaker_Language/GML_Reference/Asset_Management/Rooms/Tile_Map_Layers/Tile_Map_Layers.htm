瓦片地图层

GameMaker 房间编辑器允许你使用图层将tiles 添加到任何指定的room 。顾名思义，一个tile 地图层只是一个tile set tiles 的集合，它们都被认为在room 内处于相同的深度，通过堆叠层，你可以使一些东西在其他东西上面或下面绘制。你也可以从代码中控制图层的某些方面，添加或删除一些东西，或者在运行时改变图层的某些属性或它包含的内容。请注意，在room 编辑器中，每个图层只能有一个tile 地图（the 瓦片层)，但在代码中处理它们时，你可以为一个图层分配多个tile ，每个地图都有自己的唯一ID和属性，所以。

  • 瓦片集是你用来创建tile 图层和tile 地图的资源。
  • 瓦片层是你在Room 编辑器中创建的，用于将tile sets 。
  • 瓦片地图就是你所说的tiles ，它被添加到一个图层上，无论是在room 编辑器中还是通过代码，都是该图层上的一个单一元素

除了上面列出的设置tile 地图的函数外，你还有一些函数可以用来操作单个tiles 的数据。一个 "blob "的tile 数据是由tile 索引加上一些额外的位组成的，以显示tile 是否已经被旋转、翻转或镜像了。你也可以在tile 数据中加入你自己的位掩码，尽管这只适用于高级用户，因为你将掩码保留给tile 索引的位--基本上有19位保留给你的tile 索引，但如果你只使用其中的8位，那么你还有11位可以用来创建一个自定义掩码。然后这个掩码可以在代码中使用和检查，以创建自定义的tile 碰撞或其他。然而，你通常不需要创建你自己的tile 数据块，而会使用函数 tilemap_get()来获取tile 数据，然后用下面的函数来处理它，然后再用 tilemap_set().
