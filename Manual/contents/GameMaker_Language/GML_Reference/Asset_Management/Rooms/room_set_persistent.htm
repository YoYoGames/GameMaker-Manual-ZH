持续性的房间_设置

通过这个函数，你可以改变（或设置）你游戏中任何room 的持久性，除了当前的那个。如果一个room 的持久性被标记为 true ，它将在玩家离开后又返回时保持该room 中所有实例的状态，而如果持久性被标记为 false ，它将每次被重置为初始状态。你应该注意，一个持久化的room ，要比一个普通的room 消耗更多的内存，所以不建议在你的游戏中拥有太多的持久化 。

注意：如果switch 已经被访问过，这个功能将不会对room 关闭持久性起作用!一个持久性的room ，一旦被访问，就会被保存在内存中，要想关闭switch ，你应该去那个room ，把变量设置为 room_persistent变量为 false ，然后再次退出room 。

语法。

room_set_persistent(index, val);

争论 类型 描述
index 房间资产 The index of the room to set the persistence of.
val 布尔型 Whether the room should be persistent (true)或不(false)。

返回。

N/A

例子。

global.myroom = room_add();
room_set_width(global.myroom, 640);
room_set_height(global.myroom, 480);
room_set_persistent(global.myroom, true);

这将创建一个新的room ，并在变量 "global.myroom "中存储其索引。然后，它将设置其宽度为640像素，高度为480像素，标题为 "游戏室"，其持久性为 "true"。

返回。房间
