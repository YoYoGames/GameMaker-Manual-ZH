font_replace

font 这个函数可以用来在你的游戏中替换那些安装在系统上的fonts 。请注意，被替换的font 不能是font 浏览器中的Asset ，而必须是以前用函数添加的font 。 font_add().

当以这种方式替换font ，你可以定义font 的大小（以点为单位），以及font 是否应该是粗体或斜体，你还可以定义要包括的字符范围。如果该函数失败，那么它将默默地失败，所给的font 将不会被替换。

对于HTML5目标模块，这个函数只能用于替换Web字体，而且只有在WebGL未启用的情况下才会起作用--如果WebGL被启用，那么你必须使用一般的font 资源（即：使用 draw_set_font()和来自font 浏览器的Asset ）或位图fonts （见 font_add_sprite()).

重要提示：当以HTML5为目标时，只有当font asset 或网页font 本身包含这些字符时，才支持斜体和粗体文本。它们不会从基本的font 中合成。

在所有其他（非网络）目标上，你可以使用这个函数来替换一个文件中的font 。该文件必须使用GameMaker的Included Files功能包含在游戏包中，并且必须是 *.ttf 或*.otf 格式的font 文件，对于添加非标准的fonts ，如亚洲语或阿拉伯语，非常有用。

提示 如果你在游戏中包含这样的font 文件，请确保它有适当的许可与游戏一起发布。

当以这种方式从文件中加载font 时，font 的尺寸是以像素为单位的，而第一个和最后一个值被忽略，这意味着font 中的所有字形都将被添加。以这种方式添加的Fonts 被分配到他们自己的texture 页面，所以在使用这个功能时应该注意，因为它将在绘制时增加texture 的交换次数。还值得注意的是，fonts 在绘制时可能会显得稍大一些，因为字形可能有部分是在为font 定义的边界框之外绘制的。你也应该知道，对于一个特定的font ，在一个框架中可以渲染的字形有200个左右的限制。

注意 当你在GameMaker中加载一个font ，你必须记得在不再需要的时候再次删除它（用 font_delete())，否则会有内存泄漏的风险，这将使你的游戏变慢并最终崩溃。

语法。

font_replace(ind, name, size, bold, italic, first, last);

争论 类型 描述
ind 字体资产 The index of the (previously added) font to be replaced.
name 字符串 The name of the font to be used (eg 'Arial'), or the file path if the font is an included *。ttf 或 *.otf 文件。
size 真实的 The size of the font - points for Web Fonts, pixels for file fonts.
bold 布尔型 Whether the font is bold (true) or not (false).
italics 布尔型 Whether the font is italic (true) or not (false).
first 真实的 The first character to include (if you're unsure, go for 32).
last 真实的 The last character to include (if you're unsure, go for 128).

返回。

N/A

例子。

font_replace(myfont, "Arial", 24, true, true, 32, 128);

这将用一个新的font 替换变量 "myfont "中索引的font ，该字体大小为24pt，使用 "Arial"，是粗体和斜体。font 范围包括大写和小写字母、数字和所有常见的标点符号。
