font_texture_page_size

在使用函数时，这个内置变量可以用来获取或设置texture 页面大小。 font_add().在使用该函数添加font ，GameMaker将创建一个texture 页面cache 所需的字形，直至该变量定义的大小。这个过程如下。

  • 你使用的每个字符都会从font ，缓存到一个texture 。
  • 当texture 页面已满时（因为你使用了大的字符或很多不同的字符），GameMaker将从cache ，用你需要的新字符来替换它们。
  • GameMaker尽量不把texture ，因为不是所有的设备都能应付大的textures ，它也尽量不把texture ，因为在texture ，删除和重新添加字符需要CPU时间，导致你的游戏运行缓慢。

如果你认为你需要一个比GameMaker使用的默认1024px更大或更小的texture 页面尺寸，你可以使用这个内置的变量来设置这个值。如果你不使用 font_add() 功能，那么设置这个将不会对你的游戏性能产生影响，因为在fonts 中添加的IDE 被缓存在常规texture 页面上（你可以从字体编辑器和目标平台的游戏选项中控制）。

语法。

font_texture_page_size;

返回。

Real (maximum width/height in pixels)

例子。

font_texture_page_size = 512;
newfont = font_add("Arial", 24, true, true, 32, 128);

上面的代码将font cache texture 页面的最大尺寸设置为512px，然后在游戏中添加一个新的font 。

返回。字体
