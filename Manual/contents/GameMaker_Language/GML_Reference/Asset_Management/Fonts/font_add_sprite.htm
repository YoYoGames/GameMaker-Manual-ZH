font_add_sprite

通过这个函数，你可以使用 "精灵条 "来创建一个新的font asset ，其中每个子图像将是一个单独的符号或字母。索引是要使用的第一个字符的UTF8地图编号，使用的范围与普通font ，以"！"开始，但请记住，你应该只包括你需要的东西，你应该使用 ord()函数来获取起始字母的正确UTF8值。然后，创建的sprite font ，其子图像将被设置为来自font 的单个字形，其中 "第一个 "字形的image_index为0。你可以将 "空格 "定义为你想要的任何字符，例如，你想要的空格大小的单行，文本中的所有空格都将以该宽度呈现（但选择的图像将永远不会被呈现），或者如果你没有为空格提供sprite ，那么将使用sprite font 中最宽的字符的宽度代替。

你还可以将font 的间距设置为按比例 (true) 或不按比例 (false) ，其中按比例的font 是根据字母的实际宽度来确定间距的（例如，字母 "i "比字母 "w "所需的room ），而不按比例的font 将根据子图像的宽度来确定间距，就像单行线font 。最后，你可以定义书写时每个字母之间留出的空间，这可以是任何整数值，0表示没有空间（如果是按比例书写，字母会 "接触"）。

该函数返回一个索引值，该索引值应储存在一个变量中，因为在所有进一步引用该font ，这将是需要的，如果该函数因任何原因而失败，它将返回-1。

还值得注意的是，字符图中不能有任何空格，因此，如果你，例如，使用'.'和'0'，但不使用'/'，你需要包括'/'，或者在两个使用的字符之间留一个空白子图像。

sprite 本身可以是来自sprite 浏览器的asset Asset ，也可以是先用 sprite_add 从文件中加载，以获得加载的sprite 的索引（然后在函数中使用）。值得注意的是，为了使font 的对齐函数（如 draw_set_halign())在使用sprite font 的时候能够如期工作，sprite 的原点应该设置在左上角。如果你使用其他值，那么在使用font 绘制文本时，你将需要考虑到原点偏移。

注意 当你在GameMaker中加载一个font ，你必须记得在不再需要的时候再次删除它（用 font_delete())，否则会有内存泄漏的风险，这将使你的游戏变慢并最终崩溃。

语法。

font_add_sprite(spr, first, prop, sep);

争论 类型 描述
spr 雪碧资产 The sprite to add a font based on.
first 真实的 The index of the first available character.
prop 布尔型 Set as proportional font or not.
sep 真实的 The space to leave between each letter.

返回。

Font Asset

例子。

global.Font = font_add_sprite(spr_HUD_Font, ord("!"), true, 2);

上面的代码将从一个font asset 创建一个新的sprite asset ，并将新的font 的索引存储在变量 "global.Font "中，以便以后使用。

返回。字体
