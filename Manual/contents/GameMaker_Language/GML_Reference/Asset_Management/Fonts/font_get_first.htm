font_get_first（已弃用）。

当在GameMaker中定义一个font ，你可以定义一个包括的字符范围。这是因为font 本身实际上并不包括在你的游戏中（出于法律原因），但一个 的图像包含在一个 页面上，这就是你的游戏要使用的东西（就像其他图形 ）。 font 的图像包含在texture 页面上，这就是你的游戏将使用的东西（就像任何其他图形asset ）。这意味着你要尽量减少使用的字符数，只指定你的游戏需要的字符范围。 你的游戏将需要的字符范围，以便尽可能地优化texture 内存。这个函数可以用来查找当你的font asset 被添加到你的游戏中时使用的起始字符（作为ASCII值）。

注意 由于处理fonts 的方式发生了变化，这个函数将始终返回32，因为GameMaker将空格字符（32）强制纳入font ，并且不允许代码低于32的字符。 允许代码低于32的字符。在未来的更新中，将添加新的函数来完成类似的任务。

语法。

font_get_first(ind);

争论类型 描述
ind Index of the font to check.

返回。

例子。

num = font_get_first(fnt_Main);

上述代码将存储font 范围内的第一个字母的ASCII值，用于 "fnt_Main "中索引的font 。

返回。字体
