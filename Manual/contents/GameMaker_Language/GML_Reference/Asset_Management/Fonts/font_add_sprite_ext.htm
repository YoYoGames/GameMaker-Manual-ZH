font_add_sprite_ext

有了这个功能，你可以使用 "精灵条"（sprite 本身必须是sprite 浏览器中的asset Asset ，或者是你用 sprite_add 添加到游戏资源中的sprite ）来创建一个新的font asset ，其中每个子图像将是一个单独的符号或字母。不像普通的 font_add_sprite()sprite的子图像有一个特定的顺序，这个函数将根据函数的参数 "string_map "来映射sprite 子图像。这个参数是一个string ，你可以用它来告诉GameMaker，sprite font 的子图像是什么顺序，它将在编写文本时自动映射这些图像。因此，例如，如果你有一个 "AaBbCcDdEeFfGgHh "的字符串映射，你的sprite font ，必须有子图像以这种方式排序。你可以将 "空格 "定义为你想要的任何字符，例如你想要的空格大小的单行，文本中的所有空格将以该宽度呈现（但所选择的图像将永远不会被呈现），或者如果你没有为空格提供一个sprite ，那么将使用sprite font 中最宽的字符的宽度来代替。

你还可以将font 的间距设置为按比例 (true) 或不按比例 (false)，其中按比例的font 是根据字母的实际宽度来确定间距的（例如，字母 "i "比字母 "w "所需的room ），而不按比例的font 将根据子图像的宽度来确定间距，就像单行的font 。最后，你可以定义书写时每个字母之间留出的空间，这可以是任何整数值，0表示没有空间（如果是按比例书写，字母会 "接触"）。

该函数返回一个索引值，该索引值应储存在一个变量中，因为在所有进一步引用该font ，这将是需要的，如果该函数因任何原因而失败，它将返回-1。

值得注意的是，为了使font 的对齐功能发挥作用（如 draw_set_halign())，那么font sprite 的原点应该设置在左上角。如果你使用其他值，那么在使用font 绘制文本时，你将需要考虑到原点偏移。

注意 当你在GameMaker中加载一个font ，你必须记得在不再需要的时候再次删除它（用 font_delete())，否则会有内存泄漏的风险，这将使你的游戏变慢并最终崩溃。

语法。

font_add_sprite_ext(spr, string_map, prop, sep);

争论 类型 描述
spr 雪碧资产 The sprite to add a font based on.
string_map 字符串 String from which sprite sub-image order is taken.
prop 布尔型 Set as proportional font or not.
sep 真实的 The space to leave between each letter.

返回。

Font Asset

例子。

global.Font = font_add_sprite_ext(spr_CalcFont, "0123456789+-*/=", true, 2);

上述代码将从变量 "spr_CalcFont "中索引的font 创建一个新的asset sprite ，并将新的font 的索引存储在变量 "global.Font "中。sprite font 的子图像将被映射到指定的string 。

返回。字体
