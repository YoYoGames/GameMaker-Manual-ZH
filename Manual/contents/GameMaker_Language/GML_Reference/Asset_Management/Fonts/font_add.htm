font_add

这个函数可以用来从包含文件中的font 文件向你的游戏添加一个font 。你可以定义font 的大小（以点为单位），以及font 是否应该是粗体或斜体，你还可以定义要包括的字符范围。该函数返回一个索引值，该索引值应存储在一个变量中，因为在所有进一步引用该font 的代码中都需要这个索引值，如果该函数因任何原因而失败，它将返回 -1 。

请注意，font 文件必须是 *.ttf 或*.otf 文件，这对添加非标准的fonts 如亚洲或阿拉伯文fonts 也很有用。

警告！。 如果你在包含的文件中包括这样的font ，它必须获得许可才能与游戏一起发行。

对于HTML5目标模块，这个函数只能用于添加网络字体，而且只有在WebGL未启用的情况下才会起作用--如果WebGL被启用，那么你必须使用一般的font 资源（即：使用 draw_set_font()和来自font 浏览器的Asset ）或位图fonts （见 font_add_sprite()).

重要的是!当以HTML5为目标时，只有当font asset 或网页font 本身包含了这些字符时，才支持斜体和粗体文本。它们不会从基本的font 中合成。

当从文件中加载font 时，font 的大小是以像素为单位的，而第一个和最后一个值被忽略，这意味着font 中的所有字形都将被添加。以这种方式添加的Fonts 被分配到他们自己的texture 页面，所以在使用这个功能时应该注意，因为它将在绘制时增加texture 的交换次数。还值得注意的是，fonts 在绘制时可能会显得稍大一些，因为字形可能有部分是在为font 定义的边界框之外绘制的。你也应该知道，对于一个特定的font ，在一个框架中可以渲染的字形有200个左右的限制。

默认情况下，添加的fonts ，将启用抗锯齿。然而，你可以在任何时候使用函数来检查AA的状态 font_add_get_enable_aa()来检查抗锯齿的状态，你也可以用函数来启用/禁用抗锯齿。 font_add_enable_aa().

注意 当你在GameMaker中加载一个font ，你必须记得在不再需要的时候再次删除它（用 font_delete())，否则会有内存泄漏的风险，这将使你的游戏变慢并最终崩溃。

语法。

font_add(name, size, bold, italic, first, last);

争论 类型 描述
name 字符串 The path of the font file in the Included Files (*.ttf 或 ）。 *.otf
size 真实的 The size of the font - points for Web Fonts, pixels for file fonts.
bold 布尔型 Whether the font is bold (true)或不(false)。
italic 布尔型 Whether the font is italic (true)或不(false)。
first 真实的 The first character to include (if you're unsure, go for 32).
last 真实的 The last character to include (if you're unsure, go for 128).

返回。

Font Asset

例子。

new_font = font_add("gamefont.ttf", 24, true, true, 32, 128);

这将加载游戏附件文件中的 "gamefont.ttf" 文件，尺寸为24pt。font 将是粗体和斜体，字体范围将包括大写和小写字母、数字和所有常见的标点符号。font 被加载后，其ID将被存储在 new_font 变量中。

返回。字体
