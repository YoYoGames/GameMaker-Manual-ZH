路径_开始

path 是由一系列定义点创建的，这些定义点被连接在一起，然后用来规划一个实例的运动。它们可以用代码创建，也可以在Path 编辑器中创建，它们被分配给一个实例，在游戏中使用。然后你会用这个函数来告诉你的实例要跟随哪个path ，以什么速度来跟随path （以每步的像素来衡量），当它到达path 的终点时该如何表现，以及是跟随绝对位置还是相对位置path 。这最后一部分意味着它要么完全按照你在路径编辑器中设计和放置的path （绝对），要么从实例创建时的位置开始并跟随path （相对）。

正如你在上图中所看到的，这两个实例遵循的是以 path_start() 开始的同一个path ，但其中一个实例遵循的是绝对path ，而另一个则遵循的是相对path 。应该注意的是，当相对设置为false（绝对）时，无论实例放在path 的哪个位置，它都会 "跳 "到room 的起点。

用于定义实例在达到path 的终点时的行为的常数在这一页给出。

语法。

path_start(path, speed, endaction, absolute);

争论 类型 描述
path 路径资产 The path index to start.
speed 真实的 The speed of which to follow the path in pixels per step, negative meaning going backwards.
endaction 路径终点行动常数 What to do when the end of the path is reached (see the constants listed above).
absolute 布尔型 Whether the calling instance should follow the absolute path as it is in the editor (true) or a relative path to its current position (false).

返回。

N/A

例子。

path_start(path, 4, path_action_reverse, 0);

这将使调用的实例以每步4像素的速度跟随变量 "path "中索引的path 。它将跟随path ，相对于它自己的位置，当它到达终点时，它将改变方向，回到它来时的方向。
