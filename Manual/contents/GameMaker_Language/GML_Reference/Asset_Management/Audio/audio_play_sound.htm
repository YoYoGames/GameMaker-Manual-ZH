音频_play_sound

有了这个功能，你可以在你的游戏中播放任何声音资源。你提供声音的索引，并给它指定一个优先级，然后用它来决定当播放的声音数量超过函数设定的限制时，如何处理这些声音。 audio_channel_num().低优先级的声音将被停止，以利于高优先级的声音，优先级值可以是任何实数（实际值是任意的，可以是0到1或0到100，因为GameMaker会对它们进行同样的优先排序）。注意，在处理优先级时，数字越大，优先级越高，这样，优先级为100的声音将比优先级为1的声音更受青睐。第三个参数是让声音loop ，设置为 true ，将使声音loop ，直到停止，设置为 false ，将只播放一次声音。

这个函数也将返回一个正在播放的声音的唯一索引号，然后可以存储在一个变量中，这样你就可以用适当的函数暂停或停止它。这意味着，如果你在任何时候有多个相同的声音实例在播放，你可以只针对一个声音实例使用音频函数来处理。

语法。

audio_play_sound(index, priority, loop, [gain], [offset], [pitch], [listener_mask]);

争论 类型 描述
index 声音资产 The index of the sound to play.
priority 真实的 Set the channel priority for the sound.
loop 布尔型 Sets the sound to loop or not.
gain 真实的 OPTIONAL 获得的价值。
offset 真实的 OPTIONAL 设置起点的时间（以秒为单位）。长于给定声音长度的值将被忽略。
pitch 真实的 OPTIONAL 螺距乘数（默认为1）。
listener_mask 真实的 OPTIONAL 要为声音设置的比特掩码数据。在HTML5目标上，这将没有效果，因为目标不支持一个以上的听众。

返回。

Sound Instance ID

例1:

if health <= 0
{
    lives -= 1;
    audio_play_sound(snd_PlayerDead, 10, false);
}

上述代码检查 "健康 "全局变量，如果它小于或等于0，它将从 "生命 "全局变量中移除1，并播放一个声音。

例2:

if bbox_left > room_width
{
    audio_play_sound(snd_Goodbye, 10, false, 1.1, 0, 2);
}

上面的代码检查实例是否超过了room 的右边界，并播放一个增益为1.1、音高为2（正常音高的两倍）的声音。

返回。音频
