音频_调试

switch 这个功能可以用来显示音频系统的调试信息， true ，将其打开， false ，将其关闭。当你以这种方式debugging 音频时，覆盖层将显示以下信息。

  • Handle - OpenAL源处理（每一个播放的声音都是唯一的）。
  • 数据手柄- OpenALbuffer 手柄（对于具有相同源手柄的多个声音，显示相同的值）。
  • 已处理的缓冲区- 已经播放了多少buffers
  • 排队的缓冲区- 有多少buffers ，已经排队（并且仍在等待播放）。
  • Gain - 音频的增益值（0-65536）。
  • 播放偏移- 采样位置（当前播放位置）进入当前的缓冲区
  • 排队的Buffers Bars- 已处理/排队的buffers 的小方框图，播放偏移量（当前位置）以蓝线绘制。

对于同步的音频组，你也有这样的功能 audio_sync_group_debug().

注意 此功能在HTML5目标平台上不起作用。

语法。

audio_debug(enable);

争论类型 描述
enable Enable (set to true) 或禁用 (false) 音频debugging 。

返回。

例子。

if (debug_mode)
{
    audio_debug(true);
}
else
{
    audio_debug(false);
}

上述代码将switch ，根据游戏是否在调试模式下运行，打开或关闭音频调试叠加。

返回。音频
