音频同步组_调试

这个函数可以用来显示任何给定同步组的调试信息（组的索引是使用函数创建组时返回的值 audio_create_sync_group()).你可以用-1的值来调用该函数，以switch ，关闭覆盖层。

当你在debugging 音频同步组时，将显示叠加。

  • 哪个同步组正在调试
  • 里面有多少个频道
  • 同步组有多少次失败（这意味着它试图做太多的工作）。
  • 频道的列表
    • 红色表示这个通道目前没有做任何工作
    • 黄色表示该通道处于初始状态（还没有解码或播放）。
    • 绿色表示该通道目前正在寻找/解码
    • 紫色表示该通道已经用完了数据
  • 剖析条显示解码线程正在做什么工作
    • 蓝色表示正在首次读取一个文件
    • 红色表示线程正在读取一个文件，以确定从哪里读取。
    • 绿色表示线程正在对数据进行解码
    • 紫色表示该线程正在睡觉，直到需要工作为止

请注意，对于一般的音频debugging ，你还有以下功能 audio_debug().

注意：此功能不适用于HTML5目标平台。

语法。

audio_sync_group_debug(group_index);

争论 类型 描述
group_index 音频同步组ID The group index to debug, or -1 to switch off the debugging.

返回。

N/A

例子。

if (debug_mode)
{
    audio_sync_group_debug(sg);
}
else
{
    audio_sync_group_debug(-1);
}

上面的代码将switch ，根据游戏是否在调试模式下运行，打开或关闭音频同步组调试覆盖，显示在变量 "sg "中索引的同步组。

返回。音频同步化
