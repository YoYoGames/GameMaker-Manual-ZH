音频_播放_声音_开启

有了这个功能，你可以通过发射器播放任何声音资源，对发射器的增益、位置、音高或速度的任何改变都会影响用户听到的最终播放的声音。你提供要使用的发射器索引，要播放的声音索引，然后指定该声音是否要loop 。接下来，你可以指定一个优先级，然后用来决定当播放的声音数量超过函数设定的限制时，如何处理这些声音。 audio_channel_num().优先级较低的声音将被停止，而优先级较高的声音将被停止，优先级值可以是任何实数（实际值是任意的，可以是0到1或0到100，因为GameMaker将对它们进行同样的优先处理）。请注意，在处理优先权时，数字越大，优先权越高，例如，优先权为100的声音将比优先权为1的声音更受青睐。

这个函数也将返回一个正在播放的声音的唯一索引号，然后可以存储在一个变量中，这样你就可以用适当的函数暂停或停止它。这意味着，如果你在任何时候有多个相同的声音实例在播放，你可以只针对一个声音实例使用音频函数来处理。

语法。

audio_play_sound_on(emitter, sound, loop, priority, [gain], [offset], [pitch], [listener_mask]);

争论 类型 描述
emitter 音频发射器ID The index of the emitter to use.
sound 声音资产 The index of the sound to use.
loop 布尔型 Flags the sound as looping or not.
priority 真实的 Set the channel priority for the sound.
gain 真实的 OPTIONAL 获得的价值。
offset 真实的 OPTIONAL 设置起点的时间（以秒为单位）。长于给定声音长度的值将被忽略。
pitch 真实的 OPTIONAL 螺距乘数（默认为1）。
listener_mask 真实的 OPTIONAL 要为声音设置的比特掩码数据。在HTML5目标上，这将没有效果，因为目标不支持一个以上的听众。

返回。

Sound Instance ID

例1:

if global.SFX
{
    audio_play_sound_on(s_emit, snd_Explode, false, 1);
}

上述代码检查全局变量 "SFX"，如果它返回 true ，那么变量 "snd_Explode "中索引的声音将通过变量 "s_emit "中索引的发射器以低优先级和无循环的方式播放。

例2:

if hit == true
{
    audio_play_sound_on(s_emit, snd_Hit, false, 1, 1.3);
}

上面的代码检查执行代码的实例是否被击中。如果 true ，它通过发射器 "s_emit "播放声音 "snd_Hit"，增益略高为1.3。这里设置的增益乘以发射器的增益，得到声音播放的最终增益。

返回。音频发射器
