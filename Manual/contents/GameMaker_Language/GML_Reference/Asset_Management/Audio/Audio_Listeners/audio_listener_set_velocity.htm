音频听众设置速度

这个函数可以用来给给定的听众提供多普勒 效应，并根据从给定的相对x、y和z位置解析的矢量来模拟音频运动。默认的听众索引是0，但是你可以使用函数 audio_get_listener_info()来获得目标平台的不同指数。

如果给定的监听器永远不会移动，或者运动不是一个恒定的运动，你通常不需要设置这些值，但是，例如，如果你正在制作一个滚动游戏，玩家有一个从下到上的恒定运动，而敌人有一个从上到下的恒定运动，你将把监听器和发射器的速度（对于发射器你将使用 audio_emitter_velocity())到适当的矢量，以模拟它们在经过玩家实例时的正确多普勒效应。

下面的图片显示了如何为上面给出的游戏实例进行设置。

语法。

audio_listener_set_velocity(index, x, y, z);

争论 类型 描述
index 真实或音频听众ID The listener to change the velocity of (default 0).
x 真实的 The x velocity of the listener (default 0).
y 真实的 The y velocity of the listener (default 0).
z 真实的 The z velocity of the listener (default 0).

返回。

N/A

例子。

var num = audio_get_listener_count();
for(var i = 0; i < num; ++i;)
{
    var info = audio_get_listener_info(i);
    var data = audio_listener_get_data(info[? "index"]);
    if data[? "vx"] != 0
    {
        audio_listener_set_velocity(info[? "index"], 0, 0, 0);
    }
    ds_map_destroy(info);
    ds_map_destroy(data);
}

上面的代码检查可用的听众数量，然后loops ，如果他们的X速度不是0，就把他们的速度值设置为0、0、0。

返回。音频收听者
