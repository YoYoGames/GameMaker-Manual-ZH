音频_播放_声音_at

有了这个功能，你可以在音频空间内的一个给定位置播放任何声音资源。你提供声音的索引，然后给它在三维空间中指定一个位置。默认的听众位置是（0, 0, 0），所以这意味着如果听众没有被移动，而你想让声音来自左边（例如），你应该把X位置设置为负值（关于设置听众位置的更多信息，见 audio_listener_position()).

你也可以设置一个衰减距离（0会使声音无声，默认是100），这将使声音在离听众位置越来越远时逐渐消失。如何听到淡化本身将取决于淡化参考（也就是声源的音量通常会下降一半的距离）和滚落系数（只影响经过淡化参考距离的声音）。默认系数通常为1，不同的衰减值的效果取决于所选择的模型（关于不同的衰减模型以及这些值如何使用的完整指南，请参见函数 audio_falloff_set_model()).

接下来的两个参数是设置声音是否为loop ，以及为声音指定一个优先级。然后，这个优先级被用来决定当播放的声音数量超过函数设置的限制时，如何处理声音。 audio_channel_num().低优先级的声音将被停止，以利于高优先级的声音，优先级值可以是任何实数（实际值是任意的，可以是0到1或0到100，因为GameMaker将对它们进行同样的优先处理）。请注意，在处理优先权时，数字越大，优先权越高，例如，优先权为100的声音将比优先权为1的声音更受青睐。

这个函数将返回一个正在播放的声音的唯一索引号，然后可以存储在一个变量中，这样你就可以用适当的函数暂停它或停止它。这意味着，如果你在任何时候有多个相同的声音实例在播放，你可以只针对一个声音实例使用音频函数来处理。

语法。

audio_play_sound_at(index, x, y, z, falloff_ref, falloff_max, falloff_factor, loop, priority, [gain], [offset], [pitch], [listener_mask]);

争论 类型 描述
index 声音资产 The index of the sound to play.
x 真实的 The x position.
y 真实的 The y position.
z 真实的 The z position.
falloff_ref 真实的 The falloff reference relative to the listener (clamp).
falloff_max 真实的 The maximum falloff distance relative to the listener.
falloff_factor 真实的 The falloff factor (default 1).
loop 布尔型 Flags the sound as looping or not.
priority 真实的 Set the channel priority for the sound.
gain 真实的 OPTIONAL 获得的价值。
offset 真实的 OPTIONAL 设置起点的时间（以秒为单位）。长于给定声音长度的值将被忽略。
pitch 真实的 OPTIONAL 螺距乘数（默认为1）。
listener_mask 真实的 OPTIONAL 要为声音设置的比特掩码数据。在HTML5目标上，这将没有效果，因为目标不支持一个以上的听众。

返回。

Sound Instance ID

例1:

if global.SFX
{
    audio_play_sound_at(snd_Waterfall, x, y, 0, 100, 300, 1, true, 1);
}

上述代码检查全局变量 "SFX"，如果它返回 true ，那么变量 "snd_Waterfall "中索引的声音将在其room ，落差参考值为100，落差距离为300，落差系数为1，优先级较低。

例2:

if global.SFX
{
    audio_play_sound_at(snd_Waterfall, x, y, 0, 100, 300, 1, true, 1, 1, 2);
}

上述代码检查全局变量 "SFX"，如果它返回 true ，那么在变量 "snd_Waterfall "中索引的声音将在其room ，使用给定的衰减设置和不改变增益的情况下被循环播放。声音立即从第2段开始播放。

返回。音频
