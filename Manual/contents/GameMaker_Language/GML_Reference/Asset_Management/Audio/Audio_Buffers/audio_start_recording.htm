开始录制音频

这个函数将开始从索引的录音机源录制音频。你可以用函数获得录音机源的数量 audio_get_recorder_count()，一旦你开始录音，音频将被存储在一个临时的buffer ，并开始触发一个音频录音异步事件。这个事件在录音发生的每一步都会被触发，并将在变量中创建特殊的DS地图async_load中创建特殊的DS地图，其中包括以下键/值对。

  • "buffer_id" - 临时buffer 的ID，你可以用来检索音频数据。
  • "channel_index" - 录音通道的索引（由函数返回），这个数据来自于此。
  • "data_len" - 你收到的数据的长度（字节）。

注意，在异步事件被触发后，临时buffer 中的音频将被抹去，所以你应该把它的数据复制到你以前创建的自定义buffer 。

注意 大多数平台都支持以某种形式录制音频，但这并不意味着所有的设备都会允许，即使平台允许，所以你应该总是检查这个 audio_get_recorder_count()函数的返回值是否大于0，以验证在使用其他录音功能之前，录音设备是可用的。

语法。

audio_start_recording(recorder_index);

争论 类型 描述
recorder_index 真实的 The index of the recorder source to use.

返回。

Buffer ID

例子。

audio_record = audio_start_recording(0);

上述代码从录音机源0开始录音，将录音的通道索引存储在变量 "audio_record "中，以便在异步音频录音事件中使用。
