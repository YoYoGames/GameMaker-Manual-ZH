音频_创建_播放_队列

这个函数为音频准备了一个buffer queue 。你传递数据格式（目前只支持 buffer_u8 或 buffer_s16 ）、采样率（可以在1000hz和48000hz之间），以及音频所需的通道，从下面列出的常数之一。

恒定 描述
audio_mono 单声道（单通道）音频。
audio_stereo 立体声（双通道）音频。
audio_3D 3D（5.1）音频。


注意，当不再需要时，你应该释放用这个函数创建的任何音频queues ，以防止使用函数的内存泄漏。 audio_free_play_queue().

当queue ，这个函数将触发一个音频播放的异步事件，在这个事件中，一个特殊的DS地图将被创建在变量 async_load中创建一个特殊的DS地图，其中包含以下键/值对。

  • "queue_id"--已完成播放数据的queue
  • "buffer_id" - 不再播放的buffer 。
  • "queue_shutdown" - 在正常播放过程中被设置为0，在收到事件时被设置为1，因为 audio_free_play_queue()已经被调用。当它被设置为1时，你不希望再queue ，任何数据。

语法。

audio_create_play_queue(queueFormat, queueRate, queueChannels);

争论 类型 描述
queueFormat 缓冲区数据类型常数 The format of the buffer data to use (buffer_u8 或 ）。 buffer_s16
queueRate 真实的 The sample rate of the data in the buffer queue.
queueChannels 音频通道类型 The channels to use from one of the constants listed above.

返回。

Audio Queue ID

例子。

audio_queue = audio_create_play_queue(buffer_s16, 11052, audio_mono);

上面的代码创建了一个新的音频queue ，并将其ID分配给一个变量，以便将来参考。

返回。音频缓冲器
