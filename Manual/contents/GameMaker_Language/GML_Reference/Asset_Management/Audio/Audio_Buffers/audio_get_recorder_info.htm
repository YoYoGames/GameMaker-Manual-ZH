音频_get_recorder_info

这个函数将返回一个DS Map，其中包含给定的记录器源索引的信息。你可以用这个函数找出有多少个记录器源可用。 audio_get_recorder_count()，返回的地图将包含以下键/值对。

  • "名称" - 一个描述设备的名称
  • "索引" - 用来记录的索引
  • "data_format" - 数据将以何种格式返回（目前总是 buffer_s16 ，但将来可能会支持其他格式）。
  • "sample_rate" - 返回数据的采样率（以Hz为单位）（目前钳制在16000hz，但将来可能会改变）。
  • "channels" - 常数 audio_mono （将来可能会支持立体声和3D的进一步常数）。

注意，虽然该函数为你创建了一个DS地图，但它不会在以后再次删除它，因此你应该在不再需要该地图时自己销毁它，以防止任何内存泄漏。

注意 大多数平台都支持以某种形式录制音频，但这并不意味着所有的设备都会允许，即使平台允许，所以你应该总是检查这个 audio_get_recorder_count()函数的返回值是否大于0，以验证在使用其他录音功能之前，录音设备是可用的。

语法。

audio_get_recorder_info(recorder_index);

争论 类型 描述
recorder_index 真实的 The index of the recorder source to get the information from.

返回。

DS Map ID

例子。

len = async_load[? "data_len"];
audio_buff = buffer_create(len, buffer_fast, 1);
buffer_copy(async_load[? "buffer_id"], 0, len, buff, 0);
audio_queue_sound(audio_queue, audio_buff, 0, len);
audio_play_sound(audio_queue, 10, 0);

上述代码将在异步音频录制事件中被调用，并将一些录制的数据分配到一个buffer ，然后被添加到一个音频queue 。然后进行播放。

返回。音频缓冲器
