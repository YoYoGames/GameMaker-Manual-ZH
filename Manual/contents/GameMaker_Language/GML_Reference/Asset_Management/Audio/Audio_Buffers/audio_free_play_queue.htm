音频_free_play_queue

这个函数用来释放与给定的音频queue 相关的内存。queue 的索引是你用函数创建queue 时返回的值。 audio_create_play_queue()，这个函数应该在不再需要queue 时调用，以防止内存泄漏。释放queue ，将停止任何正在播放的声音，你不能删除正在流传的声音的buffer ，直到它所分配的queue 被释放。这个函数将触发一个音频播放的异步事件，在这个事件中，一个特殊的DS映射将在变量中创建 async_load中创建一个特殊的DS地图，其键/值对如下。

  • "queue_id"--已完成播放数据的queue
  • "buffer_id" - 不再播放的buffer 。
  • "queue_shutdown" - 在正常播放时，这个设置为0，在收到事件时设置为1，因为你已经用这个函数释放了queue 。当它被设置为1时，你不想再queue 任何数据。

语法。

audio_free_play_queue(queueIndex);

争论 类型 描述
queueIndex 音频Queue ID The index of the queue to free.

返回。

N/A

例子。

var q_id = async_load[? "queue_id"];
if (q_id == audio_queue)
{
    audio_free_play_queue(audio_queue);
    buffer_delete(audio_buffer);
}

上述代码将进入异步音频播放事件，并检查已完成播放的音频是否来自 "audio_queue"，如果是，则释放queue ，以及与之相关的buffer 。

返回。音频缓冲器
