call_later

这个函数在指定的时间或一定的时间间隔后触发一个回调函数。它通过创建一个时间源来实现（见时间源）。

要取消回调函数，见call_cancel。

重要提示虽然这个函数返回一个实际的时间源ID，但这个ID不能用于时间源函数，因为call_later和call_cancel使用的时间源是 "不可见的" - 它们不会作为任何内置时间源的子代出现，也不能用于其他时间源函数。然而，它们被包括在debug_event("ResourceCounts")返回的时间源计数中。

论点

下面是这个函数的参数明细。

期间

这是一个时间段，可以用秒或帧表示，这取决于下一个参数中指定的单位。

当使用帧作为单位时，周期必须是一个整数。非整数值被向下四舍五入（浮动），但低于1的值被向上四舍五入为1。

单位

这是周期表示的单位，可以是time_source_units_seconds或time_source_units_frames。

见。时间来源单位

你可以通过使用 time_bpm_to_seconds() ，使用每分钟的节拍值。

回调

你必须指定一个 "回调" method当时间源过期时被调用。

循环

时间源是否应该loop 。

语法。

call_later(period, unit, callback, [loop=false]);

争论 类型 描述
period 真实的 The period that the Time Source 运行，在给定的单位中
units 时间来源单位常数 The units that the given period is in
callback 方法或 脚本功能 The method or script function to call after the specified time
loop 布尔型 OPTIONAL 是否对时间源进行循环（默认为false）。

返回。

Time Source ID

例1:

var _callback = function()
{
    show_debug_message("Five seconds have passed!");
}
var _handle = call_later(5, time_source_units_seconds, _callback);

在这个例子中，我们创建了一个回调函数，在五秒过后显示一个调试信息。之后，底层的时间源被销毁。

例2:

var _create_enemy = function()
{
    var _x = random(room_width);
    var _y = random(room_height);
    instance_create_depth(_x, _y, depth, obj_enemy);
}

var _handle = call_later(10, time_source_units_seconds, _create_enemy, true);

在这个例子中，我们创建了一个回调函数，在room ，在一个随机的位置创建一个新的敌人。然后这个函数每隔10秒通过传递给call_later执行。最后一个参数被设置为 真所以这个函数每10秒就会被调用一次。

使用call_cancel和call_later返回的句柄来停止时间源和对回调的调用。
