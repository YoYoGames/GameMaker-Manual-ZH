time_source_reconfigure

这个函数用来修改一个时间源的核心属性，而不需要创建一个全新的时间源。

你指定一个现有的时间源，然后设置同样在 time_source_create()中设置的属性，但父源除外。阅读该页以了解关于这些属性的详细信息。

指定的时间源将被重置和停用，并需要重新启动。

语法。

time_source_reconfigure(id, period, units, callback, [args, repetitions, expiry_type]);

争论 类型 描述
id 时间源ID The Time Source 重新配置
period 真实的 The period that the Time Source 运行，在给定的单位中
units 时间来源单位常数 The units that the given period is in
callback 方法 The method to call when the Time Source 过期
args 阵列 OPTIONAL 一个包含传入方法的参数的数组。
repetitions 真实的 OPTIONAL 时间源应重复的次数，或-1表示无限期重复。
expiry_type 时间源到期常数 OPTIONAL 时间源是在离其到期时间最近的一帧上到期，还是在下一帧上到期

返回。

N/A

例子。

function change_spawn_delay(new_delay)
{
    time_source_reconfigure(obj_game.spawn_time_source, new_delay, time_source_units_frames, obj_game.spawn_method, [], -1);
    time_source_start(obj_game.spawn_time_source);
}

这创建了一个新的函数，改变了游戏中敌人的产卵延迟。

假设游戏使用一个名为obj_game.spawn_time_source的时间源来催生敌人，一旦催生延迟发生变化，该时间源就需要被更新。

这个功能正是这样做的，用新的延迟重新配置时间源，然后再次启动它。
