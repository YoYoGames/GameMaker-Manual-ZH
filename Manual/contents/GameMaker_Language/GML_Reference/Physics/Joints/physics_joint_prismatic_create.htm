physics_joint_prismatic_create

与渐开线关节一样，棱形关节只有一个自由度，但这种关节是相对于轴的方向性，而不是旋转性的，实际上是防止任何形式的旋转。这里有一张图片可以帮助你直观地了解它的工作原理。

我们有两个实例在锚点处连接，其运动轴由相对于物理世界(0,0)坐标的两个w_axis x/y坐标的矢量定义（所以一个x分量为0，一个y分量为1（0,1）的关节将成为一个垂直的棱形关节）。然后这个关节只能相对于这个轴线移动，就像一个弹簧或一个活塞。如果你设置了下限或上限，你基本上就是限制了沿这个轴的运动量，其中0的位置就是你用w_anchor x/y定义的确切位置，所以一个负值会沿着轴向 "左"，一个正值会向 "右"，如下图所示（要意识到 "左 "和 "右 "是相对术语！）。

你还可以将关节定义为有无电机。这意味着，当不受碰撞影响时，关节将沿轴线方向移动。这个方向由马达速度定义，正数是朝向轴线坐标（"右"），负数是朝向锚点（"左"）。max_motor_force "参数是用来限制运动速度的，这样你就不会得到一个永久加速的马达，并限制碰撞可能对运动的影响。通过这种方式，你可以使用关节马达来模拟关节摩擦，将关节速度设置为零，将最大力设置为某个小而重要的值。电机将试图阻止关节的移动，但会屈服于一个重要的负载。

和所有的关节一样，如果你把 "col "值设置为 true ，那么这两个实例可以相互作用和碰撞，但只有在它们有碰撞事件的情况下，然而如果它被设置为 false ，它们无论如何都不会发生碰撞。

语法。

physics_joint_prismatic_create(inst1, inst2, w_anchor_x, w_anchor_y, w_axis_x, w_axis_x, lower_trans_limit, upper_trans_limit, limit, max_motor_force, motor_speed, motor, col)

争论 类型 描述
inst1 实例ID The first instance to connect with the joint
inst2 实例ID The second instance to connect with the joint
w_anchor_x 真实的 The x coordinate where the joint is anchored, within the game world
w_anchor_y 真实的 The y coordinate where the joint is anchored, within the game world
w_axis_x 真实的 The x component of the axis vector
w_axis_y 真实的 The y component of the axis vector
lower_trans_limit 真实的 The lower permitted limit for the joint movement
upper_trans_limit 真实的 The lower permitted limit for the joint movement
limit 布尔型 Whether to limit the movement of the joint (true) or not (false)
max_motor_force 真实的 Sets the maximum movement speed for the motor
motor_speed 真实的 This is the speed at which the motor should move
motor 布尔型 Whether the motor should be active (true) or not (false)
col 布尔型 Whether the two instances can collide (true) or not (false)

返回。

Physics Joint ID

例子。

var mainFixture, o_id;
mainFixture = physics_fixture_create();
physics_fixture_set_circle_shape(mainFixture, sprite_get_width(sprite_index) / 2);
o_id = instance_create_layer(x+25, y, "Instances", obj_Piston);
physics_fixture_bind(mainFixture, id);
physics_fixture_bind(mainFixture, o_id);
physics_joint_prismatic_create(id, o_id, x, y, 0, 10, 0, 0, 0, 5, 0, 1, 1);
physics_fixture_delete(mainFixture);

上述代码创建并定义了一个新的夹具，然后创建了一个 "obj_Piston "的实例，将创建的夹具与两个新的objects 。然后，它们被一个棱形关节连接起来，其锚点位置与第一个实例的x/y坐标相同，轴线由x/y位置的矢量和x/y轴（在本例中是0,10，即 "向下"）形成。对沿此轴的运动量没有限制，但我们增加了一个速度为0、最大力为5的电机来模拟关节摩擦。
