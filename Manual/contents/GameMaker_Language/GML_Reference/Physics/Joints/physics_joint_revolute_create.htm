物理学_关节_复卷_创建

渐开线关节迫使两个物体共享一个共同的锚点（通常称为铰链点），该关节有一个自由度--两个物体围绕这个点的相对旋转。为了指定一个渐开线，你需要在room 中提供两个实例和一个锚点，正如你在提供的图片中看到的那样。

如果你看一下图片，你可以看到这两个实例已经被创建为重叠，在它们接触的地方，我们已经定义了一个渐开线关节。现在，由于 "ang_min_limit "和 "ang_max_limit "值的存在，这个关节的旋转自由度可以得到限制。它是如何工作的呢？好吧，让我们看一下另一张图片。

正如你所看到的，物理世界中的角度与标准的GameMaker角度不同，在GameMaker中，右边是0度，然后是逆时针方向，所以向上是90，左边是180，向下是270。不，在处理渐开线关节时，0度轴从关节位置到由函数定义的第二个实例的原点，然后角度以顺时针方向计算。如果你switch 角度限制，限制被定义为相对于这个0度轴，限制范围应该包括0，否则当room 开始时，关节会蹒跚。最后，你可以将关节定义为有无电机。这意味着，当不受碰撞影响时，关节将沿着一个方向移动，这个方向由电机速度定义，正数为顺时针，负数为逆时针。max_motor_torque "参数是用来限制旋转速度的，这样你就不会得到一个永久加速的电机，并限制碰撞可能对旋转产生的影响。通过这种方式，你可以使用关节马达来模拟关节摩擦，将关节速度设置为零，将最大扭矩设置为某个小但重要的值。电机将试图阻止关节旋转，但会屈服于一个重要的负载。

和所有的关节一样，如果你把 "col "值设置为 true ，那么这两个实例可以相互作用和碰撞，但只有在它们有碰撞事件的情况下，然而如果它被设置为 false ，它们无论如何都不会发生碰撞。

语法。

physics_joint_revolute_create(inst1, inst2, w_anchor_x, w_anchor_y, ang_min_limt, ang_max_limit, ang_limit, max_motor_torque, motor_speed, motor, col)

争论 类型 描述
inst1 实例ID The first instance to connect with the joint
inst2 实例ID The second instance to connect with the joint
w_anchor_x 真实的 The x coordinate for the joint, within the game world
w_anchor_y 真实的 The y coordinate for the joint, within the game world
ang_min_limit 真实的 The lower permitted limit for the joint angle
ang_max_limit 真实的 The upper permitted limit for the joint angle
ang_limit 布尔型 Whether the to limit the angle of the joint (true) or not (false)
max_motor_torque 真实的 Sets the maximum motor torque used to achieve the desired motor speed (in Newtons per meter)
motor_speed 真实的 This is the speed at which the motor should rotate
motor 布尔型 Whether the motor should be active (true) or not (false)
col 布尔型 Whether the two instances can collide (true) or not (false)

返回。

Physics Joint ID

例子。

var mainFixture, o_id;
mainFixture = physics_fixture_create();
physics_fixture_set_circle_shape(mainFixture, sprite_get_width(sprite_index) / 2);
o_id=instance_create_layer(x+25, y, "Instances", obj_Door);
physics_fixture_bind(mainFixture, id);
physics_fixture_bind(mainFixture, o_id);
physics_joint_revolute_create(id, o_id, x+25, y, -90, 90, 1, 0, 0, 0, 0);
physics_fixture_delete(mainFixture);

上述代码创建并定义了一个新的夹具，然后创建了一个 "obj_Door "的实例，将创建的夹具与两个新的objects 。然后，它们被一个没有马达的渐开线连接起来，角度限制在+/-90度的摆动。最后，夹具被删除，因为它不再需要了。
