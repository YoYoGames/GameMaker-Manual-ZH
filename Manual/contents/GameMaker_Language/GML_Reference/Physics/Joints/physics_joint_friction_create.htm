物理学_关节_摩擦力_创建

摩擦连接与物理模拟中的所有其他连接有点不同，因为创建的连接不会限制实例的位置或运动，而是限制其速度和旋转。它的工作原理是取力和扭矩的最大输入值，并将其应用到第二个夹具上，使速度和角动量降到与第一个实例相同的值。因此，如果你有一个静止的实例和一个移动的实例，然后用摩擦力连接它们，移动的实例将逐渐减慢，直到它也静止。如果两个实例都在移动，那么第二个实例的移动速度将被修改为与第一个实例的移动速度一致。

和其他大多数关节一样，你提供要连接在一起的实例，以及在room 。然后你提供最大的力和最大的扭矩（方向和旋转摩擦力），如果你把 "col "值设置为 true ，那么这两个实例可以相互作用和碰撞，但前提是它们有碰撞事件，然而如果它被设置为 false ，它们无论如何都不会发生碰撞。

语法。

physics_joint_friction_create(inst1, inst2, anchor_x, anchor_y, max_force, max_torque, col)

争论 类型 描述
inst1 实例ID The first instance to connect with the joint
inst2 实例ID The second instance to connect with the joint
anchor_x 真实的 The x coordinate for the joint, within the game world
anchor_y 真实的 The y coordinate for the joint, within the game world
max_force 真实的 The maximum frictional force that will be applied
max_torque 真实的 The maximum rotational force that will be applied
col 布尔型 Whether the two instances can collide (true) or not (false)

返回。

Physics Joint ID

例子。

var mainFixture, o_id;
mainFixture = physics_fixture_create();
physics_fixture_set_circle_shape(mainFixture, sprite_get_width(sprite_index) / 2);
o_id = instance_create_layer(x+300, y, "Instances", obj_Rudder);
physics_fixture_bind(mainFixture, id);
physics_fixture_bind(mainFixture, o_id);
physics_joint_friction_create(id, o_id, x, y, 10, 0.3, true);
physics_fixture_delete(mainFixture);

上面的代码将创建一个夹具，然后将其绑定到两个实例上，然后用摩擦力连接它们。
