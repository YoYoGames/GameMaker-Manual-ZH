physics_joint_pulley_create

滑轮接头是用来连接物理世界中的两个实例的，其方式是移动一个实例将直接影响另一个实例的运动。这些关节首先被固定在世界空间的两个点上，每个点都与一个实例的质心相连。这可以通过设置相对于实例原点的局部锚点X/Y坐标来改变，这意味着从实例到物理世界锚点的实际连接可以偏移到原点以外的地方。下面的图片说明了这一点。

如果你看一下inst2，你可以看到它的本地锚点已经向原点的 "左边 "偏移了20像素（记住！世界锚点是用，而本地锚点是相对于实例的原点定义的）。世界锚点是用room 坐标定义的，而本地锚点是相对于实例的原点定义的）。你还可以为滑轮接头指定一个比率，它告诉一方或另一方移动得更快，一个低于1的值使第一个实例移动得更快（例如，0.5会使它移动得快一倍），一个高于1的值使第二个实例移动得更快（例如，2会使它移动得快一倍）。

和所有的关节一样，如果你把 "col "值设置为 true ，那么这两个实例可以相互作用和碰撞，但只有在它们有碰撞事件的情况下，然而如果它被设置为 false ，它们无论如何都不会发生碰撞。

语法。

physics_joint_pulley_create(inst1, inst2, w_anchor1_x, w_anchor1_y, w_anchor2_x, w_anchor2_y, l_anchor1_x, l_anchor1_y, l_anchor2_x, l_anchor2_y, ratio, col)

争论 类型 描述
inst1 实例ID The first instance to connect with the joint
inst2 实例ID The second instance to connect with the joint
w_anchor1_x 真实的 The x coordinate where the first part of joint is anchored, within the game world
w_anchor1_y 真实的 The y coordinate where the first part of the joint is anchored, within the game world
w_anchor2_x 真实的 The x coordinate where the second part of the joint is anchored, within the game world
w_anchor2_y 真实的 The y coordinate where the second part of the joint is anchored, within the game world
l_anchor1_x 真实的 The local x coordinate where joint 1 is anchored to the first instance
l_anchor1_y 真实的 The local y coordinate where joint 1 is anchored to the first instance
l_anchor2_x 真实的 The local x coordinate where joint 2 is anchored to the second instance
l_anchor2_y 真实的 The local y coordinate where joint 2 is anchored to the second instance
ratio 真实的 Set the velocity ratio between two instances
col 布尔型 Whether the two instances can collide (true) or not (false)

返回。

Physics Joint ID

例子。

var t_fix, inst1, inst2;
t_fix = physics_fixture_create();
physics_fixture_set_circle_shape(t_fix, 20);
inst1 = instance_create_layer(150, room_height - 90, "Instances", obj_Block);
inst2 = instance_create_layer(300, room_height - 90, "Instances", obj_block);
physics_fixture_bind(t_fix, inst1);
physics_fixture_bind(t_fix, inst2);
physics_joint_pulley_create(inst1, inst2, 150, room_height - 140, 300, room_height - 140, 0, 0, 0, 0, 2, 0);
physics_fixture_delete(t_fix);

上面的代码创建并定义了一个新的夹具，创建了两个 "obj_Block "的实例，然后将这个夹具与它们绑定。然后在这两个实例之间定义了一个滑轮接头，接头没有偏移，比例为2:1（意味着实例2会移动得更快）。滑轮中的两个实例之间不会发生碰撞。
