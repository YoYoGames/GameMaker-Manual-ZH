physics_world_draw_debug

这个函数将画出room 的物理世界系统的表示，以帮助debugging ，当你在物理场景的工作中遇到困难时。有时，问题可能出现在你的游戏的渲染部分，例如，一个sprite ，可能在错误的位置或旋转中被画出来，造成不正确的物理外观，或者可能是你的计算不正确。无论哪种情况，这个函数都可以帮助你找到并解决这个问题，它可以画出你的物理系统中正在发生的事情。要做到这一点，你需要告诉它要画出系统的哪一部分，这是通过传递一个由存储在各种 "标志 "常数中的比特创建的值来实现的。

物理学调试标志
旗帜 描述
phy_debug_render_aabb 这显示了每个夹具与room 轴有关的绝对边界盒。
phy_debug_render_collision_pairs 这将显示当前处于碰撞状态的任何灯具
phy_debug_render_coms 这标志着房间里每个灯具的质量中心。
phy_debug_render_core_shapes 显示出构成房间里的装置的基本形状
phy_debug_render_joints 这将画出房间里所有灯具的每一个连接点
phy_debug_render_obb 这显示了房间里的灯具的相对边界框。
phy_debug_render_shapes 这显示了构成房间内固定装置的实际形状。

因此，为了绘制这些选项中的任何两个，我们需要对这两个选项做一个顺时针的 "或"（在GameMaker中用|表示），并将得到的变量存储起来供函数使用。下面是一个例子，说明你如何设置一个变量，以便与函数一起使用，该函数将只画出一个房间里所有固定装置的质心和连接点。

flag = phy_debug_render_coms | phy_debug_render_shapes;

这将给你带来这样的东西。

正如你在上面的代码中所看到的，屏幕上只画出了形状和质心（以及代表其状态的颜色--更多信息请查看Box 2D在线文档）。以同样的方式，您可以显示更多的，甚至是所有的，您希望调试的常量......。只要把它们 "或 "在一起就可以了还有一点需要注意的是--这个函数只能在一个深度低于你想调试的实例的绘制事件中调用，否则你将无法看到调试信息。

语法。

physics_world_draw_debug(flag)

争论 类型 描述
flag 物理学调试标志(s) The flag value for showing different debug features

返回。

N/A

例子。

flag = phy_debug_render_aabb | phy_debug_render_collision_pairs | phy_debug_render_obb;
physics_world_draw_debug(flag);

上面的代码将绘制所有的绝对边界框、相对边界框以及在当前物理系统内碰撞的固定装置。
