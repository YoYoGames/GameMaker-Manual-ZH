physics_fixture_set_sensor

有些时候，你会希望你的游戏能够检测并识别出两个带夹具的实例发生碰撞，但对碰撞没有任何物理反应。这可以通过将夹具转换为传感器来实现，这基本上意味着它们将产生一个碰撞事件，但没有物理反应，这样你就可以将这些实例作为游戏中其他事件的 "触发器 "room 。任何灯具都可以被标记为传感器，如果它所绑定的实例是静态的或运动的，则没有区别。

注意：传感器夹具将只在碰撞第一次发生时触发碰撞事件，这意味着当两个物体继续重叠时，你不会得到一个碰撞事件流（这就是传统上会发生的情况）。如果它们停止重叠，随后又重叠，就会有另一个碰撞事件被触发。

语法。

physics_fixture_set_sensor(fixture, state)

争论 类型 描述
fixture 物理夹具ID the index of the fixture
state 布尔型 whether a fixture is a sensor (true) or not (false)

返回。

N/A

例子。

physics_fixture_set_sensor(fix_Cloud, 1);

上面的代码将把 "fix_Cloud "中索引的灯具的传感器状态变成真实。
