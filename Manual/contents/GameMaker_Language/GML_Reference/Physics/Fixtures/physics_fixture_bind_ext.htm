physics_fixture_bind_ext

一旦我们定义了我们的灯具，它就必须被绑定到一个实例上。这意味着它的属性被转移到选定的实例上，而不是实际的灯具本身，因此，如果一个灯具具有相同的属性，它可以被绑定到多个实例上。你可以为目标指定一个object 索引，当时存在于room 中的所有实例都将获得该夹具的属性（但不包括后来创建的任何新的object 实例），或者你可以使用特殊的关键字other和all。你甚至可以指定一个父级object ，所有父级的子级实例也将接受该固定装置。一旦灯具被绑定到你需要的所有实例上，如果不再需要，它可以被删除，具有该灯具属性的实例将不会受到影响，并保持这些属性。

通常情况下，灯具将被绑定到实例上，质心被定位在实例的原点上，然而这并不总是你所需要的，所以这个函数也允许你将灯具绑定的X和Y位置偏移一个给定的量（如果你不需要这个，那么使用 physics_fixture_bind()来代替）。值得注意的是，一个灯具只能支持一个偏移量，因为Box2D不支持向一个灯具添加多个偏移量。

该函数还将为绑定的灯具返回一个唯一的 "id "值（不是灯具本身），然后可以使用该函数从实例中移除（"解除绑定"）物理属性。 physics_remove_fixture().这允许你从一个实例中添加和删除物理属性，而不需要破坏和重新创建objects 。

注意：当不再需要时，应该删除固定装置，因为不这样做可能导致内存泄漏，这将减慢并最终使你的游戏崩溃。

语法。

physics_fixture_bind_ext(fixture, target, xoffset, yoffset)

争论 类型 描述
fixture 物理夹具ID The fixture that is to be bound
target 实例ID或对象资产 The target instance that is to receive the fixture (can be an instance id, an object id, other, or all)
xoffset 真实的 The offset along the x-axis
yoffset 真实的 The offset along the y-axis

返回。

Real

例子。

var fix, inst;
fix = physics_fixture_create();
physics_fixture_set_circle_shape(fix, 16);
physics_fixture_set_density(fix, 1.0);
inst = instance_create_layer(x, y, "Instances", genericBodyObject);
my_fix = physics_fixture_bind_ext(fix, inst, sprite_width / 2, -(sprite_height / 2));
physics_fixture_delete(fix);

上面的代码将创建一个灯具并将其索引分配给变量 "fix"。然后定义夹具的形状和密度，再根据sprite 的宽度和高度将其绑定到实例上。绑定的夹具的索引被存储在变量 "my_fix "中。最后，夹具本身被删除，以防止内存泄漏，因为它不再需要了。

返回。固定装置
