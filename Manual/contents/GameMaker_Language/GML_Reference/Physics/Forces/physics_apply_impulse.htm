physics_apply_impulse

你不仅可以用GameMaker中的物理学对object ，而且还可以应用一个冲力。这与力略有不同，因为当它被应用时，将立即 影响速度，以及可能影响object 的扭矩（或 "旋转"），特别是如果选择应用冲力的点不与质量中心对齐（注意：质量中心不一定与原点相同！）。下面是一个例子。

正如你所看到的，玩家点击了球，这给了它一个冲动，让它向右上方移动，边走边旋转。确切的冲力是由我们从与xpos/ypos坐标相关的组件ximpulse/yimpulse得到的矢量定义的--这简单地意味着冲力被计算为从xpos/ypos到ximpulse/yimpulse的距离，单位是牛顿，而方向是我们从xpos/ypos到ximpulse/yimpulse的角度。

注意：物理世界的所有计算都使用米，每米的像素比例在room 编辑器中定义或使用 physics_world_create().

语法。

physics_apply_impulse(xpos, ypos, ximpulse, yimpulse)

争论类型 描述
xpos The x coordinate in the room where the impulse will be applied
ypos The y coordinate in the room where the impulse will be applied
ximpulse the x component of the impulse vector
yimpulse the y component of the impulse vector

返回。

例子。

if mouse_check_button_pressed(mb_left)
{
    with (place_meeting(mouse_x, mouse_y, all))
    {
        physics_apply_impulse(mouse_x, mouse_y, -10 + irandom(20), -10 + irandom(20));
    }
}

上面的代码将对按下左键时处于鼠标位置的实例应用一个随机矢量的脉冲。

返回。力量
