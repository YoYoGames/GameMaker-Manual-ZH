physics_apply_torque

当我们谈论扭矩时，我们通常谈论的是一个力或冲动使一个实例围绕其质量中心旋转的趋势，但在GameMaker中，你也可以独立地对一个实例施加扭矩，使其向一个给定的方向旋转。你应用多少扭矩来获得所需的旋转，完全取决于你分配给实例的夹具的大小和密度，因为这两个属性都是为了定义它的质量，而质量又定义了它的惯性，从而影响了扭矩一般的经验法则是，如果一个实例有很大的惯性，你将需要很大的扭矩来让它开始旋转。还应该注意的是，其他因素如重力和碰撞也会影响到一个实例的扭矩。这里有一张图片来帮助解释。

正如你所看到的，你需要旋转object 的扭矩取决于它的基本质量和惯性。这都是GameMaker根据创建实例时分配给它的夹具的大小和密度为你计算的。还要注意的是，对于逆时针旋转，这个值应该是负的。

语法。

physics_apply_torque(torque)

争论类型 描述
torque The torque (in Newton metres) to be applied

返回。

例子。

if keyboard_check(vk_left)
{
    physics_apply_torque(-100);
}

上面的代码在按下左方向键时，将使实例向逆时针方向（向左）旋转，扭矩为100Nm。

返回。力量
