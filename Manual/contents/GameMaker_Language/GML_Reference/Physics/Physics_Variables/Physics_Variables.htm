物理学变量

与 "传统的 "碰撞和运动系统一样，物理系统也有一系列的实例变量，在创建object ，分配给每个实例的变量。其中一些属性可以由用户在GML ，但许多属性是只读的，还有更多的属性只在碰撞事件中适用。

以下变量存在于所有启用了物理学的objects ，它们可以使用实例和script 函数的代码来读取和写入（即：改变）。

  • phy_active
  • phy_angular_velocity
  • phy_angular_damping
  • phy_linear_velocity_x
  • phy_linear_velocity_y
  • phy_linear_damping
  • phy_speed_x
  • phy_speed_y
  • phy_position_x
  • phy_position_y
  • phy_position_xprevious
  • phy_position_yprevious
  • phy_rotation
  • phy_fixed_rotation
  • phy_bullet

某些变量是只读变量，这意味着它们所包含的信息不能通过实例代码或script funcions来改变（但它可以被GameMaker内部改变，这就是为什么这些是变量而不是常量）。下面的列表显示了所有启用物理的对象中存在的只读变量。

  • phy_speed
  • phy_com_x
  • phy_com_y
  • phy_dynamic
  • phy_kinematic
  • phy_inertia
  • phy_mass
  • phy_sleeping

最后，有一些临时变量存在于启用了物理学的objects ，并且只在碰撞事件中创建。这些变量和上面列出的变量一样，都是只读的。

  • phy_collision_points
  • phy_collision_x
  • phy_collision_y
  • phy_col_normal_x
  • phy_col_normal_y
