gamepad_test_mapping

这个函数可以用来在那些允许的目标上设置游戏手柄的映射。你提供要设置的游戏手柄的 "插槽 "索引，然后提供地图string ，它应该是用SDL格式创建的，有以下字段。

<guid>,<description>,platform:<platform-name>,<bindings>

每个字段中应包括的细节是。

  • guid - 一个由数字和字母组成的 ，用于唯一识别设备类型（这可以使用string gamepad_get_guid()函数检索）。
  • description - 设备的Ascii描述（这可以使用 gamepad_get_description()函数检索）。
  • platform-name - 将是 "platform:android"、"platform:windows"、"linux "或 "mac"（注意，当你调用这个函数时，这将被自动附加到string ，因此你在构建自己的映射时不需要提供它 )strings
  • bindings - 一组用','隔开的条目，将实际输入与特定的 常量绑定在一起。GML

逗号分隔的条目列表遵循每个条目的特定格式。

<abstract-gp-name>:<value-binding>

这里 <value-binding> （即：游戏手柄的直接输入）可以是以下任何一种。

  • a<N> - 这代表游戏板的一个轴，其中<N>是0到1的一个轴。 gamepad_axis_count()-1
  • b<N> - 这代表游戏板的一个按钮，其中<N>是0-1的一个按钮。 gamepad_button_count()-1
  • h<N>.<mask> - 这代表游戏板的一个帽子，其中<N>代表一个从0到 gamepad_hat_count()-1，而 <mask> 将是1、2、4或8。

每个输入值都应该绑定到SDL名称上，这个名称表示为 <>abstract-gp-name> ，而这个名称又绑定到一个GML 常数。下表显示了SDL名称和GameMaker常量之间的等价关系。

当你构建了你的映射string ，它应该看起来像这样。

"050000005e040000fd020000ffff3f00,Xbox Wireless Controller,a:b0,b:b1,start:b4,dpdown:h0.4,dpleft:h0.8,dpright:h0.2,dpup:h0.1,guide:b6,leftshoulder:b9,leftstick:b7,lefttrigger:a4,leftx:a0,lefty:a1,rightshoulder:b10,rightstick:b8,righttrigger:a5,rightx:a2,righty:a3,x:b2,y:b3,platform:android"

语法。

gamepad_test_mapping(index, mapping_string);

争论 类型 描述
index 真实的 Which gamepad index "slot" to set.
mapping_string 字符串 The map string to use (see the description for more information).

返回。

N/A

例子。

var mapping = gamepad_get_guid(global.padIndex) + "," + gamepad_get_description(global.padIndex);
var len = array_length(global.PadInstances);
for (i = 0; i < len; i += 2)
{
    var left = global.PadInstances[i];
    var right = global.PadInstances[i+1];
    mapping += "," + left.sdlLabel + ":" + right.binding;
}
gamepad_test_mapping(global.padIndex, mapping);

上述代码将loop ，通过一些实例，并使用它们所包含的不同变量的值来创建一个映射string ，然后将其设置为在给定的插槽索引中的游戏手柄上使用。

返回。游戏手柄输入
