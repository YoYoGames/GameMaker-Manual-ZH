虚拟键_添加

这个功能使你能够将屏幕区域的 "触摸 "映射到键盘事件。这意味着，一旦你把一个区域分配给一个虚拟键，所有对该区域的触摸都会触发与你映射到该区域的键相应的键盘事件。你也可以把你定义的每个虚拟键分配给一个变量，然后可以在进一步的虚拟键功能中使用，以显示、隐藏和删除它们。这些键是以每个房间为单位分配的，当改变rooms ，GameMaker会自动删除。

虚拟钥匙的实际位置是基于屏幕的位置而不是room ，因此x/y值在屏幕上是绝对的。这意味着你不需要担心使用视图或相对的room 坐标，可以简单地在object 的创建事件中定义钥匙（你只需要在每个房间定义一次虚拟钥匙），然后在绘制GUI事件中绘制你的钥匙sprites ，GUI层的大小与屏幕相同。

语法

virtual_key_add(x, y, w, h, keycode);

争论 类型 描述
x 真实的 The x coordinate (left side) of the virtual key on the screen
y 真实的 The y coordinate (top side) of the virtual key on the screen
w 真实的 The width of the virtual key
h 真实的 The height of the virtual key
keycode 虚拟钥匙常数（vk_*）。 Which keyboard key event should be triggered by touching this area

返回。

Virtual Key ID

例子。

global.Left = virtual_key_add(32, 32, 64, 64, vk_left);

上面的代码创建了一个64x64像素的正方形虚拟键，在屏幕上的位置是（32，32），当触摸到这个虚拟键时将触发vk_left事件，并将这个虚拟键的索引分配给一个全局变量。
