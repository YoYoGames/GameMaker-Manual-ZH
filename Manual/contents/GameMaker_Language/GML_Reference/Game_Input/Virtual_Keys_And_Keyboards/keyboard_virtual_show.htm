键盘_虚拟显示

这个函数可以用来在运行游戏的设备上显示虚拟键盘。当你调用这个函数时，你需要为前三个参数中的每一个提供下列常数之一。

  • 键盘类型。这个常数用于设置虚拟键盘上可用的键盘组。可用的键盘类型有
虚拟键盘类型常数
键盘类型 描述
kbv_type_default 当前系统的默认键盘类型。
kbv_type_ascii 一个只有ASCII的键盘。
kbv_type_url 一个为URL输入而优化的普通键盘。通常具有".com "或其他域名键，以及"/"和". "键。
kbv_type_email 一个为电子邮件输入而优化的普通键盘。通常具有"@"和". "字符。
kbv_type_numbers 仅有数字的键盘，通常显示为数字键盘。
kbv_type_phone 一个电话垫键盘。通常只有数字的 "*"和 "#"键。
kbv_type_phone_name 为输入电话号码和姓名而优化的键盘。通常类似于ASCII键盘，但有有限的特殊字符选择。

  • 返回类型。这个常数用于设置虚拟键盘的返回/动作键上显示的内容。可用的返回类型有。
虚拟键盘返回类型常数
返回类型 描述
kbv_returnkey_default 当前系统的默认返回键标题。
kbv_returnkey_go 将返回键的标题设置为 "Go"。
kbv_returnkey_google 将返回键的标题设置为 "Google"，或在某些情况下设置为一个通用的搜索图标。
kbv_returnkey_join 将返回键的标题设置为 "Go"。
kbv_returnkey_next 将返回键的标题设置为 "下一步"。
kbv_returnkey_route 将返回键的标题设置为 "路线"。
kbv_returnkey_search 将返回键的标题设置为 "搜索"，或者在某些情况下设置为一个通用的搜索图标。
kbv_returnkey_send 将返回键的标题设置为 "发送"。
kbv_returnkey_yahoo 将返回键的标题设置为 "Yahoo"，或者在某些情况下设置为一个通用的搜索图标。
kbv_returnkey_done 设置返回键的标题为 "完成"。
kbv_returnkey_continue 将返回键的标题设置为 "继续"。
kbv_returnkey_emergency 将返回键标题设置为 "紧急呼叫"。

  • 自动大小写类型。这个常数用于确定通过虚拟键盘输入的单词应该如何自动大写。可用的自动大写类型是：。
虚拟键盘自动大小写类型常数
自动资本化类型 描述
kbv_autocapitalize_none 自动大小写功能被禁用。
kbv_autocapitalize_words 词语将被自动大写。
kbv_autocapitalize_sentences 句子将被自动加粗。
kbv_autocapitalize_characters 所有字符都将被大写。

最后一个参数是启用/禁用预测性文本，这将被设置为 true ，以允许它，否则 false ，但请注意，允许它并不意味着它将被使用，因为这将取决于设备上用户的偏好。当处于预测文本模式时，虚拟键盘将不会产生正常的GameMaker按键事件。相反，它将只更新最后按下的字符和键盘string 。这是由于无法检测用于检测按键的内部文本字段的变化是来自实际的虚拟键盘按键，还是文本建议。在这些情况下，你会希望读取 keyboard_string ，而不是读取任何种类的原始按键输入。

还需要注意的是，根据平台操作系统的不同，用户会得到不同功能的键盘，每个目标的使用有以下注意事项。

  • UWP：UWP的实现不支持 return_key_type, autocapitalization_type 或 predictive_text_enabled 。在函数中传递这些参数不会导致任何错误，只是不会对键盘产生任何影响。

  • 安卓。在Android ，不支持键盘类型 kbv_type_phone_name ，将显示ASCII键盘，返回键类型只能是 kbv_returnkey_go 、 kbv_returnkey_search 、 kbv_returnkey_next 、 kbv_returnkey_send 或 kbv_returnkey_done 。如果在该平台上使用其他返回键类型，系统将默认使用下面列出的替代返回键。
不可用的返回键 替换钥匙
kbv_returnkey_google
kbv_returnkey_yahoo		 kbv_returnkey_search
kbv_returnkey_join
kbv_returnkey_route
kbv_returnkey_emergency		 kbv_returnkey_go
kbv_returnkey_continue kbv_returnkey_next

  • AndroidTV / FireTV。ASCII键盘不支持自定义返回键类型--默认的返回键将始终显示。

  • tvOS tvOS：在tvOS ，不支持预测性文本和自动大写功能，由于键盘横跨整个屏幕，并且在 keyboard_virtual_height() ，键盘的矩形尺寸功能被禁用，因此系统事件和功能返回的键盘高度将默认为屏幕高度。还要注意的是，除非虚拟键盘被打开，否则物理（蓝牙）键盘事件将不会被广播。

调用这个函数将产生一个系统异步事件，在这个事件中， async_load DS地图将被填充以下键/值对。

  • "event_type" - 当被虚拟键盘功能触发时，这将是 "虚拟键盘状态"。

  • "screen_height" - 虚拟键盘的高度（像素）。如果键盘是不可见的，这将是0。

  • "keyboard_status" - 键盘的当前状态，以下列字符串之一返回。
    • "隐藏"
    • "隐藏"
    • "显示"
    • "可见"

语法

keyboard_virtual_show(keyboard_type, return_key_type, autocapitalization_type, predictive_text_enabled);

争论 类型 描述
keyboard_type 虚拟键盘类型常数 Determines the keyset available on the virtual keyboard (see tables at the top).
return_key_type 虚拟键盘返回类型常数 Determines what is shown on the return/action key of the virtual keyboard (see tables at the top).
autocapitalization_type 虚拟键盘自动大小写类型常数 Determines how/if the words typed via the virtual keyboard will be autocapitalized (see tables at the top).
predictive_text_enabled 布尔型 Set to true/false ，以启用/禁用预测性文本输入。

返回。

N/A

例子。

if input == false
{
    input = true;
    keyboard_virtual_show(kbv_type_numbers, kbv_returnkey_default, kbv_autocapitalize_none, false);
}

如果给定的变量没有设置为 true ，上述代码将调出操作系统虚拟键盘。
