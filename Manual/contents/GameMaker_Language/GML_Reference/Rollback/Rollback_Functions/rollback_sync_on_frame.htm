帧上的回滚_同步化

这个函数检查所有玩家在当前帧上是否同步，这意味着他们的状态不是基于预测，而是基于从他们那里收到的实际数据。在这种情况下，它返回真。

如果玩家在当前帧上没有同步，它将返回false，并尝试同步所有玩家。

试图用这个函数来同步播放器可能会导致冻结。你必须确保在一个步骤事件中不连续调用这个函数。

相反，把它作为一个切换器。见页面底部的例子。

当所有球员同步时，变量rollback_confirmed_frame ，并自动更新。

语法。

rollback_sync_on_frame();

返回。

Boolean

例子。

if (obj_game.winning_player == -1 && points >= obj_game.points_required_for_win)
{
    if (rollback_sync_on_frame())
    {
        obj_game.winning_player = player_id;
    }
}

这段代码在玩家对象的步骤事件中运行。

它检查是否没有获胜的玩家，以及当前玩家是否有获胜所需的积分。

如果两者都是真的，这意味着玩家应该赢。然而，在这样做之前，它要检查所有玩家是否同步，以确保状态正确。

如果所有的玩家都是同步的，它将设置wining_player变量。这将导致第一个条件变为假，确保rollback_sync_on_frame()不再被调用。

这意味着该函数被用作切换，而不是每一帧连续运行。
