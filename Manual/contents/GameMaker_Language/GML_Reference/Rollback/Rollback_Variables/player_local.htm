运动员_本地

当你使用 rollback_define_player()来定义一个球员object ，系统会自动为每个球员创建一个该object 的实例。该object 的每个实例都会得到 player_local 变量，它告诉你该球员实例是否是本地的。

如果你不使用 rollback_define_player() ，那么这个变量将不会为任何objects 。

你可以使用这个变量来显示哪个玩家是本地的。不建议以影响你的游戏逻辑的方式使用这个变量，这对本地和远程玩家来说应该是一样的。

语法。

player_local;

返回。

Boolean

例子。

// Draw event
if (player_local)
{
    draw_sprite(spr_arrow, 0, x, y - 60);
}

上面的代码在Draw事件中运行，并检查玩家实例是否为本地。在这种情况下，它会在实例上方画一个箭头sprite ，告诉玩家他们正在控制哪个实例。
