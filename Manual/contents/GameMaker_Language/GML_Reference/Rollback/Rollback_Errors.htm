回滚错误

在使用回滚系统时，可能会出现某些错误。这些错误是 "运行者错误"，所以它们在游戏运行时显示。

下表列出了你在使用回滚系统时可能遇到的错误和描述。它还为每个具体的错误信息提供了解决方案。

误差 描述
无法获得创建玩家的图层。你应该把图层名称传给rollback_define_player函数或创建Instances图层。 如果用rollback_define_player函数定义了播放器object ，回滚系统将尝试在该层上创建播放器object ，该层的名称可以选择作为第二个参数传递（如果没有定义，则在名称为 "Instances "的层上）。如果无法通过其名称找到该层，就会发生错误。解决的办法是明确设置创建播放器的图层名称，或者如果你不明确设置图层名称，就创建一个名称为 "Instances "的图层。
在运行多人游戏时，必须对玩家objects 进行管理。 如果播放器object 是用rollback_define_player函数定义的，而播放器object 没有被标记为托管，这个错误就会发生。解决办法是在对象编辑器中把播放器object 标记为被管理的。
获取玩家ID N的用户信息失败。 如果回滚系统无法从server ，获得关于玩家的信息，就会引发这个错误。检查开发者控制台，看是否有任何请求被阻止。
在完全连接到游戏之前无法调用rollback_chat。试着等待rollback_game_info事件。 聊天功能在玩家加入room 。 解决方案。在收到rollback_game_info事件前禁止聊天（见rollback事件）。
rollback_use_manual_start必须在rollback_create_game和rollback_join_game之前调用。 如果一个函数调用违反了内部状态机约束，这些错误就会被触发。错误信息建议了一个解决方案。
rollback_use_player_prefs必须在rollback_create_game和rollback_join_game之前调用。
rollback_use_player_prefs必须在rollback_set_player_prefs之前调用。
rollback_set_player_prefs必须在rollback_start_game之前调用。
rollback_define_player必须在rollback_create_game和rollback_join_game之前调用。
rollback_define_input必须在rollback_create_game和rollback_join_game之前调用。
rollback_define_mock_input必须在rollback_create_game和rollback_join_game之前调用。
在游戏开始前不要读取玩家信息。调用rollback_get_info()。
在游戏开始前不要读取玩家的输入。调用rollback_get_input()。
rollback_define_input_frame_delay必须在rollback_create_game和rollback_join_game之前调用。
rollback_define_input_frame_delay必须在rollback_create_game和rollback_join_game之前调用。
rollback_set_player_prefs在设置偏好时失败。 server 拒绝了播放器的预设。通常这种情况发生在以下情况。1.一个用户试图重置玩家预设，而另一个用户已经开始游戏。在这种情况下，游戏应该使用最后成功设置的用户预设，我们保证每个玩家在游戏开始前应该设置一些用户预设。2.串行化的玩家预设大小超过了最大的玩家预设大小（目前是800字节）。
rollback_create_game，玩家数量设置为N，但必须在1到4之间。 在调用rollback_create_game函数时设置了错误的玩家数量。解决方法是将rollback_create_game的第一个参数固定在1到4的范围内。
rollback_create_game在设置默认玩家偏好时失败。 参见 "rollback_set_player_prefs未能设置偏好 "的描述 。
多人回滚只在operagx目标中支持。 在调用rollback_create_game函数时，目标没有设置为OperaGX。目前多人游戏只在OperaGX平台上被支持。解决方案是将目标设置为OperaGX。
创建游戏失败（未知会话类型）。 这是回滚系统的一个内部错误。向GameMaker团队报告一个问题。
在这个实例上找不到player_id，你需要把它传给rollback_get_input()。 rollback_get_input函数可以接受玩家ID作为参数。如果没有向该函数提供参数，它将采用玩家object 的player_id字段来获取输入。如果参数player_id没有被设置，这个错误将被触发。如果用rollback_define_player创建了播放器object ，它应该有player_id的设置。解决方案。1.明确地将播放器ID参数传递给函数 2.用rollback_define_player创建播放器。
信息播放器_ID N超出了范围。 rollback_get_info在调用时有错误的player_id（或玩家object 在player_id字段中有错误的值）。解决方案。修正参数player_id的范围为1到4。
无法找到回滚数据中引用的object <id> (<name>)的实例<id>。 回滚系统的内部错误。 向 GameMaker团队 报告一个问题。
不知道如何创建OBJECT_KIND=<kind>。
RefID <id>没有object （畸形输入？）。
无效的ReferenceKind <id>（畸形的输入）。
实例映射已启用，但我们试图反序列化一个超出范围的索引。 <id>不在[0, <id>]中。
在反序列化中无效的RValue。
试图复活一个已经被删除的实例!
不能用kind=<id>（<name>）来序列化RValue。
空的输入定义。 rollback_define_input被调用，输入为空。
输入定义包含常数<键码>两次，这是不允许的。 一个键码只能被映射到一个输入。检查在输入定义中是否有任何键码被设置超过一次。
标签<label>的输入定义不是实数或实数数组。 所有描述输入值的值都应该有类型 真实.
空的模拟输入定义的播放器_id N。 见 "空的输入定义"。
模拟定义包含非真实值。 参见 "带有标签<label>的输入定义不是实数或实数阵列"。
