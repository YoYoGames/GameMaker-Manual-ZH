网络和HTML5

GameMaker提供了各种与互联网或与server ，它们可以在所有可用的目标平台上使用。这些功能是：。

  • url_open
  • url_open_ext
  • url_open_full
  • url_get_domain
  • http_request
  • http_get
  • http_get_file
  • http_post_string

除了上面列出的一般功能外，HTML5目标还有一些额外的网络功能，是专门为针对它的游戏而设计的。

  • browser_height
  • browser_width
  • browser_input_capture
  • webgl_enabled
  • clickable_exists
  • clickable_add
  • clickable_add_ext
  • clickable_change
  • clickable_change_ext
  • clickable_set_style
  • clickable_delete
  • analytics_event
  • analytics_event_ext
  • os_browser
  • http_get_request_crossorigin
  • http_set_request_crossorigin

除了HTML5目标模块的这些功能外，还有一个特殊的JavaScript变量，可以在html5game文件夹的内容被存储在一个完全不同的地方（URL）的罕见情况下使用。通常情况下，这个path 是在 JavaScript 文件中硬编码的，是相对于调用 JavaScript 的位置而言的，但您可以在调用游戏script 之前设置变量 g_GameMakerHTML5Dir ，游戏将从新定义的位置运行。比如说。

<script>var g_GameMakerHTML5Dir = "http://mygamealternatedirectory.com/";</script>

flag script 还值得注意的是，你可以使用前缀gmcallback_ ，这意味着当你为HTML5编译项目时，这些函数不会被混淆。在使用上面列出的 clickable_* 函数时，这是非常有用的（更多信息，请看这里）。
