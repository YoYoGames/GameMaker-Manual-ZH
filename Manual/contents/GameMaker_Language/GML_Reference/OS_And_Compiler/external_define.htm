外部定义

这个函数可以用来定义对一个特定的dll（用于Windows ）或dylib（用于Mac）的外部函数调用。这个文件可以是一个内含文件，也可以是extension 的一部分。

你提供文件的名称（和path ），然后是你希望定义的函数的名称。接下来你需要定义要使用的调用约定（见下面的常量列表）以及预期的结果类型（也是一个常量，如下所示）。

最后，你必须给出该函数可以接受的参数数量（从0到15），对于每个参数，你也必须指定其类型。请注意，对于有4个或更多参数的函数，所有参数的类型必须是 ty_real 。

注意：这只适用于已经作为 "包含文件 "添加到GameMaker IDE 的dll或dylib。它不会对那些作为extensions 添加的文件起作用，因为那些文件需要你在extension 包中自己定义函数。

外部调用类型常量
恒定 描述
dll_cdecl 这是默认的C、C++调用
dll_stdcall 这是标准的WinAPI调用(Windows dll only)
外部数据类型常量
恒定 描述
ty_real 一个实数参数
ty_string 一个空尾的string 参数

语法。

external_define(dll, name, calltype, restype, argnumb, argtype[0], argtype[1], ...argtype[10]);

争论 类型 描述
dll 字符串 The name of the DLL file (string)
name 字符串 The name of the function (string)
calltype 外部调用类型常量（dll_*）。 The calling convention used
restype 外部数据类型常数（ty_*）。 The type of the result to expect
argnumb 真实的 The number of arguments (0 - 10)
argtype[0 ... 10] 外部数据类型常数（ty_*）。 The types of the arguments being used

返回。

External Function

例子。

my_funcion = external_define("MyDLL.dll", "MyMin", dll_cdecl, ty_real, 2, ty_real, ty_real);

上面的示例代码将定义一个名为 "MyMin "的外部函数，有两个参数。
