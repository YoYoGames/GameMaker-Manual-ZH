暂停使用

当为移动设备开发时，你需要有一种方法来检查设备是否仍在关注你的游戏，特别是如果你的游戏是在iOS,Android 或Windows Phone 设备上，那么它可能会被电话和其他应用程序打断。你可以使用这个功能来检查设备是否暂停了。

值得注意的是，这个函数将在设备上运行包含在代码块中的任何代码，但随后GameMaker将立即暂停，因为设备caches ，游戏作为一个后台进程。因此，如果你有像下面提供的示例代码一样的东西，要创建的实例将不会实际出现或运行它的任何事件，直到游戏从缓存状态重新启动之后。这意味着你可能更喜欢让 os_is_paused 函数调用一个警报，然后在警报事件中（该事件将在设备暂停时被设置，但在解除暂停前不会运行）创建一个暂停菜单，重新启动音乐等。

如果你在Windows 、Ubuntu (Linux)、MacOS或HTML5平台上使用这个功能，那么这个功能只在游戏失去焦点时切换一步到 true ，然后它将再次返回 false ，直到游戏重新成为焦点（并再次失去焦点）。这是因为这些平台没有cache 应用程序和游戏，因此这些将继续在后台运行。

注意：在移动设备上，当操作系统暂停时，声音和音乐将被停止。除非你用适当的功能专门启动它们，否则它们不会再重新启动。

语法。

os_is_paused()

返回。

例子。

if os_is_paused()
{
    if !instance_exists(obj_PauseMenu)
    {
        instance_create_layer(0, 0, "Controllers", obj_PauseMenu);
    }
}

上面的代码检查游戏是否已经暂停，如果已经暂停，那么它将创建一个object "obj_PauseMenu "的实例，如果还没有存在。
