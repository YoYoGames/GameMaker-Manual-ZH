os_check_permission

通过这个函数，你可以检查用户是否已经授予了游戏一个特定的权限。你提供要检查的权限为string ，使用格式为 "android.permission.<permission>"，所以要检查 RECORD_AUDIO 的权限（例如），你需要调用

os_check_permission("android.permission.RECORD_AUDIO");

该函数将返回将返回三个常量中的一个--如下图所示--以告诉游戏权限的状态如何。关于应用程序权限的更多信息，请参见Android文档。

重要的是!该功能仅适用于安卓目标。

语法。

os_check_permission(permission)

争论 类型 描述
permission 字符串 The permission to check (a string)

返回。

Permission State Constant

许可状态常数
恒定 描述
os_permission_granted 这表明许可已被授予
os_permission_denied 这表明许可尚未被授予
os_permission_denied_dont_request 这表明该权限已被手机设置阻止，或者用户之前拒绝了该请求并选择了 "不要再问"。

例子。

if (os_type == os_android)
{
    if (os_check_permission("android.permission.INTERNET") == os_permission_denied)
    {
        os_request_permission("android.permission.INTERNET");
    }
}

上述代码检查操作系统的类型，如果是Android ，它就对权限进行检查，如果 "INTERNET "权限还没有被授予，它就请求它。
