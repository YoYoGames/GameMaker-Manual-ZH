os_check_permission

通过这个函数你可以请求一个特定的权限。string 你可以使用格式 "android.permission.<permission>" 提供要请求的权限，所以要请求 RECORD_AUDIO 权限（例如），你可以调用

os_request_permission("android.permission.RECORD_AUDIO");

该函数将返回触发一个系统异步事件，其中内置的 async_loadDS地图将包含以下键/值对。

  • "类型"。这是被触发的事件类型，其值将是string "permission_request_result"。
  • "<permission>"。这将函数请求的权限作为string ，其中包含以下常数之一。
恒定 描述
os_permission_granted 这表明许可已经被授予
os_permission_denied 这表明许可尚未被授予

值得注意的是，GameMaker原生支持以下权限，但被Google认为是 "危险 "的，因此必须明确申请（也请注意，有些权限可以使用Android游戏选项申请，而不需要这个功能）。

  • android.permission.WRITE_EXTERNAL_STORAGE
  • android.permission.READ_PHONE_STATE
  • android.permission.RECORD_AUDIO

关于应用程序权限的更多信息，请参见Android文档。

重要的是!该功能仅适用于安卓目标。

语法。

os_request_permission(permission)

争论 类型 描述
permission 字符串 The permission to request (a string)

返回。

N/A

例子。

if (os_type == os_android)
{
    if (os_check_permission("android.permission.INTERNET") == os_permission_denied)
    {
        os_request_permission("android.permission.INTERNET");
    }
}

上述代码检查操作系统的类型，如果是Android ，它就对权限进行检查，如果 "INTERNET "权限还没有被授予，它就请求它。
