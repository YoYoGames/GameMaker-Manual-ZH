gml_release_mode

当你的游戏被编译时，它还包括某些内部函数，在运行时进行错误检查。这些内部检查涵盖了游戏代码的许多方面，包括范围、参数和内部GML 编译器的一些一般方面，并且是 在许多情况下，是什么催生了最终的错误信息。switch 然而，这个函数可以用来关闭这些内部检查（即：从编译的游戏中完全删除它们），给你的最终项目带来速度提升。

请注意，这意味着如果你在游戏代码中做了一些不正确的事情，编译后的游戏可能会直接崩溃，或者会出现一些非常意外的行为。例如，如果一个 DS Grid ，在一个 坐标超出了定义的大小，在开发模式下，你会得到一个编译器窗口错误信息，但现有的检查会允许游戏继续运行。但是在发布模式下，这些检查就不存在了，所以代码可能会 返回一个NULL指针，或者一个坏指针，而坏指针又会指向一些随机的内存。所以在最好的情况下，你可能会得到错误的数据返回，或者在最坏的情况下，你可能会覆盖内存中的东西--数据、图形，甚至是代码本身。另一个例子 是在构建基元时--如果你提供的颜色、位置、texture 等数据的顺序与你在创建vertex 格式时定义的顺序不同，那么在释放模式下，游戏可能会崩溃或产生意想不到的结果 时，游戏可能会崩溃或产生意想不到的结果。

语法。

gml_release_mode(flag);

争论类型 描述
flag Either true 来设置发布模式，或为一般开发设置false。

返回。

例子。

gml_release_mode(true);

上面的示例代码将在编译时将游戏设置为使用释放模式。
