collision_rectangle

Collision_rectangle 使用前四个参数（x1,y1,x2,y2）来定义当前 内的一个区域，然后检查由 "obj "参数定义的room 任何 object 是否与该区域发生碰撞。这种碰撞可以被检查为精确或不精确，你也可以选择检查运行代码的实例本身或不检查。考虑一下这个图片。

这里，中间的实例使用了一个碰撞矩形来检查球objects 。现在，蓝色的没有精确的边界框，正如你所看到的，即使sprite 实际上没有接触到矩形，碰撞仍然会发生（即使你将函数中的精确选项设置为真），因为该sprite 的边界框与collision_rectangle重叠。另一方面，只有当实际的sprite 与矩形重叠时，绿球才会被视为发生碰撞。记住，要考虑精确碰撞，object sprite 和碰撞函数都必须将精确标记为开。还应该注意的是，该函数的返回值可以是任何一个被视为碰撞的实例的ID。

注意，你可以提供一个实例来代替object 索引，以检查特定的实例。 id来检查一个特定的实例，或者实例关键字 all ，或者 other （取决于事件和当前的代码范围）。

语法。

collision_rectangle(x1, y1, x2, y2, obj, prec, notme);

争论 类型 描述
x1 真实的 The x coordinate of the left side of the rectangle to check.
y1 真实的 The y coordinate of the top side of the rectangle to check.
x2 真实的 The x coordinate of the right side of the rectangle to check.
y2 真实的 The y coordinate of the bottom side of the rectangle to check.
obj 对象资产或实例ID The object to check for instance collisions.
prec 布尔型 Whether the check is based on precise collisions (true，速度较慢）或其一般的边界盒（false ，速度较快）。
notme 布尔型 Whether the calling instance, if relevant, should be excluded (true)或不(false)。

返回。

Instance ID or noone

例子。

var inst;
inst = collision_rectangle(50, 50, 200, 100, obj_Ball, false, true);
if (inst != noone)
{
    instance_destroy(inst);
}

这段简短的代码使用collision_rectangle在room ，从50x，50y（矩形的左上角）到200x，200y（矩形的右下角）检查一个叫做 "obj_ball "的object 。它将返回值存储在一个临时变量中，然后检查该值是否是一个实例ID，或者是关键字 没有人.如果不是 没有，那么它就使用存储的实例ID来销毁object 。

返回。碰撞
