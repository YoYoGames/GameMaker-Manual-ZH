collision_point

Collision_point 检查参数x1,y1指定的点是否与参数 "obj "指定的 的object 任何实例发生碰撞。这种检查可以是精确的，也可以是不精确的，但是要启用精确碰撞，你要检查的 或实例也object 必须为其 启用精确碰撞。如果没有，默认检查是基于边界盒的。下面的图片说明了这是如何工作的。sprite

记住，要考虑精确碰撞，object sprite 和碰撞函数都必须将精确标记为开启。还应该注意的是，函数的返回值可以是任何一个被认为是碰撞的实例的id，所以如果三个实例在这一点上重叠，它们中的任何一个id都可以成为函数的返回值。

注意，你可以提供一个实例来代替object 索引，以检查特定的实例。 id来检查一个特定的实例，或者实例关键字 all ，或者 other （取决于事件和当前的代码范围）。

语法。

collision_point(x, y, obj, prec, notme);

争论 类型 描述
x 真实的 The x coordinate of the point to check.
y 真实的 The y coordinate of the point to check.
obj 对象资产或实例ID The object to check for instance collisions.
prec 布尔型 Whether the check is based on precise collisions (true，速度较慢）或其一般的边界盒（false ，速度较快）。
notme 布尔型 Whether the calling instance, if relevant, should be excluded (true)或不(false)。

返回。

Instance ID or noone

例子。

if collision_point(x, y, obj_Cursor, false, true)
{
    score += 10S;
}

这里我们要检查的是object 的位置上的点，该点的代码为object "obj_Cursor"。如果有一个，那么我们就在分数变量上加10。

返回。碰撞
