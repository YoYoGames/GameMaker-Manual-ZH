collision_ellipse

Collision_ellipse 使用前四个参数（x1,y1,x2,y2）来定义当前 内的椭圆的宽度和高度，然后检查由 "obj "参数定义的room 任何 object 是否与该区域发生碰撞。这种碰撞可以被检查为精确或不精确，你也可以选择对运行代码的实例本身进行检查或不检查。考虑一下这个图片。

这里，中间的实例是使用碰撞椭圆来检查球objects 。现在，蓝色的没有精确的边界框，正如你所看到的，即使sprite 实际上没有接触到椭圆，碰撞仍然会发生（即使你把函数中的精确选项设置为真），因为该sprite 的边界框超过了 collision_circle 所定义的椭圆区域。另一方面，只有当实际的sprite 与定义的椭圆重叠时，绿球才会被认为发生碰撞。记住，要考虑精确碰撞，object sprite 和碰撞函数都必须将精确标记为开。还应该注意的是，该函数的返回值可以是任何一个被视为碰撞的实例的ID。

注意，你可以提供一个实例来代替object 索引，以检查特定的实例。 id来检查一个特定的实例，或者实例关键字 all ，或者 other （取决于事件和当前的代码范围）。

语法。

collision_ellipse(x1, y1, x2, y2, obj, prec, notme);

争论 类型 描述
x1 真实的 The x coordinate of the left side of the ellipse to check.
y1 真实的 The y coordinate of the top side of the ellipse to check.
x2 真实的 The x coordinate of the right side of the ellipse to check.
y2 真实的 The y coordinate of the bottom side of the ellipse to check.
obj 对象资产或实例ID The object to check for instance collisions.
prec 布尔型 Whether the check is based on precise collisions (true，速度较慢）或其一般的边界盒（false ，速度较快）。
notme 布尔型 Whether the calling instance, if relevant, should be excluded (true)或不(false)。

返回。

Instance ID or noone

例子。

if collision_ellipse(50, 50, 200, 100, obj_Player, false, true)
{
    instance_create_layer(obj_Player.x, obj_Player.y, "Effects", obj_Splash);
}

这将检查object "obj_Player "的50x, 50y和200x, 100y范围内的一个椭圆区域。如果与该object ，那么它将在obj_Player的x/y坐标处创建一个 "obj_Splash "实例。

返回。碰撞
