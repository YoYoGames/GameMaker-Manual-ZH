移动_bounce_all

有了这个功能，你可以命令一个实例在room，唯一的例外是那些没有分配给它们的sprite 或掩码索引的实例。你也可以告诉它在启用时使用精确的碰撞检查，但要注意这需要所有的实例都启用精确的掩码，而且当涉及到许多实例时，由于必须进行大量的处理，将大大降低你的游戏速度。这通常应该在实例的步骤事件中进行，但也可以有选择地在碰撞事件中使用，正如下面的代码例子所说明的。

语法。

move_bounce_all( adv );

争论类型 描述
adv Whether to enable advanced bouncing (true) or not (false).

返回。

例子。

if other.visible
{
    move_bounce_all(false);
}

上述代码将被放在与另一个object 的碰撞事件中。它首先检查object 是否可见，如果是，它就执行 move_bounce_all() 的动作。请注意，在这种情况下，反弹是有选择的，只对这次碰撞进行计算，而不是对每个实例、每个步骤进行计算。
