mp_grid_add_cell

有了这个功能，我们可以把给定的MP网格内的个别单元标记为 "禁止"，这意味着path ，查找功能将永远不会穿过它们。现在，通常你是在X/Y房间坐标下工作，而不是在MP网格单元坐标下工作。因此，让我们考虑下面的图片，它显示了两者之间的关联性。

正如你所看到的，mp_grid被编号为一系列的单元格（从左上角的0,0开始），当你把这个网格放在room ，room 坐标（也是从左上角的0,0开始）和网格内的单元格之间有直接的关联。在图片中，你可以看到位置为260,130的实例落在网格单元8,4上。你可以通过数学方法计算出一个位置所处的准确单元格，方法是用数字除以网格分辨率，然后向下取整；如果你使用的是2次方的网格分辨率（比如8、16、32等），你可以使用位操作符。应该注意的是，位操作的速度更快，下面的例子显示了两种方法。

语法。

mp_grid_add_cell(id, h, v);

争论 类型 描述
id MP网格ID Index of the mp_grid that is to be used
h 真实的 Horizontal position of the cell in the grid
v 真实的 Vertical position of the cell in the grid

返回。

N/A

例子。

with (obj_Wall)
{
    mp_grid_add_cell(grid, floor(x / 32), floor(y / 32));
}
with (obj_Door)
{
    mp_grid_add_cell(grid, x>>5, y>>5);
}

上述代码将使所有的 "obj_Wall "和所有的obj_Door将它们的等效单元位置添加到变量 "grid "所索引的mp_grid中。在例子的第一部分，这是通过取object 的x/y坐标并除以网格的分辨率来实现的（使用 floor ，以保持数值为整数），而在第二部分（因为网格的分辨率是2的幂）是通过将x,y坐标向右移5位来实现的。
