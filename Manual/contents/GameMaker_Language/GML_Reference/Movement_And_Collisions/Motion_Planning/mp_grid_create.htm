mp_grid_create

通过这个函数，你可以为运动规划函数创建一个mp_grid。它返回一个索引，该索引应存储在一个变量中，以便在所有其他的MP网格函数调用中使用。x和y坐标表示网格左上角的位置，hcells和vcells表示网格内要创建的水平和垂直单元的数量，单元宽度和单元高度表示单元的尺寸，单位为像素。如果你需要它们做不同的事情，你可以在同一时刻创建和维护多个网格结构，但要注意每个网格都需要内存，它们的单元格越多，需要的内存就越多。还要注意的是，MP函数的使用是相当重的处理器，所以你调用它们的频率越高，单元格的分辨率越高，那么你就越有可能出现性能下降的情况。

语法。

mp_grid_create(left, top, hcells, vcells, cellwidth, cellheight);

争论 类型 描述
left 真实的 Starting x coordinate of the mp_grid in the room
top 真实的 Starting y coordinate of the mp_grid in the room
hcells 真实的 Number of horizontal cells that the mp_grid will contain
vcells 真实的 Number of vertical cells that the mp_grid will contain
cellwidth 真实的 The width (in pixels) of each individual cell of the mp_grid
cellheight 真实的 The height (in pixels) of each individual cell of the mp_grid

返回。

MP Grid ID

例子。

global.grid = mp_grid_create(0, 0, room_width / 32, room_height / 32, 32, 32);

上述代码创建了一个全局变量 "grid"，然后生成了一个mp_grid，并将其索引（id）分配给该变量，用于所有进一步的mp_grid函数调用。该网格覆盖了room ，单元格的分辨率为32x32像素。这意味着，例如，在一个640x480的room ，网格将包含300个单元：20个水平单元（640/32）乘以15个垂直单元（480/32）得到300个单元
