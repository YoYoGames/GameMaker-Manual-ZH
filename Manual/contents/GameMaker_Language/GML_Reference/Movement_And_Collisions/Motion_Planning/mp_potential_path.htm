mp_potential_path

这个函数为实例计算一个path ，从它的当前位置和方向到由xgoal,ygoal参数指定的位置。它使用指定的步长，同时试图避免与所有实例或只与标记为实体的实例发生碰撞。 那些被标记为实体的实例。就像在函数 mp_potential_step()中一样，它使用势场步长来实现这一目标，它的方式可以通过函数来修改 mp_potential_settings(). 指示的path 必须已经存在，它将被新的path 所覆盖。该函数将返回是否找到了path （真）或不（假）。

为了避免函数永远继续计算，你需要提供一个大于1的系数--如果函数不能找到一个比这个系数乘以起点和目标之间的距离更短的path ，它将停止并报告失败。一个系数为4 通常就足够了，但如果你预计会有很长的弯路，你可以把它加长。如果它失败了，仍然会创建一个path ，沿着目标的大致方向运行，但它不会到达目标。

注意：这个函数不会移动实例。它只设置一个path ，你必须使用路径函数来移动。

语法。

mp_potential_path(path, xgoal, ygoal, stepsize, factor, checkall)

争论类型 描述
path index of the path that is to be used by the function
xgoal The target x position.
ygoal The target y position.
stepsize The speed the instance moves in pixels per step.
factor This number is used to prevent an infinite loop. It should be larger than 1. See the function description for more information.
checkall Whether to check all instances (true) or just solid ones (false).

返回。

例子。

path = path_add();
mp_potential_path(path, obj_Player.x, obj_Player.y, 3, 4, 0);
path_start(path, 3, 0, 0);

上述代码创建了一个新的path 资源，并将其索引存储在变量 "path "中。然后，它使用 mp_potential_path ，在这个代码的path 和object "obj_Player "之间生成一个object ，检查是否与 中的所有实例发生碰撞。 room 最后，它沿着该object ，即使该函数未能返回到 "obj_Player "的完整路线，也会启动path 。
