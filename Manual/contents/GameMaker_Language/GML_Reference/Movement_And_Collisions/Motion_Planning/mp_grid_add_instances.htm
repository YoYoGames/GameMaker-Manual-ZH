mp_grid_add_instances

这个函数使用实例碰撞掩码（由sprite 属性或调用实例的掩码索引决定）来标记MP网格中的禁区或非禁区单元，其中禁区单元不能被任何寻路函数穿过。你可以在函数中指定是否考虑精确碰撞，被标记的禁区单元将根据这一设置而改变。这张图片说明了这种行为。

上面的两个实例已经被添加到MP网格中，使用 mp_grid_add_instances() 函数，精确设置为真。正如你所看到的，绿色实例只把它 "接触 "的网格方格标记为禁止，这是因为它的sprite 掩码也被设置为精确。然而，第二个实例也标记了其他单元格，而这些单元格看起来并没有被触及。这是因为sprite 掩码没有设置为精确，这意味着即使你在函数中使用精确设置，也只考虑sprite 的边界盒。这也是如果你把函数的精确参数设置为false会发生的情况--即使有精确的sprite 掩码的实例也会根据它们的边界框被添加到网格中。

语法。

mp_grid_add_instances(id, obj, prec);

争论 类型 描述
id MP网格ID Index of the mp_grid that is to be used
obj 对象资产或实例ID Object index, or instance id, of the instances to be added into the mp_grid
prec 布尔型 Whether the check is based on pixel-perfect collisions (true = slow) or its bounding box in general (false = fast).

返回。

N/A

例子。

mp_grid_add_instances(grid, obj_Wall, true);

上述代码将把 "obj_Wall "的所有实例添加到变量 "grid "中索引的mp_grid中，使用精确的碰撞掩码而不是边界框。
