活动顺序

在考虑GameMaker中的事件时，应该注意的是，不能明确说明所有事件在每一步中发生的确切顺序，这只是因为它取决于GameMaker的内部运作，而且随着软件的发展，这可能会改变。然而，有一些事件总是 以相同的顺序运行。

第一组总是以相同方式发生的事件是那些在第一次进入room 。不同事件发生的顺序是。

  • 对象变量/实例变量被初始化（这些是你可以在对象编辑器或房间编辑器的IDE中定义的变量）。 如果你有一个附加有变量的object ，那么它们将在其他东西之前被创建，然后任何附加有变量的实例将在之后被定义，所以你可以在房间里用特定的实例覆盖object 变量。 编辑。
  • 创建 事件，然后是每个 实例的实例 创建代码 --所以当每个实例被创建时，它将首先运行其创建事件，然后是其实例创建代码 然后再转到下一个要创建的实例。
  • 游戏开始事件- 这个事件在游戏的第一个room ，对于从room 编辑器中放置在room 的所有实例都会被触发一次（注意，调用game_restart()将再次触发这个事件。
  • 房间创建代码- 这是在Room 编辑器中写的一次性代码，用于首次进入room 。
  • 所有实例的房间开始事件- "其他 "类别的事件之一，将对所有实例发射，无论是否持久存在

值得注意的是，你也可以在room 编辑器本身中设置特定实例的创建顺序，方法是在实例层属性窗口的列表中向上或向下移动它们。

除了这些特定的事件之外，唯一已知的顺序是无论如何都会以相同的方式发生的，属于三步 事件和不同的抽签 事件。这些事件将始终保持一致，所以如果 你的代码在游戏的每一步都依赖于特定的时间，你应该使用。

  • 开始步骤事件
  • 步骤事件（注意，步骤事件是在实例被放到新位置之前执行的）。
  • 结束步骤事件

绘图的所有事件也总是按照以下相同的顺序处理（除了窗口调整大小 事件，它的触发方式不同）。

  • 抽签前活动
  • 绘制开始事件
  • 抽奖活动
  • 绘制结束事件
  • 抽签后的活动
  • 绘制GUI开始事件
  • 绘制GUI事件
  • 绘制GUI结束事件
