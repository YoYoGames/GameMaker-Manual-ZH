系统

这个事件只能由系统级事件的回调触发（比如检测游戏手柄或自动登录XBox Live），它将返回一个DS Map，存储在变量 async_load中，包含不同的键/值对，取决于触发回调的系统级事件。这些事件是

游戏垫

当这个事件被触发时，游戏手柄被连接或断开，它将在 async_load 地图中返回以下键/值对之一。

  • "event_type" - 收到的系统事件的类型，它将是以下字符串之一。
    • "gamepad discovered" - 当系统报告有一个新的游戏板被连接时发生。
    • "gamepad lost" - 当系统失去与一个游戏板的连接时发生。
  • "pad_index" - 被添加或删除的焊盘的索引。

这个事件现在允许你把所有的游戏手柄检查逻辑从步骤事件或报警事件中移到系统事件中，并且只在实际需要时运行。

重要的是!这个事件不会被触发，除非你在游戏代码中至少有一个 gamepad_*函数或GML Visual动作。只有在项目中使用函数时，运行器才会初始化游戏板子系统，所以如果不存在这些函数，添加/删除游戏板将不会触发系统事件。

虚拟键盘

当虚拟键盘被打开或关闭时触发该事件，它将在 async_load map中返回以下键/值对。

  • "event_type" - 收到的系统事件的类型，对于虚拟键盘来说将是 "virtual keyboard status"。
  • "screen_height" - 虚拟键盘的高度（像素）。如果键盘是不可见的，这将是0。
  • "keyboard_status" - 键盘的当前状态，以下列字符串之一返回。
    • "隐藏"
    • "隐藏"
    • "显示"
    • "可见"

淘宝网

当使用UWP导出目标Xbox One并在UWP游戏选项中勾选启用Xbox Live选项时，可以触发异步系统事件。当你启动启用了Xbox Live的GameMakerUWP项目时，该项目将自动尝试默默地登录Xbox Live。这个登录尝试的结果将作为以下键/值对之一返回到 async_load map中。

  • "event_type" - 收到的系统事件的类型，它将是以下字符串之一。
  • "user signed in" - 无声用户登录已成功完成
  • "user sign in failed" - 沉默的用户登录失败（当这种情况发生时，你可以使用函数 xboxlive_show_account_picker()来让用户选择一个账户进行登录)
  • "user signed out"--用户已经签出。

Xbox Live的其他功能也可能触发这个事件，但这些功能的不同回调在相关的功能页面上有详细说明。关于Xbox Live的具体功能的更多信息，请看这里。

音频系统状态

这个事件类型是专门为使用HTML5目标时使用的，因为每次网络音频上下文状态改变时，它都会被触发。这意味着，如果你有，例如，一段循环的背景音乐，那么你可以根据这个事件的触发，暂停它或停止并重新启动它。这可以通过在 async_load 地图中寻找以下键/值对来检查。

  • "event_type" - 收到的系统事件的类型，如果音频系统已经初始化或上下文已经改变，将是string "audio_system_status" 。

如果这个事件类型被触发，那么在 async_load 地图中会有一个额外的键。

  • "status" - 音频系统的状态，它将是以下两个字符串之一
  • "available" - 音频系统已被初始化，可以播放声音。
  • "unavailable" - 音频系统没有被初始化，或者当前环境没有运行，所以不能播放声音（所有的声音播放函数都将返回-1）。

关于音频的具体功能的更多信息，请看这里。

注意：虽然这个事件是为HTML5目标设计的，但它也会在所有其他平台上被触发，但在所有平台上（除了HTML5），它只会在游戏开始时在音频引擎首次初始化时被触发一次。
