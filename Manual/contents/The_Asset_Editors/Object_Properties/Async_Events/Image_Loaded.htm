图片加载完毕

当你在HTML5上或通过URL将图片加载到GameMaker中时，这个事件会被触发。例如，假设你想加载一个sprite ，并且只在加载后将该实例的当前sprite 改为新的。你可以在创建事件（或任何其他事件）中使用类似的东西。

var _url = "http://www.angusgames.com/game/background1.png";
spr = sprite_add(_url, 0, false, false, 0, 0);

现在，这将开始将图像加载到设备或浏览器中，但它不会在等待文件加载时阻止GameMaker。相反，GameMaker将继续正常运行，直到图像被加载，回调触发了图像加载事件，在那里，一个DS Map被创建并存储在特殊变量中 async_load.该地图包含以下信息。

  • "filename"。你所要求的文件的完整path 。
  • "id"。你所加载的资源的ID。这将与你分配给该资源的变量相同。
  • "status"。返回一个小于0的错误值。

然后，你将在这个事件中把新加载的图像分配给一个sprite 。下面的代码例子演示了如何使用返回的信息。

if ds_map_find_value(async_load, "id") == spr
{
    if ds_map_find_value(async_load, "status") >= 0
    {
        sprite_index = spr;
    }
}

上述代码将首先检查已经创建的DS地图的 id ，然后检查回调的 status 。如果该值大于或等于0（表示成功），那么回调的结果将被用来设置新加载图像的。 回调的结果将被用来设置sprite 索引到新加载的图像。
