TheTile Set 编辑

瓦片组是一种图形资源，用于绘制关卡和游戏的其他静态组件。一个tile set 是由一个单一的图像组成的，然后被分割成不同的 "单元"(tiles)，然后每个tile 可以被放入room 编辑器以创建一个完整的图像。下面你可以看到两个可以作为瓷砖组的例子sprites 。

图片来源：Lost Garden

这些是人们创建tile sets 最常见的方式--要么全部塞在一起，要么用小空间把它们隔开，GameMaker可以处理这两种类型。基本上，所有的tile sets ，都必须由符合明确定义的网格的图像组成，网格的每个单元都是完全相同的大小（注意，虽然上面显示的图像都是由方形tiles ，但tile set ，也可以由矩形tiles ）。

tile sets 当创建你的tile ，请记住，左上角的网格单元必须始终 是空的，因为这是GameMaker将用于tiles "空 "room ，并用于擦除现有的tiles （即使你在图像的那一部分有像素，它们也会被忽略）。

当你第一次创建一个tile set 资源时，Tile Set 编辑窗口将打开，有以下选项。

命名

在这里你可以给你的tile set 。名称只能是字母数字，只能使用下栏"_"符号，而且不能以数字开头。

雪碧

在这里你可以选择精灵 asset ，并从中创建我们的tile set 。点击主按钮将打开Asset Explorer，在这里你可以选择你希望用于瓦片组的sprite 。

你也可以点击新精灵按钮 ，创建一个新的sprite asset （这将在asset ，为新的workspace ，打开精灵编辑器进行编辑），或者你可以使用编辑精灵按钮 ，编辑当前选择的sprite ，甚至可以选择在图像编辑器中按 ，编辑sprite 图像本身。

默认情况下，当你打开一个tile set ，瓷砖组属性窗口也会显示出来（见下图），但如果你关闭了它，那么你可以点击这里的按钮，再次重新打开它。

财产

瓦片集属性窗口用于指定基本的tile 尺寸（默认为 16px 正方形）、偏移量和tiles 之间的间隔（后两个选项用于sprites ，如本页前面的示例图像中所示，其中tiles 的各个部分之间有空隙）。你还可以为tile set 图像设置输出边框值，它可以控制图像在磁盘上的存储方式，并根据需要在每个tile 周围添加额外的像素。

由于tiles 渲染的方式，它们需要事先准备好，在每个人周围有一个边框来 "涂抹 "边缘。下面显示的图片说明了tiles 一般是如何被设备图形驱动程序映射的，tile 周围的红线显示了U,V坐标被映射到哪里（字母 "U "和 "V "被用来描述2Dtexture 网格，因为 "X"、"Y "和 "Z "已经用来描述room 和object 空间坐标）。

正如你所看到的，当在room 。现在在1:1的比例下，这一切都很好，但是当我们缩小这个tile ，会发生什么？缩小比例的问题是，当你缩小图像时，UV也会缩小，并试图适应一个更压缩的空间，但这显然是不可能的，所以图形驱动会根据可用的像素中心来选择要绘制的texel。

所以，如果我们 "只是 "移动到一个新的屏幕像素，那么，因为图形驱动使用像素的中心来决定用什么来画，它会选择一个围绕tile ，而不是tile 本身的一个文本。

上面显示的是一个tile ，它被缩减为6个屏幕像素，这表明当你重叠到下一个屏幕像素时，图形驱动必须做出选择，要么放弃这个texel，要么尝试用它认为有效的texel来填充它。让我们说，在这种情况下，它决定将重叠的texel画到屏幕像素中，而且，由于它现在明显在我们的tile ，它从下一个tile ，或者你在tiles ，这意味着，当它缩小时，你会在tiles ，在它们的绘制方式中得到裂缝，这就是很多混乱的地方...每个图形驱动程序和设备实际上都以自己的方式来做这件事，这意味着结果可能是不可预测的，看起来很糟糕......但如果你小心并提前计划，你可以绕过它。

在高端游戏中，如果你从一个texture （不是在texture 页面或texture 图集上），你可以选择使用一种叫做clamp的 texture 模式，它的作用是将最后一排的texels无限地重复到远处，让你可以毫无问题地缩放，因为它迫使硬件无论如何都要获得最后一排的像素。这就是输出边框 设置为你做的事情。

上面显示的是一个 "固定的 "tile sprite ，你可以看到它现在在每个tile 周围有一个重复的部分，这意味着当硬件超限时，它将总是从tile ，而不是其中一个空间或之后的tile ，挑选最后一个文本。因此，在创建tile set ，请记住，如果你要放大或缩小游戏视图，或者要按比例渲染tiles ，你可能需要有一个比默认值2更高的输出边界宽度或高度。

接下来，你可以选择你希望瓷砖组资源所属的纹理组。基本上，texture 组（以前在纹理组管理器中定义）是你可以设置的东西，这样你需要的所有图像assets ，为你的游戏的特定rooms 或水平可以存储在一起。因此，举例来说，你可以把所有的第1级图像放在一个texture 组中，把所有的第2级图像放在另一个组中，等等......GameMaker会尝试把所有这些分组的资产放在同一个纹理页面上，以减少你的游戏在选定的目标平台上运行时的texture 页面交换现象。

注意：这可能并不总是必要的，这种方法的性能提升将取决于目标设备是CPU绑定还是GPU绑定。

tile set 属性中的最后一个选项是禁用源Sprite 输出。这个选项在默认情况下被选中，它的作用是flags ，在为你的项目生成sprite 页面时，省略源asset texture 。sprite 这样做的前提是，作为tile set ，不会被用在其他地方，所以只需要导出一次。然而，如果你将基本的sprite 资源用于除tiles 之外的其他用途，那么你会想取消这个选项，以便在编译时将基本的sprite 和 tile set 都添加到texture 页面中。

编辑部

编辑器 部分的每个按钮将打开一个不同的编辑器，用于特定的tile set 功能。你可以根据多个tile set 一起创建tiles 刷子 ，动画瓦片，使用不同的tiles 作为单独的框架，你也可以创建自动瓦片，根据它们周围的tiles "连接 "在一起。这些编辑器中的每一个都会在本页末尾链接的章节中详细解释。

预览工具箱

这些是你可以用来预览sprite ，为tile set 。这些按钮控制了当前预览的缩放级别，你可以用它们来放大或缩小，点击 按钮将重置预览，使其与正在编辑的tile set sprite 1：1。你也可以点击窗口适合按钮 ，使整个tile set 适合当前预览窗口（这将酌情放大/缩小，使其适合）。请注意，你也可以使用 / 和鼠标滚轮 来放大和缩小，按 / + 将把画布设置为与tile set sprite 1:1。

switch 另一个特点是，你可以通过点击 按钮打开或关闭网格视图，以及设置要使用的网格颜色（点击 按钮）。当网格被激活时，它将为网格中的每一个tiles ，尊重分离等的设置，显示一个彩色的轮廓。

预览

这一部分是tile set 预览窗口。要在预览中导航，你可以使用与一般Workspaces ，即：使用鼠标中键 ，然后拖动来平移预览（或者使用 + 组合），你可以用鼠标滚轮 水平滚动，或者使用 / 键和鼠标滚轮 放大和缩小。你还可以使用预览工具箱中的不同按钮来改变缩放，并启用/禁用网格单元的覆盖。
