TheExtension 编辑

扩展包可以用来为GameMaker Language (GML)添加一些额外的函数和常量，并在项目之间轻松转移。Extension 函数也可以链接到外部SDK或用其他语言编写的源代码（如Java用于Android ，Objective-C用于iOS ，等等），可以用来扩展你的游戏功能（通常用于实现第三方API，例如用于广告、应用内购买等等）。

扩展可以用来创建一个你在所有项目中经常使用的函数库，所以你不必在每个项目中都导入相同的scripts ，你可以简单地添加一个extension asset ，其中包含你最喜欢的函数和常数的集合。这样的函数也可以是特定于某个平台的，你可以很容易地选择一个extension 可以在哪些平台上使用。

导入扩展程序

通常第三方extensions ，将来自GameMaker市场，然后你订阅的那些将在市场图书馆窗口中列出，可以从那里安装。extension 如果你没有从Marketplace ，你仍然可以通过右键单击资产浏览器中任何地方的 ，选择添加现有的，然后浏览到扩展的位置来选择它，把它添加到项目中。

注意如果你有一个 .yymps/.yymp/.gmez 包，你可以把它拖到IDE 来导入，或者到 "工具 "菜单下选择 "导入本地包"。

目标平台

一旦你在extension 浏览器中添加了一个Asset ，你应该给它一个名字和一个版本号，还应该设置你想把它导出到哪些平台。通过取消选中的平台，你可以告诉GameMaker不把extension 作为这些平台的可执行文件的一部分导出。你还应该确保extension 的属性与选定的目标平台相匹配，并根据以下情况为extension 文件选择兼容的导出目标。

  • 所有目标：GML extensions 。这些是使用extensions (GameMaker Language)制作的GML ，与所有目标平台兼容。(注意，这是Opera GX目标目前唯一支持的extension 类型。)
  • Windows（标准、UWP、Xbox）。你可以使用Windows DLL 文件（动态链接库）。请注意，如果你在Windows游戏选项中勾选了 "使用x64Windows Runtime"，那么项目的所有DLL必须是64位兼容的。32位的DLLs将不能与64位的可执行文件一起工作。
  • macOS：macOS 目标可以使用标准的 DyLib 文件，用于extensions 。
  • Ubuntu (Linux)。对于Ubuntu 目标，你需要确保你的extension 使用 .so 文件。
  • HTML5。这些平台可以使用 JavaScript extensions 。
  • 安卓。这个平台要求用 Java ，或用 Android Framework SDK ，写出一个extension 。
  • iOS/ tvOS：该平台的Extensions 是用Objective C编写的，需要一个*.mm 和一个*.h 文件，如果使用任何框架，则需要一个预编译的本地*.a 文件。
  • PlayStation。对于PlayStation，你的extensions 应该是 PRX 文件，每个目标都需要不同的 PRX ，以符合目标的规格（PS3、PS4或PS5）。如果在PSVita上工作，extension ，而不是 SUPRX 。

注意 extensions 一些GameMaker ，需要遵循特定的命名惯例，以正确识别它们所设计的平台和CPU架构。更多信息请参见命名规则。

由于一个extension ，可以有各种相关的文件，有可能为多个平台创建一个extension ，在这种情况下，你要在主extension 编辑器中指定所有的 兼容平台（即：勾选旁边的复选框），然后在extension 的每个部分的个别文件属性中，设置这些文件应该被导出的平台。

要改变特定的extension 文件或占位符的目标平台，你必须首先打开extension 属性（在extension 浏览器中双击 ，Asset ），然后在extension 属性中从右边选择你要编辑的资源文件。在打开的窗口中，你可以选择该特定文件所导出的平台。

扩展选项

你所使用的extension 可能有选项，这些选项将显示在Extension 编辑器窗口的底部。

这些将是不同的，取决于extension ，而且不是所有的都有选项。请查阅你所使用的extension 的文档，了解其选项信息。

建立时解压

"构建时解压 "选项可用于针对特定目标平台的压缩档案文件。如果这个选项被启用，该归档文件的内容将被解压缩到已建软件包的Included Files目录中，但只有在被复制到目标平台之后。

例如，如果你为你的extension 中的一个 .zip 文件启用了这个功能，然后从Windows 开始构建macOS ，那么 .zip 归档文件将从Windows 复制到macOS ，然后在目标平台（在这里是指macOS ）上编译时被解压缩。

调用Extension 函数

一旦你设置了你的extension ，你就可以像使用GameMaker Language 中的内置函数和常量一样使用它。如果你的游戏是跨平台的（例如，它被设计为在Windows和Android 目标上运行），那么你应该在调用extension 特定函数之前使用操作系统常量和函数来检查当前平台，以防止出现问题，例如：

switch(os_type)
{
    case os_android:
        call_android_extension_function(x, y, z);
    break;

    case os_windows:
        call_windows_extension_function(x, y, z);
    break;
}

请注意，对于不同的平台，你的extensions 可以共享函数名称，因此，如果你有一个extension ，有不同的文件，以（例如）在Windows ，为Android ，弹出一个消息，两个extension 文件可以共享相同的函数名称，只要它们都设置为只输出到各自的平台。

官方第三方扩展

传统版本的GameMaker 有很多内置的第三方功能，但这是低效和难以维护的，因为如果任何特定的功能被第三方改变了，那么整个GameMaker IDE和Runtime就必须更新。因此，目前的版本已经采用了使用extensions 包来为你的游戏添加第三方支持。这意味着你可以只添加那些你需要的额外功能，你甚至可以调整和编辑它们以适应你的具体需要。

这些extensions ，大部分会在你选择特定的游戏选项时为你下载和安装（例如Facebook或Google移动广告），但你可以从GameMaker知识库中每个平台的相应部分找到关于它们的进一步信息。
