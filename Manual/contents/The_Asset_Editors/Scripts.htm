TheScript 编辑

本节涉及到脚本编辑器（也叫文本编辑器），在这里你可以输入所有你的游戏运行所需的代码。这个编辑器用于创建自定义script assets ，以及为对象事件编码，添加房间创建代码，以及其他许多事情。

本节只是详细介绍了script 编辑器 的工作原理，而不是如何创建script assets ，也不是如何使用GML代码或GML Visual来制作scripts 。关于这一点，请看下面的章节。

在创建一个script asset ，你可能会被要求在GML Visual和GML Code之间进行选择。更多信息见GameMaker语言。

script 编辑器是你编写GameMaker Language 代码的地方，这是GameMaker使用的内置编程语言（详见GameMaker语言部分）。一旦你对GameMaker更加熟悉，并想充分使用它，建议你开始学习使用这种语言，因为它大大扩展了你在创建游戏时的可能性。

一个script asset ，可以通过在资产浏览器中右击 ，选择"重命名"来重命名，但要注意，script 的名字必须符合assets 的脚本规则，所以它们只能是字母数字，必须以字母开头，而且唯一可以包含的符号是"_"下栏符号。其他地方的script 编辑器不能重命名，会有一个特定于正在编辑的内容的名字，比如说object 事件。

当你打开script 编辑窗口时，它将有以下选项和布局。

脚本/事件标签

script 编辑器在一个顶部有标签的窗口中打开，让你在一个窗口中拥有多个scripts 或事件（尽管这种行为可以从首选项中改变，为每个script 或事件提供一个新窗口）。你可以在一个标签上点击 ，然后拖动它来改变标签的顺序，或者--如果你愿意--你可以把它从当前的窗口中拉出来，并把它放在 你也可以把它从当前窗口拉出来，放在workspace ，为该script 创建一个新窗口（或把它添加到不同的窗口），你也可以把script 编辑器最大化，以创建一个新workspace 。如果你把一个script 标签拖出 IDE窗口，那么就会产生一个新的IDE ，以容纳这个script 资源，可以像使用主窗口那样使用它。

请注意，如果你在script 编辑器中从一个对象事件 中编辑代码，并且你已经最大化了script 编辑器或将其放在一个单独的窗口或workspace ，那么右键 编辑器菜单将有一些额外的选项。

  • 恢复到工作区。这将使script 编辑器脱离最大化/工作区状态，并将其重新链接到object 中的workspace 。
  • 转到对象。这将带你到有代码的workspace 所处的object ，并聚焦于object 。
  • 添加/打开事件。这允许你在当前object 属于的script ，并将在Script 编辑器中为添加的事件打开一个新的代码标签。如果选择的事件中已经有了代码，那么这将在新的标签中打开。 将在一个新的标签中打开。
