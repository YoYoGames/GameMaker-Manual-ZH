TheRoom 编辑

room 编辑器是你创建游戏室的地方。每个游戏至少需要一个room ，在room ，你可以放置实例、sprites 、tiles 、paths 和背景。每一个不同的assets ，都可以放在他们自己独特的层上 ，然后可以在图层编辑器中以你希望的方式排序。由于room 编辑器的复杂性，我们将首先给你一个最重要的功能的简要概述，然后你可以从下面列出的章节链接中找到更深入的细节。

当你创建一个room 资源时，你可以在Asset 浏览器中右键点击 ，打开上下文菜单，这将允许你创建子rooms （更多信息请参见继承页），打开room 进行编辑，添加一个新的资源组以更好地组织rooms ，重命名room 或删除它。请注意，要改变room 的顺序和/或继承，你需要使用房间管理器，你可以使用Asset 浏览器右上方的菜单打开它。

room 编辑器本身就是一个工作区，因此你可以点击标签上的 ，把它从主窗口拖到一个自己的新窗口中--比如说，可能在另一个显示器中。你也可以通过将标签拖到 的顶部并释放鼠标按钮将其放回主窗口。 IDE

room 编辑器的用户界面很容易浏览，并被分成不同的独立部分。编辑器中那些被停靠的部分--房间属性和图层编辑器以及不同的图层属性部分--也可以通过简单地将它们拖出到workspace ，从停靠区中移除，并且可以通过将它们拖到workspace 的两侧或底部，再次将它们加回停靠区。

下面是对Room 编辑器各部分的简要概述。

尺子/导引器

沿着画布边缘的标尺向你显示放置在其中的东西的位置，并从（0，0）开始标记，这是画布的中心和room 的原点。你可以点击并拖动标尺，将一个水平或垂直的指南拉到room ，然后这个指南可以用来准确定位正在使用的不同assets ，因为将一个asset 靠近一个指南就会将它 "扣 "到指南上。当在assets 内定位room 时，也会临时显示智能导向器，指示assets 之间的距离，以及与room 边界或中心点的距离。

assets "捕捉 "到指南的距离，以及其他属性，可以在房间编辑器的偏好中设置。

图层编辑

room 编辑器将东西放在room 内的层上。每个层都有一个不连续的 "深度"，在层窗口中出现在列表底部的东西将被画在靠近顶部的东西下面。

重要的是!有一个最小和最大的层深度为-16000到16000。任何放置在该范围之外的图层都不会被画出来，尽管所有的事件仍将正常运行。

通过点击你想创建的图层类型的相应按钮来创建图层，这些按钮是：。

实例 这一层是用来放置将在room 中使用的实例。
瓷砖地图 这个图层是用来放置tiles ，从瓷砖集作为瓷砖地图。
路径 这个图层是用来显示和添加路径的room 。
资产 这一层是为视觉assets ，要添加到room ，如精灵和序列。
滤波器/效果 这种类型的图层定义了一个滤镜/效果，并应用于它下面的所有图层。
背景介绍 这种类型的图层定义了一个背景，它基本上是一个单一的颜色或sprite ，可以平铺在整个room 。

你也可以用文件夹按钮 ，创建一个图层文件夹，在这里你可以把选定的图层组合在一起，也可以用删除按钮 ，删除选定的图层。这里将详细讨论这些图层类型中的每一种。

请注意，你可以切换图层编辑器的继承权，这将影响图层的顺序和可见性。能见度本身可以通过点击每个图层旁边的眼睛图标 ，或者你可以点击最上面的那个图标来一次性启用/禁用所有图层。你也可以通过点击挂锁图标 ，"锁定 "一个图层（或所有图层），使它们不能被改变。请注意，被标记为不可见的图层在游戏运行时不会被显示。

图层属性

room 图层属性窗口将根据当前在图层编辑器中选择的图层而改变。

每个窗口都会有与当前选择的图层类型相关的不同属性和列表，你将能够编辑图层如何显示和图层上实际有什么的基本细节。更多细节请看这里。

房间属性

room 编辑器的这一部分是你可以定义room 的所有基本属性的地方，比如它的大小，是否使用物理学，或者它需要的任何摄像机和视口。这一部分在这里有更详细的讨论。

图层工具箱

某些图层类型将有额外的工具添加到图层工具箱中的IDE 顶部（例如，tile 图层或path 图层）。确切的工具将根据当前正在编辑的图层类型而改变，因此在解释每个图层的页面上有解释，这里。

房间画布

room 编辑器窗口的中央是进行所有实际编辑的区域。在这里你将放置你的实例和assets ，绘制你的tiles ，或定位你的paths 。你可以使用鼠标滚轮或顶部的room 控件进行放大和缩小，你可以通过按住鼠标中键 并移动鼠标，或在移动时使用 + 进行平移。

在创建asset 图层或实例图层时，您可以通过简单地从asset 浏览器中拖动Asset 或实例，然后将其放置在您希望的位置。或者，您可以从asset 浏览器中选择一个Asset 或实例，然后按住 ，以 "原地 "预览该资源，如果您另外点击鼠标左键 ，您可以添加该实例，甚至通过保持按住按钮和拖动鼠标将其 "画 "到room 层中。请注意，默认情况下，以这种方式添加到sprites 的实例和room 将被捕捉到定义的room 网格设置，但你可以按住 / ，让实例精确地跟随鼠标而不捕捉（释放 / 将允许再次捕捉）。

对于paths ，你可以在path 编辑器中创建一个新的path 图层和Room 资源--或者创建一个新的path 资源，然后把它拖到主编辑窗口，就像你对一个实例所做的那样--然后在编辑窗口中编辑path 和它的连接。对于tile sets ，你可以从tile set 编辑器中 "绘制 "它们。

请注意，你可以通过按住 / + 来选择和移动或删除同一图层中的多个assets ，而且room 编辑器中的大多数资源将允许你旋转和缩放它们（单个或一组），方法是沿着外部边缘或角落点击并按住 ，然后拖动鼠标。你也可以在任何单一的asset （tiles ，实例，sprites ）上双击 ，让编辑器平移到它，并打开所选asset 的属性窗口。

房间菜单

除了编辑器内的工具，你还会看到一个新的下拉菜单已被添加到IDE 的顶部。

这些菜单的解释如下。

  • 图层视图。如果你在任何时候关闭了图层编辑器窗口，这将重新打开它。
  • 房间属性。如果你在任何时候关闭了Room "属性 "窗口，这将重新打开它。
  • 图层属性。如果你在任何时候关闭了图层属性窗口，这将重新打开它。
  • 实例创建顺序。这将打开实例创建顺序窗口 （也可以通过使用房间属性 窗口中的按钮打开）。
    这个窗口按照创建的顺序（从上到下）列出room 中的所有实例。如果您需要在任何其他实例之前创建一个特定的实例，您可以简单地点击 ，然后将其拖到您需要的位置。
  • ResetWindows On Current Desktop:这将把room 编辑器的窗口布局重置为当前聚焦的桌面workspace 的默认值。

  • 瓦片编辑。在处理tile 图层时，该选项将在下拉菜单中被突出显示。它有以下子菜单。
    当编辑tile 图层时，你可以通过点击、按住并拖动鼠标左键选择多个tiles ，或者你可以使用 / + 选择并分组单个tiles ，或者使用 + 从选择中删除单个tiles 。从这个菜单中，你可以清除选区或反转选区，也可以剪切、复制或粘贴选区到另一个图层（甚至是同一图层但在另一个位置）。如果你关闭了tile set 编辑器的停靠窗口，最后一个选项将重新打开它。
  • 将图像转换为Tile 地图。转换图像到Tile 地图选项是一个强大的工具，可以用来导入一个单一的图像，然后从图像中提取所使用的tiles ，并在tile 编辑器中重新创建为一个room 地图层，在这个过程中创建所需的tile set 和sprite 。当你选择这个工具时，你会被要求提供一个未压缩的图像文件（PNG, GIF 或 BMP ），然后会被加载。在加载时，你将看到图像导入工具。
    在这里，你可以选择在创建tile 时如何分割tile set ，你可以设置单元的宽度和高度，以及边缘所需的任何偏移。你还可以为最终创建的sprite ，设置大致的宽度（像素）。例如，如果tile set 是由64x64的tile 单元组成的，而你在这里设置的宽度是200，那么最终创建的sprite 将是192像素宽（即：3个tile set 单元），供tile 使用。如果你把它保持在默认值0，那么GameMaker将尝试尽可能地制作一个 "方形 "的sprite ，其水平和垂直tile 单元的数量大致相同。

    在设置了图像的分割方式后，点击移除颜色选项旁边的色块将打开颜色选择器，允许你选择一个要从最终sprite 。这通常是你想要去除的背景颜色，将此色块设置为100%透明（alpha 0）以外的任何颜色都会在导入时去除所选颜色。

    最后，你可以选择命名sprite 、tile set 和tile 地图层，这个工具将为你创建。当你最终完成导入时，将创建一个sprite ，其中所有的图像都以网格的形式排列，同时还将从这个tile set 创建一个sprite 。在room 中，将创建一个新的tile 地图层，并使用生成的tile set 复制图像。请注意，该工具不会 复制tiles ，而是在一个单元格的图像数据与另一个单元格重合时进行识别（这包括旋转的tiles ）。你可以在下面的图片中看到这一点，这里只创建了一个花tile ，还有一个板条箱tile ，但这两者都有多个实例，其中一些实例在基础图片中被旋转。
