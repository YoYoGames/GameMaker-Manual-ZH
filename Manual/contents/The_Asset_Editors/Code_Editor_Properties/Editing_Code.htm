编辑代码

当你开始使用script 编辑器时，你只需按照正确的GML语法，为script asset 或事件输入你想要的代码。当你这样做时，你会从自动完成窗口得到帮助。

这个窗口会在你写代码的时候弹出来（通常是在稍微停顿之后，但可以在偏好中设置），并按字母顺序给你一个 "最合适 "的答案选择。

在使用代码编辑器时，你可以点击 ，然后拖动鼠标来选择一排代码，或者按住 ，然后点击 ，然后拖动来选择代码列。以这种方式选择的任何内容都可以使用标准的键盘快捷键 / + " C", / + "X" 和 / + "V"进行复制、剪切 和粘贴 。

值得注意的是，你可以用各种方式注释 你的代码。注释你的代码对于保持生产力是非常重要的，因为它可以保持事情的清晰，并以这样的方式将代码的各个部分分开，使每个部分的作用和存在的原因一目了然。这也意味着，如果你在一个团队中工作，那么你可以给其他成员留下注释，他们会清楚地知道你做了什么，或者某段代码是做什么的。即使你是一个单独的开发者，在你的代码中留下注释也是很重要的，特别是当你不得不离开这个项目并在以后再回来的时候。

注释你的代码很简单，因为你只需要在任何一行文本前使用"//"，或者你可以通过使用"/* text here */"注释整个段落。甚至有两个键盘快捷键来注释和取消注释选定的代码行。 / + "K" 来注释， / + + "K" 来取消注释。下面的图片展示了使用这两种方法编写的代码，以及一些JS文档注释（下面有更详细的解释）。

脚本也可以添加某些JSDoc风格的注释来定义它们所包含的函数的属性，这样当你在代码中使用这些函数时，它们就会和它们的参数及其他细节一起显示在自动完成中。你可以从JSDocScript 注释一节中了解到更多的信息。

默认情况下，所有编写的代码在每一组匹配的大括号旁边都有代码折叠的图标 {} 。这些图标的形式是一个小的加号 + 或减号 - ，点击这些图标将把代码 "折叠 "在匹配的大括号内，这样它所占用的视觉空间就会减少。

你也可以使用特殊的 #region 标签来标记一段代码，手动设置代码折叠区域。其工作原理是，你在一段代码的顶部添加一个 #region 标签（连同任何注释文本），然后在该部分的底部添加一个相应的 #endregion 标签，然后你就可以 "折叠 "代码，即：将其隐藏起来，这样你就可以在任何时候减少编辑器中的代码行数。

实际的代码看起来是这样的。

#region This is an example region

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Further comment here if required

正如你可能已经注意到的，script 文本的部分内容是彩色的。script 编辑器 "知道 "现有的objects 、内置变量和函数等......并对它们进行相应的颜色编码。颜色编码对避免错误有很大帮助，因为你可以立即看到你是否拼错了一些名字，或将一个关键字作为变量，或有一个语法错误。然而，如果你不喜欢颜色编码，在首选项中，你可以switch （你也可以使用 F10 ，在编辑器中打开或关闭它），以及自定义使用的颜色。

你也可以在编辑器的任何地方右击 ，打开以下菜单。

除了你期望的剪切、复制和粘贴代码行的常规功能外，你还可以选择评论或取消评论任何选定的文本，添加或删除缩进，以及将代码窗口分成1、2或3个独立的窗格（注意，如果你没有在正在编辑的代码中定义任何选项，与 #regions 有关的选项将不可用）。

最后一个分割成窗格的选项将把实际的窗口分割成不同的部分，然后你可以拖动一个script 或事件，让它与其他窗口一起显示，以便进行比较和编辑。

这对于比较两个scripts ，或同时查看连接的scripts ，而不需要另一个窗口或不得不在标签之间不断地switch ，是特别有用的。

这个菜单中的其他选项是切换一个breakpoint ，以便与调试器一起使用，以及将GML 代码转换为GML Visual，反之亦然。请注意，如果将代码转换为GML Visual，那么它将不会 为每一行代码创建单独的动作，而是使用一个单一的代码动作来容纳以前在事件或script 。
