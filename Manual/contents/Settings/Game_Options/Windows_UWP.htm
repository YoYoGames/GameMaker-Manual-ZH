Windows UWP游戏选项

本节概述了可用于控制你的UWP（UniversalWindows Platform）游戏项目如何被编译的不同选项。不同的部分是。

一般

UWP选项的一般部分是你可以设置产品信息的地方。

  • 显示名称。指定显示给用户的应用程序的友好名称（此string ，可本地化）。
  • 软件包名称。这是你所创建的软件包的名称，应该与你为游戏创建的开发者密钥所用的名称一致（见下面关于包装的部分）。
  • 包的显示名称。这是包的 "短名称"，最好与显示名称相同。
  • 发行商显示名称。这是你的公司或个人的名称，你希望将其作为游戏发行商。
  • 版本。你的游戏的版本号。

设置之后，你可以选择你的游戏可以运行的不同方向 ，以获得最佳的用户体验。可用的选项是。

  • 景观
  • 形象代言人
  • 景观翻转
  • 纵向翻转的

默认情况下，所有这些都将被选中，你应该取消选择那些你不需要的东西。

在设置方向选项后，你可以从以下选项中发出你的UWP应用程序需要的不同权限 的信号。

  • 启用互联网权限：Flags ，认为游戏可以访问互联网。这在默认情况下是关闭的。
  • 启用麦克风权限：Flags ，游戏可以访问设备的麦克风。这在默认情况下是关闭的。
  • 启用IAP沙盒：Flags ，游戏使用沙盒式的应用内购买。这在默认情况下是关闭的。

设备家族

在这里，你可以选择在创建你的可执行文件时，你希望Windows UWP 输出的目标设备家族。默认情况下，"支持通用设备家族 "选项被选中，这意味着该应用程序将针对所有设备和家族。为了限制应用程序只支持特定的设备家族，应该取消勾选 "通用 "选项，并勾选适当的设备家族。也可以支持通用设备家族，但限制其他设备家族只支持某些平台版本--在这种情况下，你要保持勾选通用设备家族，但也要勾选你希望支持不同平台版本的其他设备家族，并在适当的文本框中输入该家族的版本细节。关于这个逻辑是如何运作的，请看下面的微软支持页面。

  • 目标设备系列。例子

一般来说，你不会想碰目标和最低平台版本，应该把它们留在默认值，但如果你有构建问题--特别是，如果你不能安装默认设置的Windows SDK--那么你就需要把它们改成你已经安装的。如果你不确定要设置什么值，那么在Visual Studio 中创建一个新的UWP项目，看看它是如何设置这些值的。

关于设备家族的更多信息，请参见以下微软文件。

图形

图形选项是你应该配置的，以确定你的游戏将如何使用你的目标UWP设备的图形卡。包括以下选项供你修改。

  • 在像素之间进行颜色插值。打开线性插值，这基本上是 "平滑 "像素。对于清晰的像素图形，它应该是关闭的，但如果你有漂亮的阿尔法混合和平滑的边缘图形，它最好是打开。
  • 显示光标。当这个选项被启用时，默认的操作系统光标将显示在你的游戏中。如果这个选项被禁用，那么当鼠标在游戏窗口上时，将没有可见的光标，除非你在游戏项目中创建了一个光标（这个选项默认是打开的）。
  • 开始全屏。如果选中这个选项，游戏将以全屏模式启动，否则将以窗口模式启动。默认情况下这是关闭的。
  • 允许全屏切换 switch ：如果勾选了这个选项，用户可以使用标准的Ubuntu 操作系统的快捷键从全屏切换到窗口，然后再返回。默认情况下这是关闭的。
  • 缩放。在这里你可以选择保持长宽比（所以4:3的room ，在16:9的屏幕上会有 "字母框"）或完全缩放（将图像拉伸到适合全屏）。

    警告! 关闭应用程序表面将禁用在Windows 游戏选项中设置的所有缩放选项，直到它再次被打开。更多的细节请看应用程序表面。

最后是设置纹理页的大小的选项。默认（也是最兼容的）尺寸是2048x2048，但你可以从256x256到8192x8192之间的任何地方进行选择。还有一个按钮标明 预览 的按钮，它将为这个平台生成texture 页面，然后打开一个窗口，以便你可以看到它们的样子。如果你希望看到texture 页面的结构，并防止texture 页面大于（或小于）所需，这可能是非常有用的。关于texture 页面的更多信息，请看这里。

注意：请注意，texture 页面的尺寸越大，你的游戏在规格较低的PC上的兼容性就越差。

图片

在图像部分的第一部分，你可以选择设置各种瓷砖 ，你的游戏将在UWP可以运行的不同设备上使用。你可以设置以下内容（它们都应该是 *.png 格式的文件）。

  • 小瓦片图像是出现在搜索栏和用户界面其他区域的应用程序名称旁边的标志图像。它应该是44 x 44像素。
  • 中号瓦片图像应该是一个71x71像素的图像。
  • 下一个tile 图像是商店的标志tile ，它应该是50 x 50像素。
  • 宽幅图像瓦片是当tile 在其宽幅格式下出现的图像，应该是310 x 150像素。如果不提供这个图像，tile 只能以方形格式显示，不能接受基于宽幅模板类型的通知。由于用户可以决定tile 使用哪种格式，所以建议你包括一个宽幅图片tile 。还要注意的是，如果提供了宽幅图片tile ，tile 最初可能以其宽幅格式出现。
  • 大图像瓦片是大的实时tile ，应该是310 x 310像素。请注意，这只会在桌面操作系统上使用。

下一节是定义游戏标志。这基本上是你的游戏在UWP生态系统中使用的图标，值得注意的是，这些图像可以是透明的 .png ，在这种情况下，它们将被绘制在本节中指定的背景颜色上。这种背景颜色也适用于任何应用程序拥有的对话框中的按钮颜色，以及设备上商店中的应用程序描述页面。

在设置了标志之后，你可以为游戏设置闪屏。这是在你开始游戏时显示的屏幕，它不能大于620 x 300像素，和标志一样，它可以是一个透明的 .png ，在这种情况下，你选择的背景色将被用于它。

值得注意的是，GameMaker有一个项目图像生成器工具，可以用来自动创建你的游戏被编译到的所有不同目标平台所需的所有图像。如果你使用这个工具，你应该修改创建的图像以确保它们是你所需要的。

包装

GameMaker为Windows UWP 开发提供了一个默认的开发者密钥，但建议你生成自己的密钥并将其指向这里。要做到这一点，在Visual Studio 中创建一个空项目，然后把自动生成的 *.pfx 文件的副本存放在安全的地方，它将包含在空项目中（这就是开发者密钥）。然后你应该在这里链接该 *.pfx 文件。

在链接到该文件后，你应该点击相关的安装 按钮，然后在打开的命令提示符中按照指示操作。如果你不这样做，那么使用运行/调试按钮（或点击F5）将失败，你的游戏不能在UWP平台上测试（然而，如果你使用创建可执行文件，认证安装过程将自动启动，如果 *.pfx 尚未注册）。

注意：当涉及到将你的应用程序上传到商店时，你将需要为你的标题获得正确的*.pfx 。这是通过将你的应用程序与Windows 中的Visual Studio 商店相关联来实现的，这将导致Visual Studio 为该应用程序专门生成一个新的*.pfx 。还值得注意的是，你可以将任何 应用程序与Windows 中的Visual Studio 商店联系起来，这意味着你可以将一个空白项目与Windows 商店应用程序联系起来，并简单地将生成的*.pfx 复制到一个安全位置，然后可以在UWP游戏选项中使用。你可以从YoYo Games知识库中找到关于如何做的说明。

在这一节中，你还需要指定用于生成签名证书的 发行人名称。输入的内容应始终以 "CN=" 为前导，例如： CN=YoYoGames 。

淘宝网

在UWP游戏选项的Xbox Live部分，你可以激活你的UWP项目在Xbox One主机上使用。在这里激活Xbox项目允许你在Xbox One上运行该项目时进行以下操作。

  • 从应用程序中登录Xbox Live
  • 在项目中以某种方式使用Xbox Live用户身份（例如，显示游戏名或玩家照片）。

然而，在你做这些事情之前，你将需要确保你已经点击了启用Xbox Live的复选框，如果你是微软创造者计划的一部分，你也应该勾选创造者计划应用程序选项。如果你是创造者计划的一部分，你将需要添加标题ID和服务配置ID（关于这些的更多信息可以从微软找到）。

注意，目前要在你的项目中测试Xbox Live功能，你需要使用创建可执行文件而不是运行，因为UWP Xbox输出需要使用你作为打包信息的一部分提供的Windows Store签名证书（见上文），而不是GameMaker提供的临时证书，这就是你点击运行时将使用的。

一旦你启用了XBox Live，你就可以在你的项目中使用XBox Live功能，但要注意的是，对于最基本的功能，你需要成为创造者计划的一部分，对于高级功能，你需要一个ID@XBox账户。
