Windows游戏选项

本节概述了可供你使用的不同选项，这些选项控制你的Windows 游戏项目的编译方式。这些不同的部分是

一般

Windows 游戏的常规 页面是你应该提供必要信息的地方，以便在你的游戏被安装和运行时， 显示。这些信息在 "Windows 产品信息"部分，应填写完整。

这个窗口的选项 部分允许你选择游戏包的安装位置，是 "%localappdata%"还是 "%appdata%"文件夹，以及是否设置复制EXE到输出文件夹。一般来说，你不应该需要这个选项，但它是为那些有各种DLL的用户提供的，这些DLL与你的游戏相互通信，当启用时，它将在测试时把游戏EXE保持在输出文件夹中，而不是把它放在Temp 文件夹中。

接下来你可以设置Windows睡眠保证金。这个选项与减少在特定系统上运行游戏时的卡顿有关。基本上，如果你的游戏运行速度比你的room 速度快，GameMaker就会在剩余的时间里 "睡眠"，但是这种睡眠可能是相当不准确的，你往往会因为睡眠时间过长而导致你的画面耗时过长，从而导致卡顿现象。为了解决这个问题，我们可以减少睡眠时间，然后在剩下的时间里坐在一个紧密的loop ，使其更加准确--尽管坐在一个loop ，问题是它会导致CPU使用率增加，这反过来会导致你的CPU温度上升和你的风扇运行速度加快。默认情况下，这个值被设置为10，在99.99%的情况下，你用这个值就可以了，但是对于低端机器或有大量后台进程运行的机器，这可能不是一个理想的解决方案（或者对于非常高端的机器也是如此），可能需要一个1或15或其他的值。但请注意，这在很大程度上是一个系统特定的配置，在你的构建机器上有效的配置可能不适合其他用户，因此，如果有疑问，请将其设置为10。

注意 Windows ：除了睡眠保证金，你还可以在runtime ，通过调整线程调度器的分辨率，进一步控制你的性能和功耗。

下一个选项是为你的游戏启用Steam。如果你启用了这个选项，请确保你已经安装了Steamworks SDK（见这里），并且你已经在游戏的常规选项中加入了Steam应用ID。Windows 请注意，如果你在Steam ，并从IDE 版本中运行GameMaker，并且你正在为Steam 开发游戏，那么你可能还需要勾选为Steam应用程序使用替代启动器的选项。这个选项的作用是在测试时使用替代的启动程序启动你的游戏，以防止它被Steam ，用GameMaker的应用ID而不是你的游戏自己的应用ID来标记。你应该只在为Steam 开发时启用这个选项，如果你发现你的游戏启动有问题。

你拥有的倒数第二个选项是禁用文件系统沙盒的选项。勾选这个选项将禁用GameMaker沙盒来保存和加载文件，允许你的游戏从运行游戏的系统上的任何地方访问文件。这个选项默认情况下是不被选中的，因为我们建议你在所有系统上对文件保持GameMaker沙盒，当被选中时，一些保存/加载区域可能仍然不可用，这取决于操作系统级别的权限。关于沙盒的更多信息，请看GameMaker 文件系统的页面。

最后，你可以选择让GameMaker为你的Windows 项目创建一个x64可执行文件。当启用时，这个选项将为你的游戏在编译时创建一个64位的可执行文件，我们强烈建议所有新项目使用这个选项。在长期的未来，这将成为Windows 可执行文件的默认（也是唯一）格式，因为32位运行器将被逐步淘汰。请注意，当启用时，项目所使用的任何extension dlls也需要是64位的（更多信息见这里）。

图形

图形选项是你应该配置的，以确定你的游戏将如何使用你的目标Windows PC的显卡。包括以下选项供你修改。

  • 显示光标。当这个选项被选中时，常规的窗口光标将被显示出来，不选中它将意味着没有光标被显示，除非你在游戏代码中创建了一个光标。默认情况下，该选项是打开的。
  • 开始全屏。如果选中这个选项，游戏将以全屏模式启动，否则将以窗口模式启动。默认情况下这是关闭的。
  • 允许全屏切换 switch ：如果勾选了这个选项，用户可以使用标准的Windows 捷径从全屏切换到窗口，然后再返回。默认情况下这是关闭的。
  • 在像素之间进行颜色插值。打开线性插值，这基本上是 "平滑 "像素。对于清晰的像素图形，它应该是关闭的，但如果你有漂亮的阿尔法混合和平滑的边缘图形，它最好是打开。
  • 使用同步来避免撕裂。这个选项可以打开或关闭v-sync。room 注意，如果你的游戏的速度是120，而玩家的显示器刷新率是60，打开这个选项也会把你的游戏速度锁定在60。这在默认情况下是关闭的。
  • 允许窗口调整大小。勾选此选项允许用户改变游戏窗口的大小（无边界窗口选项需要关闭才能发挥作用）。该选项默认为关闭。
  • 无边界窗口。勾选此选项将使你的游戏以无边界窗口运行，移除正常的最小化、最大化和关闭按钮以及游戏标题。这在默认情况下是关闭的。
  • 缩放。在这里你可以选择保持长宽比（所以4:3的room ，在16:9的屏幕上会有 "字母框"）或完全缩放（拉伸图像以适应整个屏幕）。

    警告! 关闭应用程序表面将禁用在Windows 游戏选项中设置的所有缩放选项，直到它再次被打开。更多的细节请看应用程序表面。

最后是设置纹理页的大小的选项。默认（也是最兼容的）尺寸是2048x2048，但你可以从256x256到8192x8192之间的任何地方进行选择。还有一个按钮标明 预览 的按钮，它将为这个平台生成texture 页面，然后打开一个窗口，以便你可以看到它们的样子。如果你希望看到texture 页面的结构，并防止texture 页面大于（或小于）所需，这可能是非常有用的。关于texture 页面的更多信息，请看这里。

注意：请注意，texture 页面的尺寸越大，你的游戏在规格较低的PC上的兼容性就越差。

图片

图像部分是你提供你的游戏所需图像的地方。对于Windows ，你需要提供一个图标文件（必须是 .ico 格式）和一个闪屏（可以是 .png 、 .bmp 、 .jpg 或 .gif 格式）。闪屏将在游戏加载时显示，可以通过勾选使用闪屏选项（默认为关闭）来启用或禁用。

值得注意的是，GameMaker有一个项目图像生成器工具，可以用来自动创建你的游戏被编译到的所有不同目标平台所需的所有图像。如果你使用这个工具，你应该修改创建的图像以确保它们是你所需要的。

安装人员

Windows游戏可以用安装程序 来创建，在这里你可以定义GameMaker在创建它时将使用的各种设置和图形。首先，我们有图形选项，它将定义安装程序本身的外观。

  • 完成了。出现在安装程序的 "完成 "页面上的图形。必须是 *.bmp 格式，尺寸为164x314px。
  • 标题。安装程序运行时出现的图形。必须是 *.bmp 格式，尺寸为150x57px。

你还可以指向一个特定的许可协议文件（GameMaker提供了一个例子）和一个自定义的NSIscript 文件（详情在这里），如果你需要的话，或者点击适当的按钮返回到GameMaker提供的默认文件。
