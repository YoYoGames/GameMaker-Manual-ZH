纹理组

纹理组管理器可从IDE 的工具菜单中获得。在这里，您可以添加、删除和重命名纹理组。

GameMaker允许你将每个图形assets (sprites,tilesets, 和fonts) 分配到不同的texture 组，以尝试和优化硬件在你的游戏中任何特定时间的texture 交换次数。要做到这一点，你需要在这个窗口中定义必要的texture 组，然后通过每个适当的asset 类型，从下拉菜单中选择你希望它属于的组。

选择一个assets 组列出的多个Texture ，然后右键单击，就可以将这些assets 移到不同的组中。

另外，你可以通过资产浏览器中的鼠标右键 菜单将assets 添加到一个texture 组。你可以选择一个或多个精灵/瓷砖组/字体assets ，然后使用人民币菜单将assets 添加到任何组中，如下图所示为瓷砖组。

重要的是要明白，一个纹理组可以有多个纹理页，它们并不等同。如果你的游戏每个关卡都有很多独特的图形，那么你可以为每个关卡定义一个texture 组 ，但每个组的图形可能需要两个或更多的texture 页面 。这取决于所选择的texture 页面大小（在你的目标的游戏选项中）是否能够容纳一个texture 组中的所有图形。

默认情况下，你游戏中的所有图像将被添加到 "默认 "texture 组，该组一直存在。然而，你可以点击添加新按钮来创建一个新的texture 组，如果你选择该组的名称，你可以改变它的内容。开始时，该组的每个部分（sprites 、tilesets 和fonts ）都会显示"此组为空"，因此你需要通过你的图形assets ，并将它们添加到一个组中。你可以使用底部的"添加资源"按钮添加资源，或者从资源编辑器中选择图片的texture 组，tile set ，或font 。除了 "默认 "组以外，Texture 组也可以随时删除，这将把它们的内容移到 "默认 "组中（注意，如果你错误地删除一个组，可以使用 / + "Z" 来撤销这个动作）。

一旦你定义了texture 组--并将图形资产分配给它们--你会看到它们列在主Texture 组编辑窗口的左边，点击这个列表中的任何一个sprites ，就会在右边的图像预览窗口中显示它们。这在处理大型项目时特别有帮助，你需要能够看到哪些图像被分配到哪个组。注意，如果分配给组的图像有多个子图像（即：它们是动画的），那么在预览窗口中只显示第一个子图像。你也可以在一个资源上右击 ，打开一个小菜单，让你选择把它移到另一个texture 组或打开该asset 的编辑器。

值得注意的是，即使assets 在列表中显示为独立的，所有sprites 、tilesets 和fonts 将被添加到同一个texture 页面（或多个页面），而不是为每个资源类型获得一个单独的页面。这条规则的唯一例外是，当一个sprite asset 被标记为"独立的Texture 页面"时，在这种情况下，它将被赋予它自己的独立的texture 页面（更多细节请参见Sprite Editor中的 "纹理设置 "部分）。

组别设置

Texture 组编辑器中的组设置用于准确确定如何为一个给定的组创建texture 页面，有以下选项。

  • 允许缩放- 解除这个标记将迫使GameMaker在任何情况下都不缩放texture 组。这可能很有用，特别是在处理通用应用程序时，因为它允许你把特定的资源，如fonts ，放在一个自己的组中，当应用程序在特定设备上运行时使用。例如，fonts 在视网膜显示器上看起来很好，但当它们被缩小到在普通显示器上绘制时，它们可能看起来很糟糕，所以你可以把它们分配到一个特殊的非缩放组中--如果为iPhone开发，你可能希望检查这一点，因为应用程序已经被拒绝，因为图像被texture 缩放而变得模糊。这在默认情况下被标记为开启 。
  • 自动裁剪 - 当你在GameMaker中添加图片时，如果它们周围有任何 "空间"（即100%的透明度），当它被添加到texture 页面时，它会自动裁剪，并且 "自动裁剪 "被标记出来。但是，如果，例如，你将图像作为基元或任何类似的非重复texture ，这意味着texture 贴图的 UV 坐标将不会被正确表示。在这些情况下，你可以取消这个选项的标记，sprite 将被 "原封不动 "地添加到texture 页面中，保持它在边缘可能有的透明性。这在默认情况下被标记为开启 。
  • 生成Mipmaps- 标记这个选项（默认情况下是关闭的）将告诉GameMaker，原始texture 页面将被缩放并过滤成texture 文件中的多个分辨率。这通常是关闭的，但是如果你使用摄像机来放大或缩小游戏的大面积room ，或者你正在处理一个3D游戏，那么你可能想启用这个，然后使用Mipmapping功能来调整游戏如何处理不同的mipmap级别。当你flag 这个复选框时，"边界大小 "值将被设置为8（如果它最初被设置为小于8）。这是因为，当使用mipmaps时，你需要在texture 页面上的图像之间有更大的边框，否则在更高的mip级别上它们会开始渗入。默认值8将允许图像在子图像开始出血之前被缩小到其原始尺寸的八分之一，但你可以根据你的需要将其改为任何值。请注意，如果你使用"独立的Texture 页面"选项sprites ，那么你不需要在这里通过每个texture ，flag ，以生成mipmaps，而是可以去 "一般游戏选项"，flag "为独立的texture 页面生成mipmaps"选项。
  • 边界大小 - 边界大小将改变为assets 组生成的页面上所有texture 的边缘的重叠（或 "像素加倍"）量，以便当图像被缩放时，它们之间没有间隙（或接缝）可见。默认值是2像素，但在极端情况下，如果你将视图放大或缩小了很多，那么你可能需要增加这个值。tiles请注意，"瓦片集编辑器"有一个选项可以设置tiles 的输出边框（还提供了为什么需要这样做的解释），这与这里的边框设置是分开的--基本上，tile 是在一个临时的texture 上生成的，有一个输出边框，然后这个边框被添加到texture 页面的外边缘。还要注意的是，启用Mipmaps会自动将texture 页面的默认边框大小设置为8，以帮助防止在较高的mip级别下textures 之间的 "出血"，但这可以改变。
  • 父Texture 组- 在这里你可以选择任何一个现有的组作为当前所选组的 "父 "组。"子 "组将inherit ，对父texture 组所做的任何改变，在处理跨平台游戏时主要用于资源管理。一个有父组的texture 组本身仍然是一个texture 组，并且会像其他组一样生成texture 页面，但是给他们指定一个父组可以让你排除不同平台的大块内容。例如，你可以把一大堆Android 玩家sprites 添加到一个 "玩家 "组，把Android 敌人sprites 添加到一个 "敌人 "组，然后让它们都使用一个 "安卓 "父组。如果你把 "android "父组从Windows 或其他平台的构建中排除，子组也将被排除。
  • 纹理组格式- 在这里你可以为你的texture 页面选择格式，在BZ2 + QOI、QOI和PNG之间选择。建议坚持使用默认选项，BZ2 + QOI，因为它在保持质量的同时提供了最好的压缩。

    如果BZ2选项影响到目标平台的速度，你可以选择QOI（例如：Android ），如果你想手动操作导出的PNG或使用PNGCrush选项，你可以选择PNG。

除了这些设置，你还可以选择将任何给定的texture 组导出（或不导出）到任何给定的平台，只需在窗口的 "在...上使用组"部分选中或不选中相应的框。但请注意， <Default> texture 组总是被导出到每个平台，你不能勾选/不勾选其中任何一个（只有自定义的texture 组允许这样做）。

纹理组也与配置设置相联系。这意味着你可以为你的游戏的每个单独的配置定义texture 组，然后将sprite 、tile set 或font 资源分配给这些不同的组，当你改变到一个不同的配置时，它将被 "记住"。同样的情况也会发生在你在Texture 组窗口中选择的任何导出选项上，所以你可以在每个配置的基础上将texture 组设置为导出到特定的平台。

一个用途是，例如，你可以做一个iPad配置，并为其创建2048x2048px的texture 页面（通过游戏选项进行设置）。然后，你将把所有的图形assets ，分配给适当的texture 组，完成后，你将改变到一个不同的配置（例如为HTML5），并从Texture 组窗口，创建512x512px的新texture 组，将assets 分配给这些新组。在页面底部还有一个按钮，允许你将texture 组设置从一个配置复制到另一个。

最后，texture 组只与位图图像有关，非位图sprites 不会被分配到texture 组。其中一些将已经有自己独特的texture 页面（骨架动画sprites ），而其他的只是不需要textures ，因为它们是由矢量组成的。欲了解更多信息，请参见以下章节。
