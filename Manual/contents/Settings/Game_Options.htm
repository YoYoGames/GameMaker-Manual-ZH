游戏选项

游戏选项分为两个部分，主要选项和平台设置。主要选项从一般选项开始，在这里你可以设置某些东西，这些东西将影响游戏，无论它被设计为在哪个平台上运行。

该窗口有以下部分。

  • 默认的脚本语言。选择当前项目中的Objects 事件和Scripts 是否应该使用GML Code或GML Visual，或者是否应该在创建新script 时总是询问。
  • 游戏每秒帧数：这个值设置了GameMaker在运行时应尽量保持你的游戏的每秒帧数的限制。这个值代表了游戏loop 在一秒钟内的运行次数（一个游戏loop 被认为是一个 "帧"），并被设置为60的默认值，尽管30也经常被使用。关于哪种room 速度是理想的，有很多争论，但实际答案取决于你正在制作的游戏类型、room 的大小、实例的数量以及代码的密集程度。例如，如果你有一个大的room ，而room 速度为30，游戏可能会显得 "不流畅"，因为你看的区域越大，眼睛就越容易感觉到低的room 速度，但如果游戏很密集，那么将room 速度设置为120可能会导致游戏 "滞后"，因为由于正在进行的处理量，帧数会下降。因此，请注意这个值，并尽量根据你的游戏需要来调整它。
  • 默认绘制颜色：当没有使用代码设置颜色时，绘制文本或论坛（和其他东西）的默认颜色。
  • room 区域外的默认绘制颜色。默认颜色，将用于清除游戏外的任何区域的背景room 。
  • 生成项目图像。点击它将打开项目图像生成工具。
  • 项目开始日期。项目启动的日期。
  • 项目使用时间：自你启动项目以来已经过去的时间长度。
  • 项目GUID：这是GameMaker中为每个项目创建的唯一标识符。如果你想改变它，你可以点击创建新的GUID按钮来生成一个新的。
  • 为独立的texture 页面生成mipmaps。这个选项将让GameMaker为那些在sprites 编辑器的Texture 设置中标记了SeparateTexture Page 选项的Sprite 创建mipmaps。启用mipmapping意味着texture 的原始sprite 页面将被缩放并过滤为texture 文件中的多个分辨率。这通常是关闭的，但是如果你使用视图来放大或缩小游戏的大面积room ，或者你正在处理一个3D游戏，那么你可能想启用这个功能，然后使用Mipmapping功能来调整游戏如何处理不同的mipmap级别。请注意，如果你没有把sprites 标记为使用单个texture 页面，你仍然可以从Texture 组管理器中以每个纹理组为基础启用mipmapping。这个选项在默认情况下是关闭的。
  • 启用源代码控制。如果你希望项目在SCM（Source Control Management）下，那么你需要勾选这个。默认情况下，它是关闭的（更多信息见这里source control ）。
  • 作者。游戏作者（或出版商或工作室）的名字。
  • 项目最后更改：这部分只是显示项目最后一次被更改的日期/时间戳（这是不可以编辑的）。
  • 接受Spine 许可证。在这里你可以勾选显示Spine Runtime 许可证的方框并接受它。这将允许你在你的游戏中使用Spine sprites （当你第一次尝试在你的项目中添加一个Spine sprite ，而事先没有在这里检查许可证时，也会显示这个许可证）。
  • 碰撞兼容模式。这告诉GameMaker ，使用2022年版本之前使用的传统碰撞系统（使用整数值而不是浮点数）。阅读碰撞兼容模式以获得更多信息。默认情况下这是禁用的，建议在新项目中保持禁用。
  • 启用数组的写时复制行为。这将为当前打开的项目启用已被废弃的写时复制行为。更多信息请阅读Copy on Write。默认情况下，该功能是禁用的，建议新项目保持禁用。

主选项部分还包含一个模板信息的菜单。

资产浏览器也会有其他游戏选项供你使用，这取决于你所拥有的许可证的目标平台，而且可能不是所有的选项都可用或可见。

平台设置

存在以下特定平台的游戏选项。

  • Opera GX
  • 窗户
  • macOS
  • 乌班图
  • Windows UWP
  • HTML5
  • 安卓
  • 基础设施
  • tvOS

注意由于不同公司有不同的法律限制, 这里没有列出各种控制台游戏选项。如果你有控制台许可证, 那么你可以从YoYo游戏服务台获得这些信息, 但你必须先用你的控制台ID电子邮件联系YoYo Games, 并请求允许查看不同部分。
