合并工具

当你和其他人一起在一个项目上工作时，总是有可能出现冲突，因为多人编辑同一个文件，大多数source control 系统可以很好地尝试合并任何有冲突的文件，但是当在同一个地方有变化时，需要一点帮助才能知道该怎么做--这就是合并工具 的作用，它可以向你展示变化，让你决定。

由于每个合并工具都是不同的，有四个宏可以用来帮助指定文件给它，可以从冲突 窗口点击标有选择合并工具的按钮来设置， 。

当你点击该按钮时，偏好窗口将在Git 插件上打开。

在这里，你提供一个path 到所选择的合并工具，然后在决定做什么时使用合并工具选项宏（或宏），有以下选项。

  • ${scm_mine}:path 到你的版本
  • ${scm_theirs}:path 到他们的版本
  • ${scm_base}:path 到发生冲突的修改之前的版本
  • ${scm_merged}:path 到合并工具应该保存的地方。

作为设置的帮助，当你把path 给某些合并工具时，GameMaker会根据选择的工具，填写一些默认值来开始使用。

  • KDiff3。 "${scm_theirs} "${scm_mine}" -o "${scm_merged}"
  • 融合。 "${scm_mine}" "${scm_merged}" "${scm_theirs}"
  • Notepad++。 ${scm_merged}
  • 龟甲合并。 /base:"${scm_base}" /theirs:"${scm_theirs}" /mine:"${scm_mine}" /merged:"${scm_merged}"
  • DiffMerge: "${scm_mine}" "${scm_merged}" "${scm_theirs}"

请注意，如果你没有（或不想使用）合并工具，那么你可以在script ，并试图从那里修复它，打开IDE 。

所以，回到我们在冲突一节中所说的冲突的例子......我们做了一个从主站拉出的文件，其中一个文件是冲突的。这将打开Source Control Conflicts 窗口，你可以点击冲突的script ，现在点击Merge，因为你已经配置了合并工具。点击合并现在将打开工具，你可以处理冲突，然后使用提供的宏来进行必要的修改。一旦你完成并关闭了合并工具，GameMaker IDE 将显示以下确认窗口。

只要对这个提示回答 "是 "就可以了，因为你刚刚用合并工具自己完成了合并，然后你需要提交这个变化，以表示合并操作的结束（Source Control > Commit Changes）。现在，提交窗口将显示合并正在进行中，同时也显示一些未缓存的文件。

这些未缓存的文件是我们的script 文件，另外还有 .base, .ours 和 .theirs 。我们可以安全地删除所有这些，所以点击上面的 ，然后点击下面的 + ，将它们全部选中。接下来，我们在选中的文件上点击右键，选择删除路径来删除它们。这里还为我们预先填写了一条提交信息，说明这是一次合并，并且有一个冲突（我们可以根据自己的需要添加或编辑）。

我们现在可以点击承诺 。

由于你可能有其他文件的问题，这意味着你在使用远程或共享存储库时的工作流程应该是这样的。

Make Changes > Save > Commit > Pull > Merge > Push

差分工具

你还应该设置一个差异工具，以便在对版本库进行提交时使用。这是在首选项中设置的，与合并工具相同。

差异工具的配置方式与合并工具基本相同，但只使用 ${scm_base} 和 ${scm_theirs} 选项 - 在这种情况下， ${scm_base} 代表未修改的文件，而 ${scm_theirs} 代表文件的当前状态。工具选项应该预先填入Meld、KDiff3 和TortoiseMerge的默认选项，就像合并工具那样。注意，GameMaker 不 支持输出单文件差异补丁，因此没有Notepad++的默认选项。

为了使用Diff工具查看当前版本和以前版本之间的变化，只需打开Commit窗口，然后双击一个文件。如果该工具设置正确，它将被启动，你就可以看到文件之间的变化。
