微型网络服务器

这个小工具的主要功能是连接到GameMaker的一些可用目标平台并为其提供文件（如HTML5目标）。在离线使用时，它也为GameMaker手册提供文件。

当你在一个需要Micro WebServer 的目标上运行一个游戏时，你会看到一个新的窗口打开，因为游戏被提供给所选择的设备。

注意当你第一次在一个需要Micro WebServer 的目标上运行游戏时，你可能会得到一个防火墙警告。如果发生这种情况，你应该允许网络server ，通过防火墙进行通信。

当Micro WebServer 运行时，你可以从任何本地设备访问你的游戏（甚至远程访问，如果端口转发设置得当）。只需通过该窗口中显示的端口访问你的计算机的本地IP地址，你就可以从其他设备，如手机，或其他计算机上玩你的游戏。

例如，假设你的电脑的本地IP地址是192.168.0.110（阅读此文知道如何找到它），而Micro WebServer 窗口中显示的端口是51264。在电脑上运行游戏后，你将从手机上的浏览器连接到192.168.0.110:51264，然后玩你的游戏。

WebServer 将显示一定数量的与已提供的文件有关的调试信息，并在底部有一个按钮可以清除这些信息。在 "清除 "按钮的旁边是项目的JS根文件的path ，而在它的下面你有网络速度选项。这个选项默认设置为 "无限制"，但点击它将打开一个不同网络选项的列表。

改变这个设置也将改变右边的包大小 选项。你也可以手动编辑，在这种情况下，网络速度将被设置为 "自定义"。将其设置为除最大值以外的任何其他值，将有效地节制网络传输速度，并允许你在低带宽环境下测试游戏。

你也可以看到允许的IP列表和正在使用的端口号 ，但这些不能在Micro WebServer 中编辑。它们应该在设备管理器中为特定的设备目标设置，或者在使用 "默认 "设备时在HTML5首选项中设置。
