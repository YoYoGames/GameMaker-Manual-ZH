Debugger 的整个想法是允许你 "观察 "你的游戏的每一个细节。这是通过 "观察窗口 "来实现的，默认情况下，这些窗口是在停靠在 的输出窗口中打开的。 IDE 你可以点击每个不同的标签来查看它们所包含的观察，其中一些可以实时更新（当从debugging 工具栏中启用实时Debugger ），而其他的则需要debugger 暂停游戏。

当你在任何窗口中观察变量时，它可以是多种数据类型中的一种。然而，由于GameMaker存储某些东西的方式，Debugger 并不总是确切知道这个 变量的内容。这是因为数据结构或实例的内部ID值是整数 ，所以Debugger 并不总是知道被存储在变量中的整数值是用于数据结构还是实例，或者仅仅是一个整数。 只是一个整数值。出于这个原因，你可以在任何变量值上右键点击 ，并得到一个弹出窗口，其中列出了这个变量的可用数据类型 该变量可能是的数据类型。

字符串和数组不允许有这个选项，值得注意的是，即使一个数据结构被存储在一个实例变量中，它的 ID 编号 是全局的，所以如果你有四个实例，并且每个实例都创建了一个ds_list（例如），每个list都会有一个从0到3的ID，它们不会都是0。

一旦你选择了你的数据类型，然后只需点击其名称旁边的小 ，变量就会显示结构中的条目。这将展开结构中的 内容，你也可以通过简单地双击 ，给它一个新的值，来改变一些显示的变量的值。

注意：你不能以这种方式重新分配数组或数据结构，但你可以改变它们的一些内容（你不能改变地图中的 "键 "值，也不能改变优先级queues ）。

请注意，"自己 "和 "其他 "实例可以通过点击 来扩展，以查看正在进行的代码块的所有局部和实例变量的列表。 所有本地变量和实例变量的列表，这些变量是为正在进行的代码块所声明的。如果你在一个值上点击右键 ，你可以设置它的类型 来查看数据结构的细节。

钟表

这个窗口是为了让你添加一个特定的变量，即 你想要跟踪的特定变量。这可以是一个全局 或实例 范围的变量，一个内置的变量，甚至是一个数组 或数据结构。如果正在进行的实例代码 有被监视的变量，它就会显示在这里，这意味着你可以轻松地跟踪一个值，而不必在本地 或全局 窗口中搜索它。如果你在一个值上点击右键 ，你可以设置它的类型，删除它，或者从窗口中清除所有的监视。
