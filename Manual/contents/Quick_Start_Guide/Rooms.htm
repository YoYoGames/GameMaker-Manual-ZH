你在GameMaker中制作的所有游戏都需要至少一个room 来运行（但可以有很多很多），而一个room 只是一个空间，你可以把构成游戏的objects 的实例以及tiles 和任何其他图形资源放在这里。 当你第一次在资产浏览器中创建一个room ，你将会看到一个新的 房间编辑器workspace 用来编辑它的属性。 像这样。

默认情况下，room 工具将被放置在左边，有三个主要部分。

  • 图层编辑器。room 中的所有东西都放在图层上，有不同的图层类型供你选择（后面会详细介绍）。图层是按深度 排列的，这个深度定义了游戏运行时图层内容呈现在屏幕上的顺序。 层的内容在游戏运行时将被渲染到屏幕上。深度排序从高到低，所以深度越低越接近 "摄像机"，深度越高越远。例如，一个深度为 深度为-300的图层会在深度为-100的图层上渲染，而深度为1000的图层会在深度小于这个值的所有东西下渲染。请注意，默认情况下，GameMaker会根据你在图层编辑器中的位置来排列图层的深度。 在图层编辑器中的位置，你可以用鼠标拖动图层来重新排序，GameMaker会重新排序深度来匹配。
  • 图层属性。你添加到room 的每个图层都有自己的属性，这些属性将根据图层类型而改变。这个窗口允许你改变这些属性，并编辑该层将如何被 渲染。
  • 房间属性：Rooms 也有一些属性，这些可以在这里设置。像room 的大小，激活的摄像机视图端口和其他一些东西都可以在这里设置。

我们已经提到有不同的层类型，所以让我们来看看它们是什么，以及在构建游戏时如何使用它们。

背景层 背景层是一个可以用单一 颜色或单一 图像填充的层。它可以在room ，而且你可以有多个背景层来移动和定位。一般来说，正如它的名字所暗示的那样，它被用来 room 默认情况下，一个新的room ，总是包含一个背景层，但如果你不需要它，你可以把它删除。
实例层 实例层是你放置游戏所需的所有objects 的实例的地方。要将一个实例添加到一个图层中，只需在浏览器中点击 ，然后将其拖到object 编辑器中。 asset 浏览器，然后把它拖到room 编辑器workspace 。当你释放鼠标按钮时，该object 的实例将被添加到room 。 默认情况下，一个新的room 将总是包含一个实例层，但你可以删除它 如果你不需要它。
瓦片地图层 在创建了瓦片集之后，你需要把tiles 添加到你的room ，这是通过创建一个瓦片地图层来完成的。tile 地图层是一个允许你从任何你已经创建的tiles tile set 中添加assets ，并将自动设置为使用与tile set 单元大小相同的网格。你只能在每个网格单元中添加一个tile ，所以如果你需要多个tiles ，占据同一空间，那么你应该使用多个tile 地图层。
路径层 path 图层是唯一一个在你运行游戏时不会被实际渲染的图层。这个层更像是一个 "方便 "层，用来帮助你创建或编辑Path 资产，因为它允许你在实际的paths 空间内添加或编辑room 。
资产层 有时你想要一个漂亮的图形效果，但又不想要使用实例的开销，或者不需要它做任何其他事情，只需要画出自己。你可以使用tile 地图，但是考虑到tile 地图被限制在使用一个网格，并且每一个网格单元只能有一个图像，它们可能有点限制。 每一个网格单元只能有一张图片，它们可能有点限制性。这时你会使用asset 图层。asset 图层只是把一个sprite 资源，用你在添加到room 时设置的参数来绘制。Sprites 和 sequences 可以被添加到这个图层中，就像objects 的实例被添加到实例图层中一样，即：你在sprite 浏览器中点击 的asset ，然后 把它拖到room 编辑器中workspace 。
