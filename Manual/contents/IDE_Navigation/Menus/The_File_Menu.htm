文件菜单

file menu ，在这里你可以选择更改、保存或导入项目，以及设置GameMaker IDE 的偏好。可用的选项有：。

  • 新项目 - 这是在GameMaker中从头开始创建一个新项目。它将打开 "保存 "对话框，并提示你选择一个位置来保存新项目。一旦你给项目起了一个名字和一个保存位置，你就会被带到一个新的、空白的工作区，开始进行工作。项目文件的保存方式是： *.yyp 文件extension 。
  • 打开项目- 这将打开一个文件资源管理器，让你选择一个先前创建的 *.yyz 项目文件进行加载。请注意，在导入时，如果该项目是用低于2.3.0的版本创建的，那么你将得到以下警告信息。 如果你选择继续，那么将打开一个文件资源管理器，你可以给项目一个更新的名称和保存位置，这样原项目就不会被改写。一旦项目被转换为新的2.3.0+格式，它将在IDE ，你会得到一个消息，说转换已经成功了。你应该继续检查由转换过程产生的兼容性报告，看看在尝试运行或编译项目之前是否有什么需要修复的地方。请注意，如果你在试图打开一个用2.3.0之前的版本制作的项目时重复出现错误，那么你应该联系YoYo Games （使用帮助菜单中的 "报告错误 "选项），并提供你试图转换的原始 *.yyp 文件。
  • 导入项目- 在这里你可以导入以前作为 *.yyz 档案导出的项目（见下面的 "导出项目"），或者 - 如果你有一个用传统的GameMaker:Studio 1.4制作的项目--你可以选择这个选项来导入该产品系列的 *.gmx 或 *.gmz 文件。与打开项目一样，如果 *.yyz 是用2.3.0更新之前的GameMaker版本创建的，那么它将需要转换，因为项目文件格式随着该更新而改变。

注意：要将以前的GameMaker 项目导入GameMaker，它必须是使用1.4版本创建的，即使如此，它也可能无法直接导入。GameMaker和GameMaker之间有很多不同的地方。Studio 1.4之间有许多不同之处，因此你的老项目可能需要一些调整才能工作。请参阅GameMaker知识库中的《将GM:S 1.4游戏移植到GameMaker》一文，以及手册中关于过时的功能和兼容脚本的部分。

重要的是!导入一个包含font 资源的 *.yyz 项目可能需要你通过删除/重新添加它们来 "重新生成 "fonts ，或者在font 编辑器中改变font ，然后再把它设置回去。这是因为现有的font 信息在导出时被 *.yyz 所保留，而在以后的导入中或在另一台机器上，项目可能已经没有安装所需的fonts （或者可能有相同名称的不同font ）。只有当你看到font 导入后的渲染方式有问题时，才应该这样做。

  • 最近的项目- 你可以使用这个选项来查看最近使用的项目。选择它将重新打开 "起始页"（如果你关闭了它），你可以从那里的列表中选择一个最近使用的项目。
  • 保存项目- 这将把你的游戏保存到你创建项目时定义的相同位置（或者你从哪里打开它）。
  • 将项目另存为- 和上面一样，只是你要指定项目的位置和新名称。
  • 导出项目 - 这允许你将你的项目以文件或模板形式导出。
    • YYZ- 这将把项目导出为一个单一的 *.yyz 文件，对于发送副本给别人或保存备份非常有用。
    • 作为模板- 这将允许你把你的项目作为模板导出，以后你可以把它作为The Start Page上其他项目的基础。选择这个选项将显示以下提示。
      你应该先点击 "编辑模板信息"，确保你的模板信息是正确的，然后在第二次打开这个提示时选择 "导出"，把它保存为模板。然后你会在开始屏幕的模板菜单下看到它。
  • 首选项 - 选择这个将带你到首选项窗口，在那里你可以设置GameMaker的外观和感觉。
  • 新的集成开发环境--当你选择它时，一个新的IDE 的实例将被打开供你使用（它将在开始页上打开）。
  • 我的账户- 该选项将打开你的默认网络浏览器，并将你带到你的YoYo 账户页面。
  • 注销- 选择此选项将使你退出GameMaker。你需要用同一个许可证（或任何其他许可证）再次登录，以便以后能够使用该产品。

重要的是!如果你一直在使用GameMaker的免费许可证，然后购买了其他许可证，你可能需要退出IDE ，然后重新启动GameMaker并再次登录，以确保新的许可证被正确应用（尽管这应该自动发生）。

  • Exit - 退出GameMaker到桌面。
