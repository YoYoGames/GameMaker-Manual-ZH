Asset 浏览器

你的游戏的核心将从assets ，添加到默认位于IDE右侧的资产浏览器中。在这里你可以添加你的游戏运行所需的一切，包括游戏room 、精灵、objects 、paths 和其他一些东西。在GameMaker中，一个基本的游戏将需要一个room 来运行（新的项目在创建时总是已经创建了一个room asset ），通常至少有一个object 和一个sprite ，尽管你可能会用到更多的东西。

Asset 浏览器是你为你的游戏创建和组织assets ，以及修改某些其他方面，如平台特定设置和配置的地方。下面你可以找到对每个不同部分的解释。

搜索栏

搜索栏是你可以键入一些文字，并使用它来过滤下面的主asset 列表中显示的内容。例如，输入 "房间 "将显示所有在其名称中包含 "房间 "的assets 。要将asset 列表重新设置为默认视图，只需删除搜索栏中的任何文字。

添加资产

通过添加Asset 按钮，你可以选择在asset 列表中添加一个新的asset 。点击这个按钮将打开以下窗口。

你可以通过点击 列表图标或 网格图标来选择使用上述两种视图中的哪一种，你还可以点击 导入图标，从另一个项目中导入已经存在的asset 。如果你不想导入任何东西，那么只要选择你想创建的asset ，然后点击底部的创建按钮。请注意，你在这里并不局限于只添加一个asset ，因为你可以在窗口的底部设置要创建的数量，然后点击创建，该数量的所选asset 将为你添加到asset 浏览器中。所有以这种方式创建的assets ，将被添加到asset 列表的底部，在任何文件夹之外，也将在当前的workspace 中打开，准备进行编辑。

你可以在这里添加的assets ，在本节手册中进一步列出。

过滤器

点击Asset 过滤器按钮将打开以下窗口。

在这里你可以选择如何过滤和显示assets 浏览器中列出的asset 。你可以从A-Z或从Z-A对它们进行排序，也可以选择先对组进行排序，然后对组内的assets ，或对所有东西进行排序。如果你选择了自定义顺序过滤器，那么你可以选择一个或多个assets ，并在asset 浏览器树中向上或向下拖动，按你的要求定位（注意 "组先 "选项将被禁用）。

你也可以选择一个或多个asset 类型进行过滤，只有这些类型的项目中的assets 才会显示在asset 列表中。选择 "任何类型 "将重置过滤器。你也可以使用你指定的任何标签进行过滤（标签将在本手册的下一节讨论）。请注意，如果你选择了除 "任何类型 "以外的任何过滤器，那么过滤器按钮将改变颜色，以便直观地显示过滤正在进行。

  • 游戏选项。在这里你可以快速访问你项目的各种不同的游戏选项。这些选项控制项目中与设置和编译不同目标平台有关的各种不同方面（哪些游戏选项可用取决于你的许可证类型）。你可以在 "游戏选项"一节中找到关于每个选项的更多信息。

标签

正如以上各节所提到的，asset 浏览器让你能够为单个assets ，以及为分组文件夹添加标签。标签是一个非常强大的工具，不仅可以组织和过滤assets 浏览器中列出的asset ，而且在一般的游戏编程中也是如此，因为你可以在代码中使用指定标签。

要创建一个标签，你需要在asset ，然后选择编辑标签选项，使用人民币 菜单。这将弹出标签编辑器，你可以在那里输入你想使用的标签，然后按 ，或输入一个逗号来应用它。

你可以为一个项目指定多个标签，你也可以使用 或 / 加上鼠标左键 ，选择多个assets ，并一次为所有项目指定一个标签。一旦添加，所有的标签将在快速访问菜单和过滤选项中可见（如上文各节所述），因此可以用来快速过滤asset 列表，只显示那些带有所选标签的项目。

将一个标签分配给一个文件夹组，不会 自动分配给它里面的 。assets

需要注意的是，当标签是活动过滤器的一部分时创建的assets ，将自动被赋予这些标签。这意味着，如果你在 "一级 "标签下查看所有assets ，例如，所有在这个过滤器激活的情况下创建的新assets ，在创建时也会被添加 "一级 "标签。

最后，标签也可以在你的项目代码中使用，以识别任何asset ，然后对其采取行动，这使得它成为一个非常强大的工具，因为你可以根据一个标签或一组标签来编码行为。例如，设想你有一个带有 "Die On Touch" 标签的项目。在对项目进行编程时，你可以检查这个标签并编码适当的功能，然后艺术家和关卡设计师可以创建任何类型的新assets ，并用"Die On Touch" 标签来标记它们，然后在设计关卡时使用asset 。任何被编程为使用这个标签的代码都会自动包括这些新的assets 。

关于在项目代码中使用标签的更多信息，请看资产和标签部分。

在使用Asset 浏览器时，你可以在任何资源或文件夹上使用鼠标右键 ，打开一个选项菜单。

你可用的确切选项将取决于被点击的上下文，下面我们展示了在asset 列表中的Asset 或文件夹组上打开菜单时的内容（其他上下文将显示其中的一些内容，但不是全部）。

  • 从我的库中添加现有的- 从你的assets 库中的一个项目将asset 添加到Marketplace 列表中。请注意，使用2.3.0之前的版本创建的Marketplace assets ，将需要转换，因为项目文件格式随着2.3.0的更新而改变。

  • 创建组- 在asset 列表中添加一个新的文件夹组。

资产类型

现在你知道了Asset 浏览器是如何工作的，你应该了解一下你可以添加到其中的不同类型的asset 。下面我们将概述每一种不同的asset 类型，供你在你的项目中使用。

动画曲线

animation curve 是一种资源，它包含一条或多条曲线，表示一个数值如何随时间变化，可以使用线性插值或猫头鹰插值来在曲线中的数值之间进行。你设置的值在纵轴上可以在-1到1之间，而横轴上的持续时间总是以0到1的形式归一化，这使得你在游戏代码中使用乘法器很容易针对不同的时间范围。 Animation curves Animation curve assets 是由 "通道 "组成的，每个通道都可以有自己的曲线设置，这允许你--例如--用两个通道来描述一个空间位置的x/y位置，或者用4个通道来表示一个颜色格式的颜色梯度。在使用sequences ，也可以用代码来访问，使它们成为创建游戏的强大工具。

你可以从动画曲线部分找到更多关于添加animation curves 。

扩展是额外的文件和代码，你可以添加这些文件和代码来扩展GameMaker的功能，也可以从市场上为你的游戏添加assets 。extension 的格式将根据最终的目标平台而有所不同，不过一般来说，它们是用C++、C#或JavaScript编写的。

你可以从扩展编辑器一节中了解到更多关于添加extensions 。

字体

当你想在你的游戏中绘制文本时，这些文本将默认以标准的Arial 12点font ，但为了使文本看起来更有趣或独特，你可能想使用不同的fonts 。这就是使用字体编辑器的地方。在这里，你可以选择你电脑上的一个font ，并把它导入GameMaker，以便在你的游戏中使用，设置各种属性，如大小和重量或风格。一旦导入并添加为资源，你就可以使用适当的代码或动作来设置它的绘制。

注意：如果你要在你的游戏中添加fonts ，请确保你有必要的许可证来重新发布它，除非它是公共领域或无版权。

fonts 的人民币 菜单包含以下选项（以及本页面上进一步列出的选项）。

  • Regenerate Font- 通过这个选项，你可以强迫GameMaker重新生成font cache 。这通常只在font 损坏或在系统上发生变化时才需要。

你可以从字体编辑器一节中了解到更多关于添加fonts 。

物品

对象是一种特殊的资源，我们用它来控制游戏的各个方面，做一些特定的事情。大多数时候，它们都有与之相关的精灵，以便你在游戏中看到它们room ，但有时它们被用作 "幕后 "控制器，做与用户有关的事情或用于计时，等等。它们可以被赋予行为，它们可以对某些事件以及相互之间做出反应，你在游戏中看到的大多数东西都是基于objects 和它们的互动。objects请注意，我们说 "基于 "是因为你实际上并没有直接将room ，而是将这些objects ，这些实例 基本上是object 资源的副本（或克隆，如果你愿意）。这是一个非常重要的事情，要记住，因为实例和objects 是不一样的，各自都有自己的一套功能可以影响它们。

所有的objects ，都有一组属性，你可以在object 编辑器中指定，比如sprite ，它是否使用物理学，或者它是否是另一个object 的 "子"。它们也有一系列内置变量，然后可以在动作、scripts 和代码中使用。这些内置变量被用来定义实例在游戏中的位置room ，动画速度，运动方向和其他一系列的东西。为了控制object ，每个游戏帧被分成一系列的事件，所以你可以将代码或动作添加到一个特定的事件中，只有当事件被触发时才会运行，事件可以由鼠标按键或与其他实例的碰撞等情况触发。

对象在一个非常真实的意义上是你的游戏的组成部分，与rooms 一起将构成你所创建的任何项目的核心。

objects 的人民币 菜单包含以下选项（以及本页面上进一步列出的选项）。

  • 打开所有事件脚本- 选择此选项将在新的object ，以workspace 标签的形式打开所有分配给script 的事件。

你可以从 "对象编辑器"一节中了解到更多关于添加objects 。

路径

在创建你的游戏时，你可能需要你的一个objects 的实例来跟随一个path ，通过一个关卡。现在，这可以通过简单地在room 中创建一个位置数组来实现，然后让实例在它们之间移动，但这可能是一个麻烦的设置，而且很难改变、测试和适应不同的事情。这时，你会想要创建一个path 资源。

path 资源的基本概念相当简单--你通过在path 编辑器中绘制它来定义一个path ，然后你可以在一个object 的事件中放置一个动作（或代码）来告诉object 在游戏中跟随那个特定的path room 。你可以设置跟随path 的速度和其他一些与path 在room 内的位置和方向有关的动作。

你可以从 "路径编辑器"一节中了解到更多关于添加paths 。

房间

你在GameMaker中制作的所有游戏至少需要一个room 来运行（但可以有很多很多），而一个room 基本上是一个空间，你把构成你的游戏的objects 的实例放在那里，游戏的动作将发生在那里。然而，Room 编辑器是你可用的最强大的资源之一，因为它不仅允许你添加objects 的实例，而且可以设置进入游戏的 "视图 "room ，添加背景和tiles ，以创建游戏世界，创建独立于对象/实例系统的额外的特殊效果，以及独立于object 的运行代码。 Rooms 也可以设置为继承 其他rooms 的属性，这意味着你可以创建一个单一的room ，并加载tiles ，例如，然后制作另一个room ，inherits 这些tiles ，这样你就不必再重新创建它们了。

rooms 的人民币 菜单包含以下选项（以及本页面上进一步列出的选项）。

  • 创建子房- 这将创建一个新的room ，作为所选room 的子房。rooms 子房room 是新的assets inherit ，它的属性、层和层内容来自父房room ，并将在房间管理器中显示与父房room 的链接。

你可以从房间编辑器一节中了解更多关于添加rooms 。

脚本

script 资源只是一个容器，用于存放你编写的一个或多个函数。例如，假设你想让你所有的敌人objects ，当他们死亡时创造一些不同的实例--比如一些金子、爆炸和一些血以及播放声音。现在你可以把所有需要的代码添加到每个需要的object ，但一般来说，你最好在script 资源中创建一个单一的函数，然后调用它。这意味着你只需要在objects 中添加一行代码来调用这个新的脚本函数，如果你想改变什么，你只需要在script 中改变一次，所有的objects 都会自动运行新的代码，而不是在每个object ，在多个地方改变同一个东西。

scripts 的人民币 菜单包含以下选项（以及本页面上进一步列出的选项）。

  • 转换为GML Visual / GML- 这将把选定的script 转换为GML Visual 或GML 。如果script 是GML ，那么它将被放在一个执行代码的动作中，而GML Visual 将被直接转化为GML 代码（如改变GML Visual 为代码一节所示）。

你可以从关于脚本编辑器的章节中了解更多关于添加scripts 。

序列

一个sequence asset ，本质上是一个其他资源的集合，随着时间的推移执行动态动画。它们可以包含sprites 、实例、声音和其他东西，其中的每一个都可以通过将sequence 分配给assets 中的特定轨道 ，在sequence 本身中设置为移动或改变颜色，或随着时间的推移而开始/停止。这些轨道然后可以有不同的属性应用于它们，它们可以是静态的或随时间变化。sequence 的所有编辑工作都在序列编辑器中进行，该编辑器由以下部分组成。

  • 画布-- 你把构成序列的assets 。
  • 轨道面板--在这里你可以放置assets ，创建asset 轨道，也可以创建参数轨道，以便随着时间的推移操纵assets 。
  • Dope Sheet- 这是你添加/删除/编辑每个轨道的keyframes ，并控制它们随时间变化的行为。

sequences 请注意，你可以在sequences ，以创建复杂的动画和效果，而且sequences ，也可以通过代码访问和改变，让你尽可能多地控制它们在游戏中的显示和使用方式。

你可以从 序列编辑器一节中了解更多关于添加sequences 。

着色器

着色器是一个非常强大的工具，可以用来操纵你的游戏渲染到屏幕上的图形，允许令人难以置信的快速效果，范围可以从，例如，添加一个微妙的颜色色调到一个sprite ，直到全屏扭曲效果。基本上，它是一个由两部分组成的程序，直接在显卡上运行，由于GPU在做所有的工作，所以速度非常快，并为你的游戏代码释放了CPU周期。完整的shader 是由一个vertex shader 程序和一个片段shader 程序（也被称为像素shader ）组成的。这两个小程序一起工作，以操纵显卡渲染到屏幕上的内容。这样，你就可以实时操纵实际渲染到显示屏上的位置、颜色和阿尔法值，buffer 。

声音

声音是任何游戏的重要组成部分，既可以增加游戏的深度，以声音效果的形式给出反馈，也可以以音乐的形式增加气氛。GameMaker接受 WAV, MP3 和 OGG 格式的文件。

一般来说， WAV 文件用于任何简短的声音效果，因为尽管它们通常是较大的文件，但由于它们不需要任何类型的解码来播放，所以可以立即播放。 MP3 和 OGG 文件应该用于背景音乐或任何播放时间较长或文件大小较大的效果。这些文件比WAV文件小得多，但由于它们在播放前必须进行解码，因此有一个与之相关的CPU开销。

人民币 ，声音的菜单包含以下选项（以及本页面上进一步列出的选项）。

时间表

在大多数游戏中，你会希望某些事情在特定的时间点发生。现在，你可以尝试通过使用实例中的报警事件来实现这一点，但当事情变得太复杂时，这就不再起作用了，尤其是你被限制在只有12个报警。这就是为什么我们有timeline 资源。在一个timeline ，你指定哪些动作必须在游戏时间的任何特定时刻发生，你可以在一个object ，在其不同的事件以及代码中使用所有可用的动作。一旦你创建了一条时间线，你就可以把它分配给一个object ，然后该object 的实例就会在指定的时间点上执行动作和代码，当放在或创建在一个room 。

你可以从时间线编辑器一节中了解更多关于添加timelines 。
