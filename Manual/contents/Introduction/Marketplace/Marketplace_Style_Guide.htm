市场风格指南

本页是为那些为GameMakerMarketplace创建资产和服务的人准备的。在这里，我们给你一个简短的指南，帮助你在设置资产清单时描述和塑造你的品牌assets 。

虽然遵循这些准则并不是完全必要的，但它会 使你的asset 更受终端用户欢迎，我们也更有可能在asset 的主页上介绍你的Marketplace 。关于你的asset 本身的内容的一些更一般的准则，你也可能想阅读本指南。

显示名称

你的显示名称必须是你的asset ，并且不包含任何商标词，除非你自己是商标的持有人。你应该记住，虽然你的显示名称应该充分描述你的产品，但只有大约11个字符可以容纳在主商店页面上显示的方框中。关于这一点，请看下面的 "图标 "部分。

描述

你的asset 的描述应该充分说明你的asset 的用途，如果你没有在asset 本身中包括这些信息，那么它是如何使用的。描述不一定要长，但一句话除了重复显示名称外，可能不会吸引很多客户如果你的asset 包是基于图形的，你可能想列出它包含的图像类型。如果asset 是一张音乐专辑，你可能希望提供一个曲目列表。如果asset 是基于代码的，通常有助于添加一个简短的代码块，显示你可以使用它的类型，等等，等等。

如果你格式正确，使用以下标记，你的描述也会更加醒目。

  • 斜体字。要创建斜体文本，请将您的文本放在一对星号（*）之间，例如。

    *这个文本是斜体的*。
  • 粗体字。要创建粗体字，请将您的文字放在两对星号（*）之间，例如：。

    **这个文本是粗体的**。
  • 粗体和斜体文字。要创建黑体-斜体文本，请将您的文本放在三对星号（*）之间，例如。

    ***此文本为粗体和斜体***
  • URL参考。要创建URL引用，你要把你想要链接的文本放在方括号[ ]之间，然后像这样用清晰的间距来引用链接。

    这是一个通往网页的[链接]。
    [链接]：http://gamemaker.io

    提示。为了使你的页面更加整洁，最好将参考文献放在页面的底部。
  • 子弹型列表。要创建一个子弹头列表，请确保在列表的上方和下方有一个换行符，然后用星号（*）或连字符（-）开始每一行，例如。

    * 列表第一项
    * 列表二
    * 列表三
  • 编号列表。要创建一个有编号的列表，请确保在列表的上方和下方有一个换行符，然后用连续的数字开始每一行，例如。

    1.列表项一
    2.列出第二项
    3.列出第三项
  • 代码块。要创建一个缩进的代码块，请确保在代码的上方和下方有一个换行符，然后每行以四个 空格开始，例如。

    Lorem ipsum 看一看我的代码。

    game_maker_function(arguments)。
    yet_another_function(more, arguments, here)。

关键词

当决定你的asset 的关键词时，尽量使用人们通常会搜索的与你的asset 有关的词和一些使你的asset 独特的词的混合。例如，对于一个平台引擎来说，一组好的关键词可能是。平台、跳跃 和重力 ，以及描述你自己独特的asset ，如僵尸、太空 或老虎。你可能还想在每个asset ，包括一个代表你的出版商名称的单一标签，以确保你的assets ，如果用户专门寻找由你设计的assets 。

专题图文

如果你想让你的asset ，在Marketplace 主页上的主要展示片中出现，那么专题图是必须的。网站本身不会在专题图上显示标题或说明，因此，在专题图上包括你的asset 的标题和简短的说明总是一个好主意。例如，如果你创建了一个shader ，仅仅使用一个普通的屏幕截图作为特征图形可能是不够的，因为用户可能不会意识到asset 是一个shader ，一个script 还是图片中使用的sprites 。当在你的图形中添加文字或重要的图标时，你应该牢记页面上的其他项目。下面是一张图片，代表了在你的特色图形上包括重要文字和图像的一个体面的安全区。

图标

你的游戏的图标是非常重要的，因为它可以在所有的类别列表中看到，是向浏览市场主页面的人宣传你的asset 的唯一真正方式。你的图标应该对每个资产都是独一无二的；在每一个asset ，重复使用你的出版商标志图标，只会让潜在的消费者一目了然，并不能真正宣传每个单独的asset 。你还应该注意，你的资产的显示标题在市场列表中会被截断到11个字符左右，因此，如果你的asset 的名称相当长，你可能希望在你的图像中再次包括它，以便用户可以看到全文。请看下面一个很好的例子，asset 的图标。

屏幕截图

截图是很多人都在纠结的事情，尤其是当他们的asset 不是基于图形的时候。然而，屏幕截图对于塑造你的asset ，即使它完全是基于代码的，也很重要。以下是为你的市场asset 创建屏幕截图的一些 "该做 "和 "不该做"。

做的事。

  • 如果你的作品注重视觉效果，请附上你的asset 的模拟图或屏幕截图。
  • 请自由地将你的asset 在GameMakerIDE内的截图，只要IDE 不是截图的主要焦点。注意确保在这些截图中，其他程序和图标不会出现在GameMaker ：例如，在你的电脑的任务栏中。
  • 如果需要的话，请附上贵公司的标志和一些描述使用贵公司的主要优势的文字asset 。

不要的。

  • 不要 在截图中抱怨截图是强制性的，或带有 "空白"、"截图"、"占位 "或其他不值得自己截图的内容的信息。如果你的asset ，并不重在视觉效果，用你的截图来解释更多关于产品的内容。如果你需要的话，将描述复制并粘贴到图片中，加上一个漂亮的背景或你的出版商标志的副本。任何与你的asset 有关的东西都比一个笑话或一个空白屏幕好。
  • 不要 在你的截图中使用YoYo Games 徽标、YoYo Games 名称、GameMaker 名称或任何其他YoYo Games 官方图像或文字标记。使用商店的人已经知道asset 与GameMaker ，包括YoYo Games 的标志可能会混淆你的asset 的品牌形象。这一点也适用于其他图像，如你的特色图形和你的asset 图标。然而，如上所述，如果你的GameMaker 需要，你可以显示IDE asset 本身的操作。
  • 不要 在你的截图中包括有版权的图片，如现有游戏中的sprites 。如果你的asset 是一个平台引擎，但你在艺术方面不是很擅长，不要被诱惑使用现有游戏中的sprites 作为放置物。如果你的asset 被标记为侵犯他人或公司的权利，你的asset 可能会被从商店中删除。

视频

视频预告片是推销你的asset 的一个好方法。如果你的asset 包含声音文件作为主要卖点，这一点尤其重要，因为这是用户在下载asset 。如果你的asset 是基于声音的，你应该在视频预告片的音频中包括几个声音片段或一个相当大的音乐。

如果你的asset 是基于代码的，那么看到asset 的操作可能会有帮助，即使这意味着只是从你自己的GameMaker 的副本中录制15秒左右的游戏录像。

最好的视频是宽屏，至少是1280 x 720像素，尽管这并不太重要，只要你的asset ，就能得到公平的体现。

上述针对截图的品牌准则也适用于视频，除非得到许可，否则你不应该在视频中加入GameMaker ，或其他人的知识产权。


遵循上述准则并不能保证你的asset ，但肯定会有很大的帮助，让你的assets ，因为他们可能是可取的！绝对没有坏处。我们希望你觉得这篇文章有帮助，像往常一样，如果你有任何问题，请随时提交票据或在社交媒体上询问我们。
