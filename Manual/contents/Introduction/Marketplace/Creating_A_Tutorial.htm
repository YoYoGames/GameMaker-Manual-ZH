创建一个Marketplace 教程

创建一个教程与创建一个普通的Marketplace asset 包类似，只是现在你必须加入特定的包含文件，以及包括一个初始的教程页面（和一个可选的视频页面）。教程workspace ，看起来会是这样的：正如上面对一般Marketplace Assets 所解释的那样，你需要选择一个出版商，然后给你的教程起一个名字和一个唯一的ID（反向URL格式），然后在 "类型 "下拉菜单中选择教程。这将为您提供额外的选项，以添加一个基本的HTML文件来启动教程，以及一个（可选）视频文件和URL。视频文件应该使用Marketplace 中提供的模板--就像教程的其他部分一样--你可以在这里获得，当用户加载教程时，视频将被加载到IDE 中的一个单独的框架中。 注意，你为开始页和视频页提供的页面是包含文件中的相对paths ，你可以打开资源管理器，选择GameMaker填充字段的默认页面以外的内容。

实际上，创建教程需要在教程项目中添加 "包含文件"，这些文件必须以非常具体的方式添加，以便IDE ，正确检测哪些文件将被用于教程中。基本的文件夹结构如下图所示。

因此，我们有两个主要的文件夹。

  • 教程内容。所有的教程 HTML 、用于设计风格的 CSS 、所需的图片（不是资源所需的图片，而是在教程中显示的图片），以及视频模板（如果需要），都放在这里。如果你想把教程分成几个章节，你也可以有子文件夹，但你至少应该有一个基本的 HTML 文件，作为教程的起始页。
  • 教程资源。这个文件夹中还有两个子文件夹，分别是图像 和声音 资产。如果你的教程需要用户添加一个sprite （例如），并且你想把它包括在教程中，那么你将把图像资源放在图像文件夹中。这些文件夹被导入到教程项目中，当用户打开sprite 编辑器并去添加图像时，资源管理器窗口的默认path ，这将使用户更容易找到必要的assets 。

为了使你的教程在IDE ，你必须有这样的文件夹结构，以及有一个基本的 HTML 开始页，并有一个 CSS 文件来进行样式设计。在你的教程项目中可以使用一些特殊的链接，这些链接可以跳到视频的某些部分，或者打开特定章节或功能的手册，这些都在Marketplace 的官方教程模板中给出。

重要的是!我们强烈建议你在尝试创建你自己的教程之前，从市场上下载教程模板。它们包含了HTML的所有重要部分的例子，以及CSS和图像，可以在教程中重新创建按钮和菜单，让你自由地担心内容，而不是如何显示它。

一旦你设置了包含的文件并提供了所需的信息，你可以点击"好"，教程将被上传到Marketplace ，完成后在workspace 的列表中打开一个asset ，这样你就可以编辑它并添加图标、文字、屏幕截图等。一旦教程asset ，用户就可以在 "起始页"的 "教程 "部分找到它，可以在那里下载和安装。
