创建一个Marketplace 资产

当你从IDE 市场菜单中选择 创建软件包选项时，你将被带到以下工作区： 这里你必须首先选择你想发布asset 包的发布者身份（你应该已经在你的发布者控制面板上设置了这个身份），然后给出一个显示名称和一个版本号。然后，将根据显示名称和使用的出版商账户为你创建一个软件包ID，但如果你愿意，可以编辑成其他内容，只要保持反向URL格式即可。然后你需要选择要制作的asset 包的类型。

  • 资产：这是你希望在市场上出售的assets (sprites,scripts,objects, 等等...) 的一个集合。
  • 演示。这是一个演示，用于展示一些概念或游戏理念。
  • 教程。正在上传的asset 是一个教程，旨在从GameMaker中运行IDE （更多细节见此页面）。

一旦你完成了这一工作，你就可以通过右边的部分选择你想从项目中添加到assets 包中的哪些Asset 。你可以添加所有的assets ，或者只添加其中的一部分，也可以删除你可能错误添加的任何内容。假设你已经在GameMaker中创建了要上传的asset ，并且当前项目包含了你需要上传的所有内容。

请记住!为了能够创建或使用asset 包，你必须事先创建一个用户配置文件证书。你可以从Marketplace 首选项中做到这一点，这里有解释。

当你对选择的assets ，并希望将你的包上传到Marketplace ，只需点击底部的 "好"按钮，asset 包将被创建并上传到Marketplace 。当上传完成后，一个新的workspace ，将打开一个浏览器视图的asset 页面供你编辑。关于如何设置的更多信息，请见以下页面。

  • 列出资产

如果你需要更新一个Marketplace Asset ，你所做的和你创建原始包时差不多，只是现在你需要从一个可用包的下拉列表中指定要更新的包。

当更新时，你将不能改变包的ID，你应该注意显示的版本号将是当前的 版本号，你应该适当地增加它，否则更新将失败（因为你不能有两个具有相同版本号的资产包）。还有一个字段，你可以为asset 的更改日志添加一些文本，这将与asset 一起上传，并显示在Marketplace 页面上。

重要提示！用GameMaker2.3+版本制作的Assets ，只对那些使用2.3及以上版本的用户有效。如果你正在更新以前版本的asset ，该asset 仍将提供给未更新的用户，并将与更新的2.3+版本共存。

通过网页上传

你可以通过实际的assets 网页而不是Marketplace 上传IDE ，如果通过IDE 上传因某种原因而失败，这可能是必要的。首先，你要创建软件包，并尝试通过IDE ，如上图所示，如果失败了，你就去输出窗口，检查日志中是否有一行显示你所创建的软件包在尝试上传前被保存到哪里。比如说。

在Windows上。

Created package file:
C:\Users\WINDOWSUSERNAME\AppData\Roaming\GameMaker\GM_USERNAME\Marketplace\Packages\com.publishername.assetname.yymp

在MacOS上。

Created package file:
/Users/MACUSERNAME/.config/GameMaker/GM_USERNAME/Marketplace/Packages/com.publishername.assetname.yymp

你需要导航到该文件位置，并将 *.YYMP 文件复制到一个安全位置，因为这将是你要上传到Marketplace 的文件。

Marketplace 这时，你需要在你的网络浏览器中打开asset ，并浏览到asset ，然后进入版本部分，然后点击上传版本按钮，如下图所示。

这将带你到新版本的上传页面，如果需要的话，你可以在这里添加关于正在上传的版本的信息，还可以通过点击Choose FIle按钮并选择先前创建的 *.YYMP 文件来提供该文件。

添加文件将上传版本，并显示一个 "进度 "条，当上传完成并对细节满意时，你就可以点击 "保存"按钮，将asset 到列表中。请注意，asset 将在处理阶段停留数分钟，然后才能发布。
