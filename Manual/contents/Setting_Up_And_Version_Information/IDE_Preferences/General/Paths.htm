路径偏好

路径 paths 偏好控制GameMaker中的元素所需要的不同文件IDE ，以及你需要的任何外部编辑器。在IDE中存在以下选项。

  • 临时目录。保存所有临时文件夹的位置。默认情况下，在Windows ，这是

    \Users\<Username>\AppData\Local\Temp\GameMaker

    而在macOS ，它是。

    /var/folders/<Path_Hash>/GameMaker

  • IDEcache 目录。IDE cache 的保存位置。默认情况下，在Windows 上这是

    \Users\<Username>\AppData\Roaming\GameMaker\Cache

    而在macOS ，它是。

    /Users/<Username>/.config/GameMaker/Cache

  • 资产cache 目录。为每个项目保存asset cache 的位置。默认情况下，在Windows ，这是

    \Users\<Username>\AppData\Roaming\GameMaker\Cache

    而在macOS ，它是。

    /Users/<Username>/.config/GameMaker/Cache

  • "我的项目 "位置。GameMaker最初创建新项目的位置。默认情况下，在Windows ，这是

    \Users\<Username>\Documents\GameMaker

    而在macOS ，它是

    /Users/<Username>/GameMaker

  • 关闭时自动删除临时目录。勾选此选项将强制GameMaker自动删除它为每个项目创建的用于编译等的临时文件夹...。这个设置是默认开启的，取消勾选它将switch （意味着你以后必须手动删除任何临时文件）。
  • 关闭时自动删除asset cache 。如果勾选这个选项，那么当你退出GameMaker时，asset 编译器cache 文件夹将被删除。这在默认情况下是关闭的，启用它将意味着每次你加载和运行任何项目时，cache 都需要被重建（这可能需要时间，取决于游戏的大小）。
  • 关闭时自动删除IDE cache 。如果选中此项，那么当你退出GameMaker时，IDE 编译器cache 文件夹将被删除。默认情况下这是关闭的，启用它将意味着每次启动GameMaker时，cache 都需要重建。
  • 删除临时文件夹。点击这个按钮将删除项目的临时文件夹。
  • 删除Asset 缓存。点击这个按钮将删除项目的编译器asset cache 。
  • 删除IDE 缓存。点击此按钮将删除IDE 缓存。

在设置paths 到外部编辑方面有以下选项。

  • SWF文件的外部编辑器/查看器的路径。如果你正在使用SWF格式sprites ，你可以将此设置为你喜欢的查看器/编辑器的path ，当你点击Sprite 编辑器中的编辑图像按钮，那么它将打开给定的程序。这里的默认值是没有path 。
  • Spine 文件的外部编辑器/查看器的路径。如果你正在使用Spine 格式的sprites ，你可以将此设置为你喜欢的查看器/编辑器的path ，当你点击Sprite 编辑器中的编辑图像按钮，那么它将打开给定的程序。这里的默认值是没有path 。
