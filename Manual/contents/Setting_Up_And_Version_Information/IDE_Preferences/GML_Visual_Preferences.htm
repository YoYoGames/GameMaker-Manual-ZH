GML视觉首选项

倾向性 GML视觉首选项是用来定义Visual编辑器的外观和感觉。下面列出了不同的选项，以及每个选项的作用描述。

  • GML Visual工具箱的默认位置。默认情况下，GML Visual工具箱（所有不同的动作库都在其中）被放在编辑器的右边，但你可以使用这个选项让它默认在左边。
  • GML Visual 节点概述的可见性模式。节点 "概览 "是在行动Workspace ，例如这个Destroy Instance行动中某些行动旁边出现的简短描述性文字。

    你可以选择始终启用这些，或者只在动作被折叠时启用（默认设置），或者完全禁用它们。
  • 自动响应GML Visual转换为GameMaker语言。当你将一个GML Visual事件或script 到GML 代码时，默认情况下你会收到一条信息，警告你将要做什么，并询问你是否要继续。把这个选项设置为 "OK "将简单地同意改变，而不显示消息。
  • 对GML Visual节点删除的自动响应。当你删除一个GML Visual事件节点时，默认情况下，你会收到一条消息，警告你将要做什么，并询问你是否要继续。把这个选项设置为 "OK"，就会简单地同意删除而不显示消息。
  • GML Visual工具箱动作缩略图的最大尺寸。在动作工具箱的底部，你有一个滑块来改变图标的大小。GML Visual图标的大小。这个设置将被用来定义允许的最大尺寸（默认64px）。
  • GML Visual工具箱动作缩略图的最小尺寸。在动作工具箱的底部，你有一个滑块来改变图标的大小。GML Visual图标的大小。这个设置将被用来定义允许的最小尺寸（默认为16px）。
  • GML Visual工具箱动作缩略图的默认尺寸。这个设置将被用来定义工具箱中所有动作图标的默认尺寸（默认为32px）。
  • 显示最近使用的动作的最大数量。在动作工具箱的顶部有一个库，里面有你最近使用过的动作。你可以用这个选项来改变将显示的最近动作的最大数量（默认为5个）。
  • 连接GML Visual动作的链的厚度。这定义了在一个代码块中连接不同动作的链状图形的厚度。GML视觉代码块中不同动作的链状图形的厚度（默认为3px）。
  • 鼠标放下开始拖动节点的时间（ms）。当你点击一个节点并拖动它时，你将在动作代码内重新定位该节点。然而，如果你按住鼠标片刻，然后移动，你将在workspace ，但不在代码的顺序内移动节点位置。这个设置设置了在按下鼠标按钮时必须等待的毫秒数，然后才可以移动节点（不是重新定位）。默认值是350。
  • 隐藏GML Visual节点概述面板。这里你可以选择隐藏动作代码编辑器的概述面板。默认情况下这是禁用的，启用它将从编辑器中删除动作链的纯文本列表。
