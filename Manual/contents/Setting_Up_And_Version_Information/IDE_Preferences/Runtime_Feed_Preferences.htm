运行时喂食偏好

Runtime Feeds 首选项用于显示和改变当前安装的Runtime以及当前使用的runtime 。GameMaker可以在任何时候安装几个不同的runtime 工具，你可以在这个首选项中设置使用哪一个。理想情况下，你希望使用最新的runtime ，但有时它可能会有问题，或者你可能需要使用较旧的工具来进行比较，等等......因此你可以在这里改变GameMaker用来构建游戏的runtime 工具。

这个偏好的最初部分与你从哪里获取runtime 工具有关，需要你输入一个RSSfeed的URL。然后，这个feed将被扫描，以获得runtime 工具的更新，然后可以下载和安装。这一部分默认显示主runtime feed URL，但是你可以点击添加 按钮来创建一个新的Runtime Feed，在这里你可以给RSS feed的名称和URL，然后点击应用来告诉GameMaker访问它。你需要关闭并重新打开首选项窗口以显示新的Feed。

一旦创建了Runtime 饲料，它将作为一个子类别列在本节中，而且你应该始终有至少一个饲料被列出（默认情况下，这将是主 饲料）。无论使用何种runtime ，它们都将具有相同的功能，下面以主饲料为例概述。

Runtime 饲料偏好是用来选择、安装和卸载不同的 工具。在顶部，你可以看到当前正在使用的 工具，它的创建日期，以及它在硬盘驱动器上的存储位置。在这些细节下面，你可以找到一个所有可用的 ，你可以从这个列表中选择任何一个来改变为用于编译你的项目的 "运行 " 。runtime runtime runtimes runtime

要选择一个不同的runtime ，只需在列表中找到它，然后双击 。如果runtime 还没有下载和安装，那么会提示你这样做，一旦安装完毕，会要求你关闭并重新启动IDE ，以便新的runtime 可以被初始化。

这个列表详细列出了所有可用的runtimes ，它们的版本号，它们的发布日期，以及--如果它们被安装了--它们被安装到的path 。右手边还显示了不同的状态图标，如下所示。
