文本编辑器 编辑器首选项

文本编辑器首选项用于定义不同的代码/脚本编辑器的外观和感觉。主页面有以下选项。

  • 语法检查延迟（毫秒）。语法检查器运行前的延迟时间（以毫秒计）。这个默认值是2000ms。
  • 代码完成延迟（毫秒）。延迟时间（以毫秒计）。
  • 在全屏编辑器中打开scripts 。勾选此选项将使script 资源自动在一个新的workspace 。如果不勾选此选项，它们将在当前workspace 的一个窗口中打开。这个选项在默认情况下是不被选中的。
  • 在全屏编辑器中打开shaders 。勾选此选项将使shader 资源自动在一个新的workspace 。如果不勾选此选项，它们将在当前workspace 的一个窗口中打开。这个选项在默认情况下是不被选中的。
  • 在全屏编辑器中打开object 事件。勾选这个选项将使object 事件代码编辑器在一个新的workspace 中自动打开。如果不勾选这个选项，它们将在一个与object 事件编辑器连锁的窗口里打开。这个选项在默认情况下是不被选中的。
  • 打开事件scripts ，作为同一窗口中的标签。当添加代码或GML Visual到一个事件时，代码窗口被链到事件窗口，当这个选项被选中时，任何进一步被打开编辑的事件将被添加到同一窗口，作为标签沿着顶部。取消勾选该选项将使每个事件作为它自己独特的链式窗口打开。这个选项在默认情况下是被选中的。
  • 在同一窗口中以标签形式打开链接的scripts 。当你有多个链接的scripts ，并打开它们时，默认情况下，它们将被分组在同一个窗口中作为标签。取消该选项将迫使它们在各自的窗口/工作区中打开。该选项默认为选中。
  • 焦点时将文本编辑器平移到视图中。当你从一个object 事件中打开一个文本编辑器时，选择这个选项将自动平移workspace ，使编辑器进入边界内。这个选项在默认情况下是关闭的。
  • 默认代码窗口宽度：设置代码窗口打开时的默认宽度（单位：像素）。默认值为900。
  • 默认的代码窗口高度。设置代码窗口打开时的默认高度（单位：像素）。默认值是600。
  • 最小代码窗口宽度：设置代码窗口允许的最小宽度（像素）。默认值为300。
  • 最小代码窗口高度。设置代码窗口允许的最小高度（单位：像素）。默认值为200。

除了上述选项外，还有两个子类别，即代码编辑器和视觉编辑器，如下所述。

代码编辑器

颜色偏好是用来改变代码和script 编辑器显示不同文本的方式。每个条目都有一些子条目，允许你准确地定义每个单独的文本应该是什么样子，如上面的例子图片中所示。一般来说，你可以为每个不同的选项设置颜色、font 、大小和样式。再往下看，你还可以为诸如代码括号的行选择等设置更一般的样式。

在颜色偏好下，你有不同的造型选项。这些偏好用于设置代码/脚本编辑器的行为，有以下选项。

  • 保留制表符。如果选中，这将在编辑器中保留制表符--' \t'--，而不选中则意味着制表符被转换为空格。默认是关闭。
  • Tab的大小。按 "Tab "键应缩进代码的字符空间数。默认为4。
  • 显示空白处。显示 " ." 以显示空白（见下面的图片）。默认为关闭。
  • 显示行号。是否在左下方显示行号。默认为打开。
  • 显示标签线。显示" ...."，以显示标签空间。默认为关闭。
  • 启用颜色编码。启用或禁用编辑器的颜色编码。如果关闭，代码将不会被样式化，而当开启时，它将使用上面颜色部分给出的格式进行样式化。默认为开启。
  • 文档行数限制：为任何给定的编辑器文档设置最大行数。默认值是0（0或更少基本上是无限的），任何其他值将限制允许的行数。
  • 匹配新行的缩进。为你的代码启用或禁用自动缩进功能。默认为打开，当添加新的一行时，它将保持在一行上设置的任何制表符。
  • 大括号后自动缩进。当添加大括号时，启用或禁用代码的自动缩进功能 {} 。默认为打开，当添加新的大括号 { 和新的行时，将在下一行的代码上加标签。
  • 启用代码补全。启用或停用代码完成窗口。当编写代码并启用该功能时，你会得到一个弹出窗口，显示你可以根据当前输入的文本使用的可能的功能。默认为开启。
  • 自动添加函数括号。当你使用自动完成功能时，启用或禁用自动添加括号 () 。当编写代码并启用该功能时，IDE 将自动添加函数所需的两个大括号 () ，如果函数需要参数，则将光标置于其中；如果不需要参数，则将光标置于其后。这个功能只有在函数被从自动完成中选择时才起作用。默认情况是打开。
  • 撤销/重做堆栈限制：在这里你可以设置撤销/重做堆栈限制。任何低于0的值基本上都是无限的（受可用内存的限制），而正值将把堆栈限制在这个数字上，如果达到极限，将删除堆栈尾部的任何操作。默认值是-1。
  • 突出选择匹配。这将轻而易举地突出显示任何与其他地方重复的代码部分。因此，如果你选择了一个函数，例如，这个函数在当前代码窗口中的所有实例也会被突出显示。默认是打开。
  • 对过时的功能进行彩色编码/代码完成。这将突出显示任何一段已经过时的代码。默认为打开。

视觉编辑

注释偏好控制着被添加到GML Visual scripts 的任何注释的不同颜色和格式化选项。这些选项是

  • 颜色：被写入窗口的文本的默认颜色（默认ARGB#FF53E270）。
  • 字体名称。要使用的font （可用的fonts ，取决于你系统上安装的字体）。
  • 风格。要使用的font 风格，如粗体 或斜体 （注意，不是所有的窗口都允许相同的风格，而且还取决于所使用的font ）。
  • 大小。为给定的输出窗口设置font 的尺寸。
  • 抗锯齿。在这里你可以设置输出窗口的抗锯齿模式font 渲染。你可以选择让IDE ，通过选择使用默认值来决定，或者你可以通过选择这两个选项中的任何一个来强制打开 或关闭 它。
