设备管理器

通过点击目标列表中的铅笔按钮 ，可以访问设备管理器。

点击这个按钮将打开设备管理器，在这里你可以为你希望编译和测试的目标平台配置不同的设备（注意，可用的平台将取决于你的License，可能不是所有的平台都能看到）。下面你可以找到关于为每个目标平台配置设备的细节。

安卓

在这里你可以点击 添加新设备按钮，手动添加一个新的Android 设备到列表中，或者你可以点击 检测设备按钮，让GameMaker自动检测通过有线连接的任何设备。你还可以设置一个 安卓虚拟设备，点击标有 运行AVD。这将打开Android 虚拟设备管理器，你可以创建一个新的AVD（见 这里有更多信息）。一旦你创建了一个，你需要从AVD管理器中启动它，然后在GameMaker中点击 检测设备按钮，它应该被自动添加到设备列表中。

一旦你添加了一个设备（物理或虚拟）。 你就可以用 测试连接"按钮，或者通过点击 "删除设备 "按钮，再次从列表中删除它。 删除设备按钮。


HTML5

HTML5设备管理器是用来将不同的浏览器添加到设备列表中，以便在运行时进行定位。开始时，你应该 点击 检测浏览器按钮，这将为您机器上安装的每个浏览器创建设备。如果你的浏览器没有被列出，你也可以点击 添加新的浏览器按钮，然后给设备起个名字，把GameMaker指向浏览器可执行文件的path 。

可以设置额外的可用字段，以帮助控制浏览器如何运行你的项目，有以下选项。

  • 浏览器命令行参数。在这里，您可以添加任何命令行参数，以附加到浏览器path 。你可以从以下主要流行浏览器的链接中找到这些参数的清单。
    • Chromium (Chrome)
    • 火狐浏览器
    • 互联网浏览器
  • 网页Server 地址。网络服务器的地址 服务器的地址（默认是127.0.0.1）。你可能需要根据不同的API的规格来改变这个地址，例如，Facebook会要求将这个地址设置为 http://localhost (然后在他们的应用程序控制台上将其列入白名单)。
  • WebServer 端口: 设置Web 的 网络server 的端口号（默认是51264）。
  • 允许的网络Server IPs / CIDR IP范围。根据需要设置允许的网络server 、主机名和CIDR IP范围。
